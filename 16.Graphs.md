## 1. What is a Graph?

### 1.1 The Definition

A **Graph** `G = (V, E)` is a collection of **vertices (nodes)** and **edges (connections)**.

```
  1 --- 2        1 → 2
  |   / |        ↓   ↓
  |  /  |        3 → 4
  3 --- 4

UNDIRECTED             DIRECTED
Edge (1,2) = (2,1)     Edge (1,2) ≠ (2,1)
```

### 1.2 Graph Types

| Type | Description | Example |
|---|---|---|
| **Undirected** | Edges have no direction | Facebook friends |
| **Directed (Digraph)** | Edges have direction (arrows) | Twitter follows |
| **Weighted** | Edges have costs/weights | Road distances |
| **Unweighted** | All edges equal weight | Maze paths |
| **Cyclic** | Contains at least one cycle | Social networks |
| **Acyclic** | No cycles | DAG (dependency graphs) |
| **DAG** | Directed Acyclic Graph | Task scheduling, Git |
| **Tree** | Connected acyclic undirected graph | File systems |
| **Bipartite** | Nodes in 2 groups, edges only between | Matching problems |
| **Complete** | Every pair has an edge | K_n, fully connected |

### 1.3 Key Terminology

| Term | Definition |
|---|---|
| **Vertex/Node** | A point in the graph |
| **Edge** | Connection between two vertices |
| **Degree** | Number of edges connected to a node |
| **In-degree** (directed) | Number of incoming edges |
| **Out-degree** (directed) | Number of outgoing edges |
| **Path** | Sequence of vertices connected by edges |
| **Cycle** | Path that starts and ends at the same vertex |
| **Connected** (undirected) | Path exists between every pair |
| **Strongly Connected** (directed) | Path exists in BOTH directions for every pair |
| **Component** | Maximal connected subgraph |
| **Sparse** | Few edges (E ≈ V) |
| **Dense** | Many edges (E ≈ V²) |

### 1.4 Properties & Formulas

| Property | Formula |
|---|---|
| Max edges (undirected) | V × (V-1) / 2 |
| Max edges (directed) | V × (V-1) |
| Tree edges | V - 1 |
| Sum of degrees (undirected) | 2 × E |
| Sum of in-degrees = sum of out-degrees (directed) | = E |

## 2. Graph Representations

### 2.1 Adjacency List ⭐ (Most Common)

```cpp
// Unweighted
vector<vector<int>> adj(n);
adj[0].push_back(1);  // edge 0 → 1
adj[1].push_back(0);  // edge 1 → 0 (undirected: add both)

// Weighted
vector<vector<pair<int,int>>> adj(n);  // {neighbor, weight}
adj[0].push_back({1, 5});  // edge 0 → 1 with weight 5

// Building from edge list
void buildGraph(int n, vector<vector<int>>& edges) {
    vector<vector<int>> adj(n);
    for (auto& e : edges) {
        adj[e[0]].push_back(e[1]);
        adj[e[1]].push_back(e[0]);  // undirected
    }
}
```

### 2.2 Adjacency Matrix

```cpp
vector<vector<int>> matrix(n, vector<int>(n, 0));
matrix[0][1] = 1;  // edge 0 → 1
matrix[1][0] = 1;  // edge 1 → 0 (undirected)
// For weighted: matrix[0][1] = weight;
```

### 2.3 Edge List

```cpp
vector<tuple<int,int,int>> edges;  // {u, v, weight}
edges.push_back({0, 1, 5});
```

### 2.4 When to Use Which

| Representation | Space | Check Edge | Iterate Neighbors | Best For |
|---|---|---|---|---|
| **Adjacency List** | O(V + E) | O(degree) | O(degree) ✅ | Sparse graphs, most problems |
| **Adjacency Matrix** | O(V²) | O(1) ✅ | O(V) | Dense graphs, Floyd-Warshall |
| **Edge List** | O(E) | O(E) | O(E) | Kruskal's MST, edge processing |

```
├──────────────────────────────────────────────────────────────────┤

│                                                                  │

│  → Undirected / Directed / Weighted / Grid?                      │
│                                                                  │
│  Step 3: WHAT ARE YOU ASKED TO FIND?                             │
│                                                                  │
│  "Is there a path?"                 → BFS or DFS                 │
│  "Shortest path (unweighted)?"      → BFS ✅                    │
│  "Shortest path (weighted)?"        → Dijkstra / Bellman-Ford    │
│  "Connected components / islands?"  → BFS/DFS/Union-Find         │
│  "Cycle detection?"                 → DFS (coloring) or Topo     │
│  "Ordering with dependencies?"      → Topological Sort           │
│  "Minimum cost to connect all?"     → MST (Kruskal/Prim)         │
│  "Can we 2-color?"                  → Bipartite check (BFS/DFS)  │
│  "Shortest path all pairs?"         → Floyd-Warshall             │

```

### 3.2 BFS vs DFS — When to Use Which

| Use Case | BFS | DFS |
|---|---|---|
| Shortest path (unweighted) | ✅ Yes | ❌ No |
| Level-by-level exploration | ✅ Yes | ❌ No |
| Nearest something | ✅ Yes | ❌ No |
| Connected components | ✅ Yes | ✅ Yes |
| Cycle detection | ✅ Possible | ✅ Better |
| Topological sort | ✅ Kahn's | ✅ DFS-based |
| Path finding / exploration | ❌ More memory | ✅ Less memory |
| Backtracking | ❌ No | ✅ Yes |

## 4. BFS — Breadth-First Search ⭐⭐

### 4.1 How BFS Works

```
BFS explores nodes LEVEL BY LEVEL (closest first).
Uses a QUEUE (FIFO).

Start → Visit all neighbors → Visit their neighbors → ...

        1
       / \
      2   3
     / \   \
    4   5   6

BFS from 1: [1] → [2, 3] → [4, 5, 6]
Level 0: {1}
Level 1: {2, 3}
Level 2: {4, 5, 6}
```

### 4.2 BFS Template — Adjacency List

```cpp
void bfs(int start, vector<vector<int>>& adj, int n) {
    vector<bool> visited(n, false);
    queue<int> q;
    
    visited[start] = true;
    q.push(start);
    
    while (!q.empty()) {
        int node = q.front(); q.pop();
        
        // Process node here
        
        for (int neighbor : adj[node]) {
            if (!visited[neighbor]) {
                visited[neighbor] = true;
                q.push(neighbor);
            }
        }
    }
}
// Time: O(V + E), Space: O(V)
```

### 4.3 BFS Shortest Path (Unweighted) ⭐

```cpp
vector<int> shortestPath(int start, vector<vector<int>>& adj, int n) {
    vector<int> dist(n, -1);
    queue<int> q;
    
    dist[start] = 0;
    q.push(start);
    
    while (!q.empty()) {
        int node = q.front(); q.pop();
        
        for (int neighbor : adj[node]) {
            if (dist[neighbor] == -1) {
                dist[neighbor] = dist[node] + 1;
                q.push(neighbor);
            }
        }
    }
    return dist;  // dist[i] = shortest distance from start to i
}
// BFS guarantees shortest path in unweighted graphs!
```

### 4.4 BFS on Grid ⭐

```cpp
int dx[] = {0, 0, 1, -1};  // right, left, down, up
int dy[] = {1, -1, 0, 0};

void bfsGrid(vector<vector<int>>& grid, int sr, int sc) {
    int m = grid.size(), n = grid[0].size();
    vector<vector<bool>> visited(m, vector<bool>(n, false));
    queue<pair<int,int>> q;
    
    visited[sr][sc] = true;
    q.push({sr, sc});
    
    while (!q.empty()) {
        auto [r, c] = q.front(); q.pop();
        
        for (int d = 0; d < 4; d++) {
            int nr = r + dx[d], nc = c + dy[d];
            if (nr >= 0 && nr < m && nc >= 0 && nc < n &&
                !visited[nr][nc] && grid[nr][nc] == 1) {
                visited[nr][nc] = true;
                q.push({nr, nc});
            }
        }
    }
}
```

## 5. DFS — Depth-First Search ⭐⭐

### 5.1 How DFS Works

```
DFS explores as DEEP as possible before backtracking.
Uses RECURSION or a STACK (LIFO).

        1
       / \
      2   3
     / \   \
    4   5   6

DFS from 1: 1 → 2 → 4 → (backtrack) → 5 → (backtrack) → 3 → 6
```

### 5.2 DFS Template — Recursive

```cpp
void dfs(int node, vector<vector<int>>& adj, vector<bool>& visited) {
    visited[node] = true;
    
    // Process node here
    
    for (int neighbor : adj[node]) {
        if (!visited[neighbor]) {
            dfs(neighbor, adj, visited);
        }
    }
}

// Call for all components:
void dfsAll(vector<vector<int>>& adj, int n) {
    vector<bool> visited(n, false);
    for (int i = 0; i < n; i++) {
        if (!visited[i]) dfs(i, adj, visited);
    }
}
// Time: O(V + E), Space: O(V) for visited + O(h) for stack
```

### 5.3 DFS Template — Iterative (Stack)

```cpp
void dfsIterative(int start, vector<vector<int>>& adj, int n) {
    vector<bool> visited(n, false);
    stack<int> st;
    
    st.push(start);
    
    while (!st.empty()) {
        int node = st.top(); st.pop();
        
        if (visited[node]) continue;
        visited[node] = true;
        
        // Process node here
        
        for (int neighbor : adj[node]) {
            if (!visited[neighbor]) {
                st.push(neighbor);
            }
        }
    }
}
```

### 5.4 DFS on Grid

```cpp
void dfsGrid(vector<vector<int>>& grid, int r, int c) {
    int m = grid.size(), n = grid[0].size();
    if (r < 0 || r >= m || c < 0 || c >= n || grid[r][c] == 0) return;
    
    grid[r][c] = 0;  // mark visited (or use separate visited array)
    
    dfsGrid(grid, r + 1, c);
    dfsGrid(grid, r - 1, c);
    dfsGrid(grid, r, c + 1);
    dfsGrid(grid, r, c - 1);
}
```

### 📝 Part 1 Summary — BFS vs DFS

| Feature | BFS | DFS |
|---|---|---|
| Data structure | Queue | Stack / Recursion |
| Order | Level by level | Deep then backtrack |
| Shortest path (unweighted) | ✅ Yes | ❌ No |
| Space | O(width) | O(height/depth) |
| Better for | Shortest path, nearest, levels | Cycle detection, paths, backtracking |

## 6. Pattern 1: Connected Components & Flood Fill ⭐

### 6.1 Number of Connected Components

```cpp
int countComponents(int n, vector<vector<int>>& adj) {
    vector<bool> visited(n, false);
    int count = 0;
    
    for (int i = 0; i < n; i++) {
        if (!visited[i]) {
            dfs(i, adj, visited);
            count++;
        }
    }
    return count;
}
// Each DFS/BFS call explores one full component.
```

### 6.2 Number of Islands (LC 200) ⭐⭐

```cpp
int numIslands(vector<vector<char>>& grid) {
    int m = grid.size(), n = grid[0].size(), count = 0;
    
    for (int i = 0; i < m; i++) {
        for (int j = 0; j < n; j++) {
            if (grid[i][j] == '1') {
                count++;
                dfs(grid, i, j);  // sink the island
            }
        }
    }
    return count;
}

void dfs(vector<vector<char>>& grid, int r, int c) {
    if (r < 0 || r >= grid.size() || c < 0 || c >= grid[0].size() ||
        grid[r][c] == '0') return;
    
    grid[r][c] = '0';  // mark visited
    dfs(grid, r+1, c);
    dfs(grid, r-1, c);
    dfs(grid, r, c+1);
    dfs(grid, r, c-1);
}
// Time: O(m × n), Space: O(m × n) worst case
```

### 6.3 Flood Fill (LC 733)

```cpp
vector<vector<int>> floodFill(vector<vector<int>>& image, int sr, int sc, int color) {
    int original = image[sr][sc];
    if (original == color) return image;
    dfs(image, sr, sc, original, color);
    return image;
}

void dfs(vector<vector<int>>& img, int r, int c, int original, int color) {
    if (r < 0 || r >= img.size() || c < 0 || c >= img[0].size() ||
        img[r][c] != original) return;
    
    img[r][c] = color;
    dfs(img, r+1, c, original, color);
    dfs(img, r-1, c, original, color);
    dfs(img, r, c+1, original, color);
    dfs(img, r, c-1, original, color);
}
```

### 6.4 Max Area of Island (LC 695)

```cpp
int maxAreaOfIsland(vector<vector<int>>& grid) {
    int maxArea = 0;
    for (int i = 0; i < grid.size(); i++)
        for (int j = 0; j < grid[0].size(); j++)
            if (grid[i][j] == 1)
                maxArea = max(maxArea, dfs(grid, i, j));
    return maxArea;
}

int dfs(vector<vector<int>>& grid, int r, int c) {
    if (r < 0 || r >= grid.size() || c < 0 || c >= grid[0].size() ||
        grid[r][c] == 0) return 0;
    grid[r][c] = 0;
    return 1 + dfs(grid, r+1, c) + dfs(grid, r-1, c) +
               dfs(grid, r, c+1) + dfs(grid, r, c-1);
}
```

## 7. Pattern 2: Cycle Detection ⭐

### 7.1 Cycle in Undirected Graph — DFS

```cpp
bool hasCycleUndirected(int n, vector<vector<int>>& adj) {
    vector<bool> visited(n, false);
    for (int i = 0; i < n; i++)
        if (!visited[i] && dfsCycle(i, -1, adj, visited))
            return true;
    return false;
}

bool dfsCycle(int node, int parent, vector<vector<int>>& adj, vector<bool>& visited) {
    visited[node] = true;
    for (int neighbor : adj[node]) {
        if (!visited[neighbor]) {
            if (dfsCycle(neighbor, node, adj, visited)) return true;
        } else if (neighbor != parent) {
            return true;  // visited neighbor that's not parent = CYCLE!
        }
    }
    return false;
}
// Key: if we reach a visited node that isn't our parent → cycle exists.
```

### 7.2 Cycle in Directed Graph — DFS (Three Colors) ⭐

```cpp
bool hasCycleDirected(int n, vector<vector<int>>& adj) {
    // 0 = white (unvisited), 1 = gray (in progress), 2 = black (done)
    vector<int> color(n, 0);
    
    for (int i = 0; i < n; i++)
        if (color[i] == 0 && dfsCycleDir(i, adj, color))
            return true;
    return false;
}

bool dfsCycleDir(int node, vector<vector<int>>& adj, vector<int>& color) {
    color[node] = 1;  // gray: processing
    
    for (int neighbor : adj[node]) {
        if (color[neighbor] == 1) return true;   // gray → CYCLE (back edge)!
        if (color[neighbor] == 0 && dfsCycleDir(neighbor, adj, color))
            return true;
    }
    
    color[node] = 2;  // black: done
    return false;
}
// Gray = in current DFS path. If we hit gray → cycle!
```

### 7.3 Cycle Detection — BFS (Kahn's Algorithm)

```cpp
bool hasCycleKahns(int n, vector<vector<int>>& adj) {
    vector<int> indegree(n, 0);
    for (int u = 0; u < n; u++)
        for (int v : adj[u]) indegree[v]++;
    
    queue<int> q;
    for (int i = 0; i < n; i++)
        if (indegree[i] == 0) q.push(i);
    
    int processed = 0;
    while (!q.empty()) {
        int node = q.front(); q.pop();
        processed++;
        for (int neighbor : adj[node]) {
            indegree[neighbor]--;
            if (indegree[neighbor] == 0) q.push(neighbor);
        }
    }
    return processed != n;  // if not all processed → cycle exists!
}
```

## 8. Pattern 3: Topological Sort (DAG) ⭐⭐

### 8.1 Core Intuition

> **Topological Sort:** Linear ordering of nodes such that for every directed edge u → v, u comes before v.
> Only possible for **DAGs** (Directed Acyclic Graphs).

```
Prerequisites: Calc → Linear Alg → ML
               Calc → Probability → ML
               Stats → Probability

Topological order: Calc, Stats, Linear Alg, Probability, ML
(many valid orderings possible)
```

### 8.2 Kahn's Algorithm — BFS (In-degree) ⭐

```cpp
vector<int> topoSort(int n, vector<vector<int>>& adj) {
    vector<int> indegree(n, 0);
    for (int u = 0; u < n; u++)
        for (int v : adj[u]) indegree[v]++;
    
    queue<int> q;
    for (int i = 0; i < n; i++)
        if (indegree[i] == 0) q.push(i);
    
    vector<int> order;
    while (!q.empty()) {
        int node = q.front(); q.pop();
        order.push_back(node);
        for (int neighbor : adj[node]) {
            indegree[neighbor]--;
            if (indegree[neighbor] == 0) q.push(neighbor);
        }
    }
    
    if (order.size() != n) return {};  // cycle exists!
    return order;
}
// Time: O(V + E), Space: O(V)
```

### 8.3 DFS-based Topological Sort

```cpp
vector<int> topoSortDFS(int n, vector<vector<int>>& adj) {
    vector<bool> visited(n, false);
    stack<int> st;
    
    for (int i = 0; i < n; i++)
        if (!visited[i]) dfsTopo(i, adj, visited, st);
    
    vector<int> order;
    while (!st.empty()) {
        order.push_back(st.top());
        st.pop();
    }
    return order;
}

void dfsTopo(int node, vector<vector<int>>& adj, vector<bool>& visited, stack<int>& st) {
    visited[node] = true;
    for (int neighbor : adj[node])
        if (!visited[neighbor]) dfsTopo(neighbor, adj, visited, st);
    st.push(node);  // push AFTER all children processed (postorder)
}
```

### 8.4 Course Schedule (LC 207) ⭐

```cpp
bool canFinish(int numCourses, vector<vector<int>>& prerequisites) {
    vector<vector<int>> adj(numCourses);
    vector<int> indegree(numCourses, 0);
    
    for (auto& p : prerequisites) {
        adj[p[1]].push_back(p[0]);  // p[1] → p[0]
        indegree[p[0]]++;
    }
    
    queue<int> q;
    for (int i = 0; i < numCourses; i++)
        if (indegree[i] == 0) q.push(i);
    
    int count = 0;
    while (!q.empty()) {
        int node = q.front(); q.pop();
        count++;
        for (int next : adj[node]) {
            indegree[next]--;
            if (indegree[next] == 0) q.push(next);
        }
    }
    return count == numCourses;
}
```

### 8.5 Course Schedule II — Return Order (LC 210)

```cpp
vector<int> findOrder(int numCourses, vector<vector<int>>& prerequisites) {
    vector<vector<int>> adj(numCourses);
    vector<int> indegree(numCourses, 0);
    
    for (auto& p : prerequisites) {
        adj[p[1]].push_back(p[0]);
        indegree[p[0]]++;
    }
    
    queue<int> q;
    for (int i = 0; i < numCourses; i++)
        if (indegree[i] == 0) q.push(i);
    
    vector<int> order;
    while (!q.empty()) {
        int node = q.front(); q.pop();
        order.push_back(node);
        for (int next : adj[node]) {
            indegree[next]--;
            if (indegree[next] == 0) q.push(next);
        }
    }
    return order.size() == numCourses ? order : vector<int>{};
}
```

## 9. Pattern 4: Bipartite Check

### 9.1 Core Intuition

> A graph is **bipartite** if you can color all nodes with 2 colors such that no two adjacent nodes share the same color.
> Equivalent to: the graph has NO odd-length cycles.

### 9.2 Bipartite Check — BFS (LC 785) ⭐

```cpp
bool isBipartite(vector<vector<int>>& graph) {
    int n = graph.size();
    vector<int> color(n, -1);  // -1 = uncolored
    
    for (int i = 0; i < n; i++) {
        if (color[i] != -1) continue;
        
        queue<int> q;
        q.push(i);
        color[i] = 0;
        
        while (!q.empty()) {
            int node = q.front(); q.pop();
            for (int neighbor : graph[node]) {
                if (color[neighbor] == -1) {
                    color[neighbor] = 1 - color[node];  // opposite color
                    q.push(neighbor);
                } else if (color[neighbor] == color[node]) {
                    return false;  // same color = NOT bipartite
                }
            }
        }
    }
    return true;
}
```

## 10. Pattern 5: Shortest Path ⭐⭐

### 10.1 Algorithm Comparison

| Algorithm | Graph Type | Time | Negative Weights? | Use Case |
|---|---|---|---|---|
| **BFS** | Unweighted | O(V + E) | N/A | Simple shortest path |
| **Dijkstra** | Weighted (non-negative) | O((V+E) log V) | ❌ No | GPS, routing |
| **Bellman-Ford** | Weighted (any) | O(V × E) | ✅ Yes | Detect negative cycles |
| **Floyd-Warshall** | All pairs | O(V³) | ✅ Yes | Small graphs, all pairs |
| **0-1 BFS** | Weights 0 or 1 only | O(V + E) | ❌ No | Grid with free/cost cells |

### 10.2 Dijkstra's Algorithm ⭐⭐

```cpp
vector<int> dijkstra(int start, vector<vector<pair<int,int>>>& adj, int n) {
    vector<int> dist(n, INT_MAX);
    priority_queue<pair<int,int>, vector<pair<int,int>>, greater<>> pq;  // min-heap
    
    dist[start] = 0;
    pq.push({0, start});  // {distance, node}
    
    while (!pq.empty()) {
        auto [d, u] = pq.top(); pq.pop();
        
        if (d > dist[u]) continue;  // stale entry (lazy deletion)
        
        for (auto [v, w] : adj[u]) {
            if (dist[u] + w < dist[v]) {
                dist[v] = dist[u] + w;
                pq.push({dist[v], v});
            }
        }
    }
    return dist;
}
// Time: O((V + E) log V), Space: O(V + E)
// ONLY works with non-negative weights!
```

### 10.3 Network Delay Time (LC 743) ⭐

```cpp
int networkDelayTime(vector<vector<int>>& times, int n, int k) {
```

### 10.4 Bellman-Ford Algorithm

```cpp
vector<int> bellmanFord(int start, int n, vector<tuple<int,int,int>>& edges) {
    vector<int> dist(n, INT_MAX);
    dist[start] = 0;
    
    // Relax all edges V-1 times
    for (int i = 0; i < n - 1; i++) {
        for (auto [u, v, w] : edges) {
            if (dist[u] != INT_MAX && dist[u] + w < dist[v]) {
                dist[v] = dist[u] + w;
            }
        }
    }
    
    // Check for negative cycles (Vth iteration)
    for (auto [u, v, w] : edges) {
        if (dist[u] != INT_MAX && dist[u] + w < dist[v]) {
            return {};  // negative cycle detected!
        }
    }
    return dist;
}
// Time: O(V × E), Space: O(V)
// Handles negative weights. Detects negative cycles.
```

### 10.5 Cheapest Flights Within K Stops (LC 787)

```cpp
int findCheapestPrice(int n, vector<vector<int>>& flights, int src, int dst, int k) {
    vector<int> dist(n, INT_MAX);
    dist[src] = 0;
    
    for (int i = 0; i <= k; i++) {
        vector<int> temp = dist;  // use previous round's distances
        for (auto& f : flights) {
            int u = f[0], v = f[1], w = f[2];
            if (dist[u] != INT_MAX && dist[u] + w < temp[v]) {
                temp[v] = dist[u] + w;
            }
        }
        dist = temp;
    }
    return dist[dst] == INT_MAX ? -1 : dist[dst];
}
// Modified Bellman-Ford with K+1 iterations (at most K stops = K+1 edges).
```

### 10.6 Floyd-Warshall — All-Pairs Shortest Path

```cpp
void floydWarshall(vector<vector<int>>& dist) {
    int n = dist.size();
    // dist[i][j] = direct edge weight, or INF if no edge, dist[i][i] = 0
    
    for (int k = 0; k < n; k++)           // intermediate node
        for (int i = 0; i < n; i++)       // source
            for (int j = 0; j < n; j++)   // destination
                if (dist[i][k] != INT_MAX && dist[k][j] != INT_MAX)
                    dist[i][j] = min(dist[i][j], dist[i][k] + dist[k][j]);
}
// Time: O(V³), Space: O(V²)
// Simple triple loop. Good for small dense graphs.
```

## 11. Pattern 6: Minimum Spanning Tree ⭐

### 11.1 Core Intuition

> **MST** connects ALL vertices with minimum total edge weight, forming a tree (V-1 edges, no cycles).

### 11.2 Kruskal's Algorithm (Edge-based + Union-Find) ⭐

```cpp
struct Edge {
    int u, v, weight;
    bool operator<(const Edge& other) const { return weight < other.weight; }
};

int kruskal(int n, vector<Edge>& edges) {
    sort(edges.begin(), edges.end());
    
    vector<int> parent(n), rank_(n, 0);
    iota(parent.begin(), parent.end(), 0);
    
    auto find = [&](auto& self, int x) -> int {
        return parent[x] == x ? x : parent[x] = self(self, parent[x]);
    };
    
    auto unite = [&](int x, int y) -> bool {
        int px = find(find, x), py = find(find, y);
        if (px == py) return false;
        if (rank_[px] < rank_[py]) swap(px, py);
        parent[py] = px;
        if (rank_[px] == rank_[py]) rank_[px]++;
        return true;
    };
    
    int mstWeight = 0, edgesUsed = 0;
    for (auto& e : edges) {
        if (unite(e.u, e.v)) {
            mstWeight += e.weight;
            edgesUsed++;
            if (edgesUsed == n - 1) break;
        }
    }
    return edgesUsed == n - 1 ? mstWeight : -1;  // -1 if not connected
}
// Time: O(E log E), Space: O(V)
```

### 11.3 Prim's Algorithm (Vertex-based + Min-Heap)

```cpp
int prim(int n, vector<vector<pair<int,int>>>& adj) {
    vector<bool> inMST(n, false);
    priority_queue<pair<int,int>, vector<pair<int,int>>, greater<>> pq;
    pq.push({0, 0});  // {weight, node}
    
    int mstWeight = 0, edgesUsed = 0;
    while (!pq.empty() && edgesUsed < n) {
        auto [w, u] = pq.top(); pq.pop();
        if (inMST[u]) continue;
        
        inMST[u] = true;
        mstWeight += w;
        edgesUsed++;
        
        for (auto [v, weight] : adj[u]) {
            if (!inMST[v]) pq.push({weight, v});
        }
    }
    return edgesUsed == n ? mstWeight : -1;
}
// Time: O((V + E) log V), Space: O(V + E)
```

### 11.4 Kruskal vs Prim

| | Kruskal | Prim |
|---|---|---|
| Approach | Edge-based (sort + union-find) | Vertex-based (min-heap) |
| Best for | Sparse graphs (E ≈ V) | Dense graphs (E ≈ V²) |
| Time | O(E log E) | O((V+E) log V) |
| Needs | Edge list + Union-Find | Adjacency list + Min-Heap |

### 11.5 Min Cost to Connect All Points (LC 1584)

```cpp
int minCostConnectPoints(vector<vector<int>>& points) {
    int n = points.size();

    for (int i = 0; i < n; i++)
        for (int j = i + 1; j < n; j++) {
            int dist = abs(points[i][0] - points[j][0]) + abs(points[i][1] - points[j][1]);
            edges.push_back({i, j, dist});
        }
```

## 12. Pattern 7: Union-Find (Disjoint Set Union) ⭐⭐

### 12.1 Core Intuition

> Union-Find answers: "Are these two elements in the same group?" 
> and "Merge these two groups."
> Both operations in nearly O(1) amortized (with path compression + union by rank).

### 12.2 Union-Find Implementation

```cpp
class UnionFind {
    vector<int> parent, rank_;
    int components;
    
public:
    UnionFind(int n) : parent(n), rank_(n, 0), components(n) {
        iota(parent.begin(), parent.end(), 0);
    }
    
    int find(int x) {
        if (parent[x] != x) parent[x] = find(parent[x]);  // path compression
        return parent[x];
    }
    
    bool unite(int x, int y) {
        int px = find(x), py = find(y);
        if (px == py) return false;  // already in same set
        
        // union by rank
        if (rank_[px] < rank_[py]) swap(px, py);
        parent[py] = px;
        if (rank_[px] == rank_[py]) rank_[px]++;
        
        components--;
        return true;
    }
    
    bool connected(int x, int y) { return find(x) == find(y); }
    int getComponents() { return components; }
};
// find: O(α(n)) ≈ O(1) amortized
// unite: O(α(n)) ≈ O(1) amortized
```

### 12.3 When to Use Union-Find vs BFS/DFS

| Scenario | Union-Find | BFS/DFS |
|---|---|---|
| Dynamic connectivity (edges added one by one) | ✅ Best | ❌ Rebuild every time |
| Static connectivity | ✅ Good | ✅ Good |
| Count components | ✅ Track with counter | ✅ Count DFS starts |
| Cycle detection (undirected) | ✅ If unite returns false → cycle | ✅ DFS with parent |
| MST (Kruskal's) | ✅ Required | ❌ |
| Shortest path | ❌ | ✅ BFS/Dijkstra |

### 12.6 Accounts Merge (LC 721) ⭐

```cpp
vector<vector<string>> accountsMerge(vector<vector<string>>& accounts) {
    int n = accounts.size();
    UnionFind uf(n);
    unordered_map<string, int> emailToId;
    
    for (int i = 0; i < n; i++) {
        for (int j = 1; j < accounts[i].size(); j++) {
            string email = accounts[i][j];
            if (emailToId.count(email)) {
                uf.unite(i, emailToId[email]);
            } else {
                emailToId[email] = i;
            }
        }
    }
    
    unordered_map<int, set<string>> groups;
    for (auto& [email, id] : emailToId) {
        groups[uf.find(id)].insert(email);
    }
    
    vector<vector<string>> result;
    for (auto& [id, emails] : groups) {
        vector<string> merged = {accounts[id][0]};
        merged.insert(merged.end(), emails.begin(), emails.end());
        result.push_back(merged);
    }
    return result;
}
```

### 13.2 Surrounded Regions (LC 130) ⭐

```cpp
void solve(vector<vector<char>>& board) {
    int m = board.size(), n = board[0].size();
    
    // Step 1: Mark all 'O' connected to border (can't be captured)
    for (int i = 0; i < m; i++) {
        if (board[i][0] == 'O') dfs(board, i, 0);
        if (board[i][n-1] == 'O') dfs(board, i, n-1);
    }
    for (int j = 0; j < n; j++) {
        if (board[0][j] == 'O') dfs(board, 0, j);
        if (board[m-1][j] == 'O') dfs(board, m-1, j);
    }
    
    // Step 2: Flip remaining 'O' to 'X', restore '#' to 'O'
    for (int i = 0; i < m; i++)
        for (int j = 0; j < n; j++) {
            if (board[i][j] == 'O') board[i][j] = 'X';
            if (board[i][j] == '#') board[i][j] = 'O';
        }
}

void dfs(vector<vector<char>>& board, int r, int c) {
    if (r < 0 || r >= board.size() || c < 0 || c >= board[0].size() ||
        board[r][c] != 'O') return;
    board[r][c] = '#';  // safe
    dfs(board, r+1, c); dfs(board, r-1, c);
    dfs(board, r, c+1); dfs(board, r, c-1);
}
// Key insight: start from borders, not from interior!
```

### 13.3 Pacific Atlantic Water Flow (LC 417) ⭐

```cpp
vector<vector<int>> pacificAtlantic(vector<vector<int>>& heights) {
    int m = heights.size(), n = heights[0].size();
    vector<vector<bool>> pacific(m, vector<bool>(n, false));
    vector<vector<bool>> atlantic(m, vector<bool>(n, false));
    
    for (int i = 0; i < m; i++) {
        dfs(heights, pacific, i, 0);
        dfs(heights, atlantic, i, n-1);
    }
    for (int j = 0; j < n; j++) {
        dfs(heights, pacific, 0, j);
        dfs(heights, atlantic, m-1, j);

    for (int i = 0; i < m; i++)
        for (int j = 0; j < n; j++)
            if (pacific[i][j] && atlantic[i][j])
                result.push_back({i, j});
    return result;
}

void dfs(vector<vector<int>>& h, vector<vector<bool>>& ocean, int r, int c) {
    ocean[r][c] = true;
    int dx[] = {0,0,1,-1}, dy[] = {1,-1,0,0};
    for (int d = 0; d < 4; d++) {
        int nr = r+dx[d], nc = c+dy[d];
        if (nr >= 0 && nr < h.size() && nc >= 0 && nc < h[0].size() &&
            !ocean[nr][nc] && h[nr][nc] >= h[r][c])
            dfs(h, ocean, nr, nc);
    }
}
// Reverse DFS from each ocean inward. Both reachable → answer.
```

## 14. Pattern 9: Multi-Source BFS & 0-1 BFS

### 14.1 Multi-Source BFS — Rotten Oranges (LC 994) ⭐

```cpp
int orangesRotting(vector<vector<int>>& grid) {
    int m = grid.size(), n = grid[0].size(), fresh = 0;
    queue<pair<int,int>> q;
    
    // Push ALL rotten oranges as starting points
    for (int i = 0; i < m; i++)
        for (int j = 0; j < n; j++) {
            if (grid[i][j] == 2) q.push({i, j});
            if (grid[i][j] == 1) fresh++;
        }
    
    if (fresh == 0) return 0;
    
    int dx[] = {0,0,1,-1}, dy[] = {1,-1,0,0};
    int time = 0;
    while (!q.empty()) {
        int size = q.size();
        for (int i = 0; i < size; i++) {
            auto [r, c] = q.front(); q.pop();
            for (int d = 0; d < 4; d++) {
                int nr = r + dx[d], nc = c + dy[d];
                if (nr >= 0 && nr < m && nc >= 0 && nc < n && grid[nr][nc] == 1) {
                    grid[nr][nc] = 2;
                    q.push({nr, nc});
                    fresh--;
                }
            }
        }
        time++;
    }
    return fresh == 0 ? time - 1 : -1;
}
// Multi-source BFS: all sources start at time 0.
// Simulates simultaneous spreading.
```

### 14.3 0-1 BFS — Shortest Path with 0/1 Weights

```cpp
vector<int> bfs01(int start, vector<vector<pair<int,int>>>& adj, int n) {
    vector<int> dist(n, INT_MAX);
    deque<int> dq;
    
    dist[start] = 0;
    dq.push_back(start);
    
    while (!dq.empty()) {
        int u = dq.front(); dq.pop_front();
        
        for (auto [v, w] : adj[u]) {
            if (dist[u] + w < dist[v]) {
                dist[v] = dist[u] + w;
                if (w == 0) dq.push_front(v);  // weight 0: front of deque
                else        dq.push_back(v);   // weight 1: back of deque
            }
        }
    }
    return dist;
}
// Time: O(V + E), Space: O(V)
// Use DEQUE instead of priority queue for O(V+E) instead of O((V+E)logV).
```

## 15. Pattern 10: Backtracking on Graphs

### 15.1 Word Search (LC 79) ⭐

```cpp
bool exist(vector<vector<char>>& board, string word) {
    int m = board.size(), n = board[0].size();
    for (int i = 0; i < m; i++)
        for (int j = 0; j < n; j++)
            if (dfs(board, word, i, j, 0)) return true;
    return false;
}

bool dfs(vector<vector<char>>& board, string& word, int r, int c, int idx) {
    if (idx == word.size()) return true;
    if (r < 0 || r >= board.size() || c < 0 || c >= board[0].size() ||
        board[r][c] != word[idx]) return false;
    
    char temp = board[r][c];
    board[r][c] = '#';  // mark visiting
    
    bool found = dfs(board, word, r+1, c, idx+1) ||
                 dfs(board, word, r-1, c, idx+1) ||
                 dfs(board, word, r, c+1, idx+1) ||
                 dfs(board, word, r, c-1, idx+1);
    
    board[r][c] = temp;  // backtrack (restore)
    return found;
}
```

## 17. Common Mistakes & Debugging

### 17.1 Top 12 Graph Mistakes

| # | Mistake | Fix |
|---|---|---|
| 1 | **Forgetting visited array** → infinite loop | Always mark visited BEFORE pushing to queue/recursing |
| 2 | **BFS: marking visited when popping** | Mark visited when PUSHING, not when popping (avoids duplicates) |
| 3 | **DFS on grid: not checking bounds** | Always check `r >= 0 && r < m && c >= 0 && c < n` first |
| 4 | **Directed cycle: using undirected logic** | Use three colors (white/gray/black) for directed cycle detection |
| 5 | **Topo sort: wrong edge direction** | Prerequisites: `[a, b]` means b → a (b before a). Build adj correctly! |
| 6 | **Dijkstra with negative weights** | Use Bellman-Ford for negative weights. Dijkstra gives wrong answers! |
| 7 | **Union-Find: no path compression** | Always use path compression + union by rank for O(α(n)) |
| 8 | **Grid: forgetting diagonal directions** | 4-directional vs 8-directional. Read problem carefully! |
| 9 | **Multi-source BFS: pushing sources one at a time** | Push ALL sources into queue BEFORE starting BFS |
| 10 | **Not handling disconnected graphs** | Loop through ALL nodes to start BFS/DFS (not just node 0) |
| 11 | **Floyd-Warshall: wrong loop order** | `k` (intermediate) must be the OUTERMOST loop |
| 12 | **Creating adjacency list with wrong size** | `vector<vector<int>> adj(n)` — make sure `n` matches node count |

### 17.2 Debugging Checklist

```
□ Is my graph directed or undirected? Am I adding edges correctly?
□ Am I marking visited at the right time (push vs pop)?
□ Are my indices 0-based or 1-based? Does adj size match?
□ For BFS shortest path: am I using BFS (not DFS)?
□ For weighted shortest path: am I using Dijkstra (not BFS)?
□ For grids: bounds checking before accessing grid[nr][nc]?
□ For topo sort: is my edge direction correct (prereq → course)?
```

## 18. Interview FAQs

#### Q1: BFS vs DFS — how to decide?

**A:** **BFS** for shortest path (unweighted), level-by-level, nearest something. **DFS** for cycle detection, path existence, topological sort, backtracking. When in doubt, both work for connectivity — BFS is more natural for shortest path, DFS for exploration.

#### Q2: When do I use Union-Find over BFS/DFS?

**A:** Union-Find is best for **dynamic connectivity** — when edges arrive one at a time and you need to answer "Are these connected?" after each addition. Also required for **Kruskal's MST**. For static problems, BFS/DFS and Union-Find are interchangeable.

#### Q3: Dijkstra vs Bellman-Ford vs Floyd-Warshall?

**A:** **Dijkstra:** Single source, non-negative weights, O((V+E) log V). **Bellman-Ford:** Single source, handles negative weights, detects negative cycles, O(VE). **Floyd-Warshall:** All pairs, O(V³), use only for small graphs where you need ALL distances.

#### Q4: How do I detect a cycle in a directed vs undirected graph?

**A:** **Undirected:** DFS with parent tracking — if we visit a node that's already visited AND isn't our parent → cycle. **Directed:** Three-color DFS (white/gray/black) — if we hit a gray node → back edge → cycle. Or use Kahn's (if topo sort doesn't include all nodes → cycle).

#### Q5: How do I know it's a topological sort problem?

**A:** Keywords: "prerequisites," "dependencies," "ordering," "schedule," "build order." Anything where some tasks must come before others = DAG = topological sort.

#### Q6: Adjacency list vs adjacency matrix?

**A:** **List** for sparse graphs (E << V²) — saves memory, faster neighbor iteration. **Matrix** for dense graphs, Floyd-Warshall, or O(1) edge lookup. In interviews, almost always use adjacency list.

#### Q7: Why is BFS guaranteed to find the shortest path in unweighted graphs?

**A:** BFS explores nodes in order of distance from source. Level 0 (distance 0) first, then level 1 (distance 1), etc. The first time we reach a node is guaranteed to be the shortest path.

#### Q8: What's the difference between a tree and a graph?

**A:** A **tree** is a connected acyclic undirected graph with exactly V-1 edges. A **graph** can have cycles, disconnected components, and any number of edges. Every tree is a graph, but not every graph is a tree.

## 19. Real-World Scenarios

### 🔍 Scenario 1: Google Maps — Shortest Route

**Problem:** _"Find the fastest route from A to B."_

**Answer — DSA:** Road network = weighted directed graph. Intersections = nodes, roads = edges, travel time = weights. **Dijkstra's algorithm** (or A* with heuristics) finds the shortest path. Traffic updates = dynamic edge weights.

Used in: Google Maps, Waze, Uber, ride-sharing routing.

---

### 🔍 Scenario 2: Social Networks — Friend Recommendations

**Problem:** _"Recommend friends (people you may know)."_

**Answer — DSA:** Social network = undirected graph. Users = nodes, friendships = edges. **BFS from user** to find 2-hop connections (friends of friends). Connected components analysis reveals communities.

Used in: Facebook, LinkedIn, Instagram, Twitter.

---

### 🔍 Scenario 3: Package Managers — Dependency Resolution

**Problem:** _"Install packages in the right order."_

**Answer — DSA:** Package dependencies = DAG. Packages = nodes, "depends on" = directed edges. **Topological sort** gives a valid installation order. Cycle detection catches circular dependencies.

Used in: npm, pip, apt-get, Maven, Gradle.

---

### 🔍 Scenario 4: Internet — Web Crawling

**Problem:** _"Index all pages on the web."_

**Answer — DSA:** Web = directed graph. Pages = nodes, hyperlinks = edges. **BFS** from seed URLs crawls level by level (closer pages first). Visited set prevents re-crawling. PageRank uses graph structure.

Used in: Google Search, Bing, web scrapers, SEO tools.

---

### 🔍 Scenario 5: Airlines — Flight Networks

**Problem:** _"Find the cheapest flight with at most K stops."_

**Answer — DSA:** Airport network = weighted directed graph. **Modified Bellman-Ford** with K+1 relaxation rounds (LC 787). Or BFS/DFS with pruning. Dijkstra with state = (city, stops_remaining).

Used in: Google Flights, Skyscanner, Kayak.

---

### 🔍 Scenario 6: Network Infrastructure — MST

**Problem:** _"Connect all offices with minimum cable cost."_

**Answer — DSA:** Offices = nodes, possible connections = weighted edges. **Kruskal's or Prim's MST** gives minimum total cable length. Union-Find tracks which offices are already connected.

Used in: Telecom networks, power grids, fiber optic planning.

---

### 🔍 Scenario 7: Operating Systems — Deadlock Detection

**Problem:** _"Detect if processes are deadlocked."_

**Answer — DSA:** Resource allocation graph = directed graph. Processes = nodes, "waiting for" = edges. **Cycle detection** in directed graph → if cycle exists → deadlock! OS uses this to decide which process to terminate.

Used in: Database lock managers, OS kernels, distributed systems.

---

### 🔍 Scenario 8: Epidemiology — Disease Spread

**Problem:** _"Model how a virus spreads through a population."_

**Answer — DSA:** Contact network = graph. People = nodes, contacts = edges. **Multi-source BFS** from infected people models simultaneous spread. "Rotten oranges" = disease spreading (LC 994).

Used in: COVID-19 modeling, contact tracing, vaccination strategies.

---

### 🔍 Scenario 9: Git — Commit Graph

**Problem:** _"Find the merge base of two branches."_

**Answer — DSA:** Git commit history = DAG. Commits = nodes, parent pointers = edges. Finding merge base = **LCA** in a DAG. Topological sort determines build order for CI/CD.

Used in: Git (merge, rebase, cherry-pick), GitHub, GitLab.

---

### 🔍 Scenario 10: Recommendation Engines — Bipartite Graphs

**Problem:** _"Match users to products they'd like."_

**Answer — DSA:** Users and products form a **bipartite graph**. Edges = interactions (views, purchases, ratings). **Maximum bipartite matching** finds optimal user-product pairings. Graph-based collaborative filtering uses neighbor similarity.

Used in: Netflix, Amazon, Spotify, YouTube recommendations.
