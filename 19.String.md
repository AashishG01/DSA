## 1. Strings in C++ — Fundamentals

### 1.1 C-style Strings vs `std::string`

```cpp
// C-style string — char array, null terminated
char s1[] = "hello";     // ['h','e','l','l','o','\0']
char s2[10] = "hello";   // remaining filled with '\0'

// std::string — modern C++, preferred!
string s3 = "hello";
string s4("hello");
string s5(5, 'a');       // "aaaaa"
```

| Feature | `char[]` | `string` |
|---|---|---|
| Size | Fixed at compile time | Dynamic (auto-resize) |
| Null terminator | Required `\0` | Handled internally |
| Bounds checking | None (dangerous) | `.at()` throws, `[]` doesn't |
| Operations | `strcmp`, `strcpy`, `strlen` | `==`, `=`, `.size()` |
| Memory | Stack | Heap (usually) |
| Use in interviews | Rarely | **Always** ⭐ |

### 1.2 String Basics

```cpp
string s = "hello world";

// Access
s[0];                // 'h' — no bounds check
s.at(0);             // 'h' — throws if out of range
s.front();           // 'h'
s.back();            // 'd'

// Size
s.size();            // 11 (same as .length())
s.length();          // 11
s.empty();           // false

// Modification
s.push_back('!');    // "hello world!"
s.pop_back();        // "hello world"
s += " foo";         // "hello world foo"
s.append("bar");     // "hello world foobar"
s.insert(5, ",");    // "hello, world foobar"
s.erase(5, 1);       // "hello world foobar"
s.clear();           // ""
```

### 1.3 ASCII Values — Essential Table

```
'0'-'9': 48-57     digits
'A'-'Z': 65-90     uppercase
'a'-'z': 97-122    lowercase

Key relationships:
  'a' - 'A' = 32      (lowercase to uppercase difference)
  c - '0' = digit      (char to int conversion)
  c + '0' = char       (int to char conversion)
  tolower(c)            (any char to lowercase)
  toupper(c)            (any char to uppercase)
  isalpha(c)            (is letter?)
  isdigit(c)            (is digit?)
  isalnum(c)            (is letter or digit?)
```

### 2.1 Search Operations

```cpp
string s = "hello world hello";

s.find("world");          // 6 (first occurrence position)
s.find("xyz");            // string::npos (not found)
s.rfind("hello");         // 12 (last occurrence)
s.find_first_of("aeiou"); // 1 (first vowel position — 'e')

if (s.find("world") != string::npos) { /* found */ }
```

```cpp

a < b;   // true (lexicographic)
```

## 3. 🧠 How to Identify String Problems — Intuition

### 3.1 The Decision Framework

```
┌──────────────────────────────────────────────────────────────────┐
│                   WHICH STRING PATTERN?                          │
├──────────────────────────────────────────────────────────────────┤
│                                                                  │
│  "Reverse", "palindrome check", "compare from both ends"         │
│  → TWO POINTERS                                                  │
│                                                                  │
│  "Longest substring with...", "smallest window containing..."    │
│  → SLIDING WINDOW                                                │
│                                                                  │
│  "Anagram", "permutation", "frequency", "unique chars"           │
│  → FREQUENCY MAP (int[26] or unordered_map)                     │
│                                                                  │
│  "Find pattern in text", "substring search"                     │
│  → KMP / RABIN-KARP                                              │
│                                                                  │
│  "Palindromic substrings", "longest palindrome"                  │
│  → EXPAND FROM CENTER / DP                                       │
│                                                                  │
│  "Build / decode / encode string"                                │
│  → STRING BUILDING (stack, StringBuilder, recursion)             │
│                                                                  │
│  "Prefix search", "autocomplete", "word dictionary"              │
│  → TRIE                                                          │
│                                                                  │
└──────────────────────────────────────────────────────────────────┘
```

### 3.2 Key Insight: `int[26]` vs `unordered_map`

```
Use int freq[26] when:
  → Only lowercase letters (a-z)
  → O(1) space (fixed 26 slots)
  → Faster than hashmap

Use unordered_map<char, int> when:
  → Mixed case, digits, special chars
  → Unknown character set
  → Unicode support needed
```

## 4. Pattern 1: Two Pointers on Strings ⭐

### 4.1 Valid Palindrome (LC 125) ⭐

```cpp
bool isPalindrome(string s) {
    int l = 0, r = s.size() - 1;
    while (l < r) {
        while (l < r && !isalnum(s[l])) l++;
        while (l < r && !isalnum(s[r])) r--;
        if (tolower(s[l]) != tolower(s[r])) return false;
        l++; r--;
    }
    return true;
}
// Skip non-alphanumeric, compare from both ends.
```

### 4.2 Reverse String (LC 344)

```cpp
void reverseString(vector<char>& s) {
    int l = 0, r = s.size() - 1;
    while (l < r) swap(s[l++], s[r--]);
}
```

### 4.3 Reverse Words in a String (LC 151) ⭐

```cpp
string reverseWords(string s) {
    // Step 1: Reverse entire string
    reverse(s.begin(), s.end());
    
    int n = s.size(), i = 0, j = 0;
    for (int k = 0; k < n; k++) {
        if (s[k] != ' ') {
            if (j > 0) s[j++] = ' ';  // add space between words
            int start = j;
            while (k < n && s[k] != ' ') s[j++] = s[k++];
            reverse(s.begin() + start, s.begin() + j);  // reverse each word
        }
    }
    s.resize(j);
    return s;
}
// Trick: reverse all, then reverse each word individually.
```

### 4.4 Valid Palindrome II — Remove at most one char (LC 680)

```cpp
bool validPalindrome(string s) {
    int l = 0, r = s.size() - 1;
    while (l < r) {
        if (s[l] != s[r])
            return isPalin(s, l + 1, r) || isPalin(s, l, r - 1);
        l++; r--;
    }
    return true;
}

bool isPalin(string& s, int l, int r) {
    while (l < r) {
        if (s[l++] != s[r--]) return false;
    }
    return true;
}
```

## 5. Pattern 2: Sliding Window on Strings ⭐⭐

### 5.1 Longest Substring Without Repeating Characters (LC 3) ⭐⭐

```cpp
int lengthOfLongestSubstring(string s) {
    int freq[128] = {};  // ASCII character set
    int maxLen = 0, left = 0;
    
    for (int right = 0; right < s.size(); right++) {
        freq[s[right]]++;
        while (freq[s[right]] > 1) {
            freq[s[left]]--;
            left++;
        }
        maxLen = max(maxLen, right - left + 1);
    }
    return maxLen;
}
// Expand right, shrink left when duplicate found.
```

### 5.2 Minimum Window Substring (LC 76) ⭐⭐

```cpp
string minWindow(string s, string t) {
    int need[128] = {}, have = 0, required = 0;
    for (char c : t) { if (need[c]++ == 0) required++; }
    
    int minLen = INT_MAX, minStart = 0;
    int left = 0;
    int window[128] = {};
    
    for (int right = 0; right < s.size(); right++) {
        char c = s[right];
        window[c]++;
        if (window[c] == need[c]) have++;
        
        while (have == required) {
            if (right - left + 1 < minLen) {
                minLen = right - left + 1;
                minStart = left;
            }
            char d = s[left];
            window[d]--;
            if (window[d] < need[d]) have--;
            left++;
        }
    }
    return minLen == INT_MAX ? "" : s.substr(minStart, minLen);
}
// Shrink window when all characters are satisfied.
```

### 5.3 Longest Repeating Character Replacement (LC 424) ⭐

```cpp
int characterReplacement(string s, int k) {
    int freq[26] = {}, maxFreq = 0, maxLen = 0, left = 0;
    
    for (int right = 0; right < s.size(); right++) {
        freq[s[right] - 'A']++;
        maxFreq = max(maxFreq, freq[s[right] - 'A']);
        
        // Window is invalid if (windowSize - maxFreq) > k
        while ((right - left + 1) - maxFreq > k) {
            freq[s[left] - 'A']--;
            left++;
        }
        maxLen = max(maxLen, right - left + 1);
    }
    return maxLen;
}
// Key: characters to replace = windowSize - maxFreqInWindow.
```

## 6. Pattern 3: Frequency Counting & HashMap ⭐

### 6.1 Valid Anagram (LC 242) ⭐

```cpp
bool isAnagram(string s, string t) {
    if (s.size() != t.size()) return false;
    int freq[26] = {};
    for (char c : s) freq[c - 'a']++;
    for (char c : t) freq[c - 'a']--;
    for (int i = 0; i < 26; i++)
        if (freq[i] != 0) return false;
    return true;
}
// Increment for s, decrement for t. All zero = anagram.
```

### 6.2 First Unique Character (LC 387)

```cpp
int firstUniqChar(string s) {
    int freq[26] = {};
    for (char c : s) freq[c - 'a']++;
    for (int i = 0; i < s.size(); i++)
        if (freq[s[i] - 'a'] == 1) return i;
    return -1;
}
```

### 6.3 Group Anagrams (LC 49) ⭐⭐

```cpp
vector<vector<string>> groupAnagrams(vector<string>& strs) {
    unordered_map<string, vector<string>> groups;
    for (string& s : strs) {
        string key = s;
        sort(key.begin(), key.end());
        groups[key].push_back(s);
    }
    
    vector<vector<string>> result;
    for (auto& [key, group] : groups)
        result.push_back(group);
    return result;
}
// Key = sorted string. All anagrams sort to the same key.

// O(n) key alternative using frequency:
string getKey(string& s) {
    int freq[26] = {};
    for (char c : s) freq[c - 'a']++;
    string key;
    for (int i = 0; i < 26; i++)
        key += string(1, 'a' + i) + to_string(freq[i]);
    return key;
}
```

### 6.4 Ransom Note (LC 383)

```cpp
bool canConstruct(string ransomNote, string magazine) {
    int freq[26] = {};
    for (char c : magazine) freq[c - 'a']++;
    for (char c : ransomNote) {
        if (--freq[c - 'a'] < 0) return false;
    }
    return true;
}
```

### 6.5 Isomorphic Strings (LC 205) ⭐

```cpp
bool isIsomorphic(string s, string t) {
    int mapS[128] = {}, mapT[128] = {};
    for (int i = 0; i < s.size(); i++) {
        if (mapS[s[i]] != mapT[t[i]]) return false;
        mapS[s[i]] = i + 1;  // +1 to differentiate from default 0
        mapT[t[i]] = i + 1;
    }
    return true;
}
// Both characters must map to the same "last seen" index.
```

## 7. Pattern 4: String Matching — KMP, Rabin-Karp ⭐

### 7.1 Why Not Brute Force?

```
Text:    "AAAAAAAAAB"
Pattern: "AAAAB"

Brute force: Try every position → O(n × m) = O(50) for this case.
KMP:         Skip known matches → O(n + m) = O(15).
```

### 7.2 KMP Algorithm (Knuth-Morris-Pratt) ⭐⭐

```cpp
// Step 1: Build the LPS (Longest Proper Prefix which is also Suffix) array
vector<int> buildLPS(string& pattern) {
    int m = pattern.size();
    vector<int> lps(m, 0);
    int len = 0, i = 1;
    
    while (i < m) {
        if (pattern[i] == pattern[len]) {
            lps[i++] = ++len;
        } else {
            if (len > 0) len = lps[len - 1];  // don't increment i
            else lps[i++] = 0;
        }
    }
    return lps;
}

// Step 2: Search using LPS
int kmpSearch(string& text, string& pattern) {
    int n = text.size(), m = pattern.size();
    vector<int> lps = buildLPS(pattern);
    
    int i = 0, j = 0;  // i for text, j for pattern
    while (i < n) {
        if (text[i] == pattern[j]) {
            i++; j++;
            if (j == m) return i - m;  // found at position i - m
        } else {
            if (j > 0) j = lps[j - 1];  // skip to last matching prefix
            else i++;
        }
    }
    return -1;  // not found
}
// Time: O(n + m), Space: O(m) for LPS array
```

### 7.3 LPS Array — Intuition

```
Pattern: "ABCABD"
LPS:     [0,0,0,1,2,0]

Index 0: "A"       → no proper prefix = suffix → 0
Index 1: "AB"      → no match → 0
Index 2: "ABC"     → no match → 0
Index 3: "ABCA"    → "A" is prefix and suffix → 1
Index 4: "ABCAB"   → "AB" is prefix and suffix → 2
Index 5: "ABCABD"  → no match → 0

LPS tells us: on mismatch at index j, jump to lps[j-1] instead of 0.
This skips re-comparing characters we already know match.
```

```cpp
    return kmpSearch(haystack, needle);
}
```

### 7.5 Rabin-Karp Algorithm (Rolling Hash)

```cpp
int rabinKarp(string& text, string& pattern) {
    int n = text.size(), m = pattern.size();
    if (m > n) return -1;
    
    long long base = 31, mod = 1e9 + 7;
    long long patHash = 0, txtHash = 0, power = 1;
    
    // Compute initial hashes
    for (int i = 0; i < m; i++) {
        patHash = (patHash + (pattern[i] - 'a' + 1) * power) % mod;
        txtHash = (txtHash + (text[i] - 'a' + 1) * power) % mod;
        if (i < m - 1) power = power * base % mod;
    }
    
    for (int i = 0; i <= n - m; i++) {
        if (patHash == txtHash) {
            if (text.substr(i, m) == pattern) return i;  // confirm match
        }
        if (i < n - m) {
            txtHash = (txtHash - (text[i] - 'a' + 1)) % mod;
            txtHash = txtHash / base;  // simplified rolling
            txtHash = (txtHash + (text[i + m] - 'a' + 1) * power) % mod;
            txtHash = (txtHash + mod) % mod;
        }
    }
    return -1;
}
// Average: O(n + m), Worst: O(n × m) if many hash collisions.
// Good for multiple pattern search.
```

## 8. Pattern 5: Palindromes ⭐⭐

### 8.1 Expand from Center — Core Technique

```cpp
// Expand around center to find palindrome
int expandFromCenter(string& s, int left, int right) {
    while (left >= 0 && right < s.size() && s[left] == s[right]) {
        left--; right++;
    }
    return right - left - 1;  // length of palindrome
}
```

### 8.2 Longest Palindromic Substring (LC 5) ⭐⭐

```cpp
string longestPalindrome(string s) {
    int start = 0, maxLen = 1;
    
    for (int i = 0; i < s.size(); i++) {
        int len1 = expandFromCenter(s, i, i);      // odd length (center = i)
        int len2 = expandFromCenter(s, i, i + 1);  // even length (center = i, i+1)
        int len = max(len1, len2);
        
        if (len > maxLen) {
            maxLen = len;
            start = i - (len - 1) / 2;
        }
    }
    return s.substr(start, maxLen);
}
// Time: O(n²), Space: O(1)
// Try every center (n centers for odd, n-1 for even).
```

### 8.3 Palindromic Substrings — Count All (LC 647)

```cpp
int countSubstrings(string s) {
    int count = 0;
    for (int i = 0; i < s.size(); i++) {
        count += countPalin(s, i, i);      // odd
        count += countPalin(s, i, i + 1);  // even
    }
    return count;
}

int countPalin(string& s, int l, int r) {
    int count = 0;
    while (l >= 0 && r < s.size() && s[l] == s[r]) {
        count++; l--; r++;
    }
    return count;
}
```

### 8.4 Longest Palindromic Subsequence (LC 516) — DP

```cpp
int longestPalindromeSubseq(string s) {
    int n = s.size();
    vector<vector<int>> dp(n, vector<int>(n, 0));
    
    for (int i = n - 1; i >= 0; i--) {
        dp[i][i] = 1;
        for (int j = i + 1; j < n; j++) {
            if (s[i] == s[j])
                dp[i][j] = dp[i+1][j-1] + 2;
            else
                dp[i][j] = max(dp[i+1][j], dp[i][j-1]);
        }
    }
    return dp[0][n-1];
}
// Subsequence (not substring) — chars don't need to be contiguous.
// dp[i][j] = longest palindromic subsequence in s[i..j]
```

### 8.5 Palindrome Partitioning (LC 131) ⭐

```cpp
vector<vector<string>> partition(string s) {
    vector<vector<string>> result;
    vector<string> current;
    backtrack(s, 0, current, result);
    return result;
}

void backtrack(string& s, int start, vector<string>& curr,
               vector<vector<string>>& result) {
    if (start == s.size()) { result.push_back(curr); return; }
    
    for (int end = start; end < s.size(); end++) {
        if (isPalin(s, start, end)) {
            curr.push_back(s.substr(start, end - start + 1));
            backtrack(s, end + 1, curr, result);
            curr.pop_back();
        }
    }
}

bool isPalin(string& s, int l, int r) {
    while (l < r) if (s[l++] != s[r--]) return false;
    return true;
}
```

## 9. Pattern 6: Anagrams & Permutations

### 9.1 Find All Anagrams in a String (LC 438) ⭐

```cpp
vector<int> findAnagrams(string s, string p) {
    if (s.size() < p.size()) return {};
    
    int freq[26] = {}, window[26] = {};
    for (char c : p) freq[c - 'a']++;
    
    vector<int> result;
    for (int i = 0; i < s.size(); i++) {
        window[s[i] - 'a']++;
        if (i >= p.size()) window[s[i - p.size()] - 'a']--;
        
        if (memcmp(freq, window, sizeof(freq)) == 0)
            result.push_back(i - p.size() + 1);
    }
    return result;
}
// Fixed-size sliding window of size p.length().
// Compare frequency arrays at each step.
```

### 9.2 Permutation in String (LC 567) ⭐

```cpp
bool checkInclusion(string s1, string s2) {
    if (s1.size() > s2.size()) return false;
    
    int freq[26] = {}, window[26] = {};
    for (char c : s1) freq[c - 'a']++;
    
    for (int i = 0; i < s2.size(); i++) {
        window[s2[i] - 'a']++;
        if (i >= s1.size()) window[s2[i - s1.size()] - 'a']--;
        if (memcmp(freq, window, sizeof(freq)) == 0) return true;
    }
    return false;
}
```

## 10. Pattern 7: String Building & Manipulation ⭐

### 10.1 Using Stack for String Problems

```
Stack is perfect when:
→ Matching brackets/parentheses
→ Decoding nested structures
→ Removing adjacent duplicates
→ Building result character by character
```

### 10.2 Valid Parentheses (LC 20) ⭐

```cpp
bool isValid(string s) {
    stack<char> st;
    for (char c : s) {
        if (c == '(' || c == '{' || c == '[') st.push(c);
        else {
            if (st.empty()) return false;
            if (c == ')' && st.top() != '(') return false;
            if (c == '}' && st.top() != '{') return false;
            if (c == ']' && st.top() != '[') return false;
            st.pop();
        }
    }
    return st.empty();
}
```

### 10.3 Decode String (LC 394) ⭐⭐

```cpp
string decodeString(string s) {
    stack<int> countStack;
    stack<string> strStack;
    string current;
    int num = 0;
    
    for (char c : s) {
        if (isdigit(c)) {
            num = num * 10 + (c - '0');
        } else if (c == '[') {
            countStack.push(num);
            strStack.push(current);
            current = "";
            num = 0;
        } else if (c == ']') {
            int repeat = countStack.top(); countStack.pop();
            string temp = strStack.top(); strStack.pop();
            for (int i = 0; i < repeat; i++) temp += current;
            current = temp;
        } else {
            current += c;
        }
    }
    return current;
}
// "3[a2[c]]" → "accaccacc"
```

### 10.4 Remove All Adjacent Duplicates (LC 1047)

```cpp
string removeDuplicates(string s) {
    string result;
    for (char c : s) {
        if (!result.empty() && result.back() == c)
            result.pop_back();
        else
            result.push_back(c);
    }
    return result;
}
// Use string as a stack! push_back/pop_back.
```

### 10.5 String to Integer — atoi (LC 8) ⭐

```cpp
int myAtoi(string s) {
    int i = 0, n = s.size(), sign = 1;
    long result = 0;
    
    while (i < n && s[i] == ' ') i++;           // skip whitespace
    if (i < n && (s[i] == '+' || s[i] == '-'))
        sign = (s[i++] == '-') ? -1 : 1;        // handle sign
    
    while (i < n && isdigit(s[i])) {
        result = result * 10 + (s[i++] - '0');
        if (result * sign > INT_MAX) return INT_MAX;
        if (result * sign < INT_MIN) return INT_MIN;
    }
    return result * sign;
}
```

### 10.6 Longest Common Prefix (LC 14)

```cpp
string longestCommonPrefix(vector<string>& strs) {
    if (strs.empty()) return "";
    string prefix = strs[0];
    
    for (int i = 1; i < strs.size(); i++) {
        while (strs[i].find(prefix) != 0) {
            prefix = prefix.substr(0, prefix.size() - 1);
            if (prefix.empty()) return "";
        }
    }
    return prefix;
}
```

## 11. Pattern 8: Tries & Advanced String Structures

### 11.1 Trie (Prefix Tree) ⭐⭐

```cpp
struct TrieNode {
    TrieNode* children[26] = {};
    bool isEnd = false;
};

class Trie {
    TrieNode* root;
public:
    Trie() : root(new TrieNode()) {}
    
    void insert(string word) {
        TrieNode* node = root;
        for (char c : word) {
            int idx = c - 'a';
            if (!node->children[idx])
                node->children[idx] = new TrieNode();
            node = node->children[idx];
        }
        node->isEnd = true;
    }
    
    bool search(string word) {
        TrieNode* node = findNode(word);
        return node && node->isEnd;
    }
    
    bool startsWith(string prefix) {
        return findNode(prefix) != nullptr;
    }
    
private:
    TrieNode* findNode(string& s) {
        TrieNode* node = root;
        for (char c : s) {
            int idx = c - 'a';
            if (!node->children[idx]) return nullptr;
            node = node->children[idx];
        }
        return node;
    }
};
// Insert: O(L), Search: O(L), StartsWith: O(L)
// where L = word length
```

### 11.2 When to Use Tries

| Use Case | Why Trie? |
|---|---|
| Autocomplete / type-ahead | Fast prefix lookup |
| Spell checker | Find similar words |
| Word search in grid (LC 212) | Prune invalid paths early |
| Longest common prefix | Trie structure naturally gives this |
| Dictionary / word validation | O(L) lookup |

## 13. Common Mistakes & Debugging

### 13.1 Top 12 String Mistakes

| # | Mistake | Fix |
|---|---|---|
| 1 | **Off-by-one in substr** `s.substr(i, len)` not `s.substr(i, j)` | `substr(start, LENGTH)` — second param is length, not end index! |
| 2 | **Comparing with `==` on C-strings** | Use `strcmp()` for `char[]`. `==` compares pointers, not content! |
| 3 | **Not handling empty string** | Check `s.empty()` or `s.size() == 0` before accessing `s[0]` |
| 4 | **Modifying string while iterating** | Use index-based loop or build a new string |
| 5 | **String concatenation in loop = O(n²)** | Use `string.reserve()` or build with `ostringstream` |
| 6 | **Forgetting case sensitivity** | Use `tolower()` / `toupper()` when needed |
| 7 | **`string::npos` check missing** | Always check `if (pos != string::npos)` after `.find()` |
| 8 | **Overflow in atoi/parseInt** | Check against `INT_MAX/10` before multiplying |
| 9 | **Frequency array size 26 for mixed input** | Use `int[128]` for ASCII or `unordered_map` for Unicode |
| 10 | **Not handling leading/trailing spaces** | Trim with `while (s[i] == ' ') i++;` |
| 11 | **KMP: confusing LPS with suffix array** | LPS = longest PROPER prefix which is also suffix |
| 12 | **Palindrome: forgetting even-length** | Check BOTH `expand(i, i)` AND `expand(i, i+1)` |

### 13.2 String Performance Tips

```
□ string + string in a loop → O(n²). Use reserve() or push_back().
□ Pass strings by reference (string&) to avoid copying.
□ s.size() returns size_t (unsigned). Careful with s.size() - 1 when empty!
□ Use int[26] instead of unordered_map for lowercase-only problems.
□ substr() creates a new string — O(L) operation, not O(1).
□ Compare two frequency arrays with memcmp() for O(26) = O(1).
```

## 14. Interview FAQs

#### Q1: When should I use KMP vs brute force?

**A:** KMP gives O(n + m) vs brute force O(n × m). Use KMP when: (1) The pattern has repeated prefixes (allows skipping). (2) n and m are large. In interviews, know KMP but mention `s.find()` exists. The interviewer wants to see you understand the LPS concept.

#### Q2: How do I handle case-insensitive comparisons?

**A:** Convert both to lowercase with `tolower()`: `if (tolower(a) == tolower(b))`. For entire strings: `transform(s.begin(), s.end(), s.begin(), ::tolower)`. Don't modify the original if you need it later.

#### Q3: String vs StringBuilder — does C++ need StringBuilder?

**A:** Not exactly. C++ `string` with `.push_back()` or `+=` on single chars is efficient (amortized O(1) like `vector`). The problem is `s = s + other_string` in a loop — this creates copies. Use `s.reserve(n)` to pre-allocate or `ostringstream` for heavy concatenation.

#### Q4: When to use Trie vs HashMap for string lookups?

**A:** **Trie:** prefix-based queries (autocomplete, startsWith), space-efficient for many strings with common prefixes. **HashMap:** exact word lookup, simpler to implement, O(1) average lookup. Use Trie when you need prefix operations.

#### Q5: How do I approach "longest substring with at most K distinct characters"?

**A:** Sliding window template: expand right pointer, track character frequencies with a map. When distinct count > K, shrink left pointer until condition is restored. Track max window size.

#### Q6: What's the difference between subsequence and substring?

**A:** **Substring:** contiguous characters (e.g., "bcd" in "abcde"). **Subsequence:** characters in order but not necessarily contiguous (e.g., "ace" in "abcde"). Substring problems → sliding window. Subsequence problems → DP or two pointers.

#### Q7: How do I handle Unicode strings?

**A:** Use `unordered_map<char, int>` instead of `int[26]`. For wide characters, use `wstring`. In interviews, clarify the character set: "Is the input ASCII only or Unicode?" This changes space complexity.

#### Q8: Why is the sliding window pattern so common for strings?

**A:** Because many string problems ask for "the longest/shortest substring with some property." The sliding window is O(n) and naturally handles substrings with variable conditions by adjusting left and right pointers.

## 15. Real-World Scenarios

### 🔍 Scenario 1: Search Engines — Text Indexing

**Problem:** _"How does Google find web pages matching your query?"_

**Answer — DSA:** Search engines use **inverted indices** (word → list of documents). Query processing uses **string matching** and **tries** for autocomplete. **TF-IDF** scores use word frequency counting. Page content is tokenized and indexed.

Used in: Google Search, Elasticsearch, Apache Solr.

---

### 🔍 Scenario 2: Text Editors — Find & Replace

**Problem:** _"How does Ctrl+F find all occurrences in a document?"_

**Answer — DSA:** **KMP** or **Boyer-Moore** algorithm for pattern matching. "Replace All" involves finding all matches and building a new string. Regular expressions extend this with pattern compilation (NFA/DFA).

Used in: VS Code, Sublime Text, Vim, grep, sed.

---

### 🔍 Scenario 3: DNA Sequencing — Bioinformatics

**Problem:** _"Find a gene sequence within a genome."_

**Answer — DSA:** DNA = string over alphabet {A, C, G, T}. Finding gene patterns = **string matching** (KMP, Rabin-Karp). Sequence alignment uses **DP** (edit distance, longest common subsequence). Suffix arrays enable fast genome search.

Used in: BLAST, GenBank, medical diagnostics.

---

### 🔍 Scenario 4: Compilers — Lexical Analysis

**Problem:** _"How does a compiler break code into tokens?"_

**Answer — DSA:** The lexer reads source code character by character, building tokens. Keywords are looked up in a **trie** or hash set. String literals are parsed with escape handling. Regular expressions define token patterns.

Used in: GCC, Clang, V8 JavaScript engine, Python interpreter.

---

### 🔍 Scenario 5: Plagiarism Detection

**Problem:** _"Detect if two documents are suspiciously similar."_

**Answer — DSA:** **Rabin-Karp** rolling hash computes fingerprints of text windows. **Shingling:** break text into k-grams, compare sets. **Edit distance** measures how many changes needed to transform one into another. Locality-sensitive hashing enables efficient comparison.

Used in: Turnitin, MOSS, Google duplicate content detection.

---

### 🔍 Scenario 6: URL Routing (Web Frameworks)

**Problem:** _"Match an incoming URL to the correct handler."_

**Answer — DSA:** URL patterns like `/users/:id/posts` are stored in a **Trie**. Each path segment is a trie node. Dynamic segments (`:id`) are wildcards. Route matching = trie prefix search with backtracking for wildcards.

Used in: Express.js, Django, Flask, Spring MVC.

---

### 🔍 Scenario 7: Data Compression — LZ77/LZW

**Problem:** _"Compress text by eliminating redundancy."_

**Answer — DSA:** LZ77 finds repeated substrings using a **sliding window** over the text. Matches are encoded as (offset, length) pairs. LZW builds a **dictionary** (trie) of seen substrings. Both rely heavily on string matching techniques.

Used in: gzip, ZIP, PNG, GIF.

---

### 🔍 Scenario 8: Autocomplete & Spell Check

**Problem:** _"Suggest completions as the user types."_

**Answer — DSA:** **Trie** stores all dictionary words. As user types each character, traverse the trie and collect all children. For spell check, use **edit distance** (DP) to find closest dictionary words. **Frequency counts** rank suggestions by popularity.

Used in: Google Search, smartphone keyboards, IDE autocomplete.

---

### 🔍 Scenario 9: Log Parsing & Monitoring

**Problem:** _"Extract timestamps, IPs, and error codes from server logs."_

**Answer — DSA:** **Regex** (compiled to DFA) matches patterns like IP addresses. **String splitting** tokenizes log lines. **Frequency counting** identifies most common errors. Sliding window over time-series detects anomaly bursts.

Used in: ELK Stack, Splunk, Datadog, CloudWatch.

---

### 🔍 Scenario 10: Cryptography — String Hashing

**Problem:** _"Create a unique fingerprint for a password."_

**Answer — DSA:** Password hashing (bcrypt, SHA-256) transforms strings into fixed-length hashes using **modular arithmetic** and **bit operations**. **Rolling hash** (Rabin-Karp) is used in file deduplication. **HMAC** uses string hashing for message authentication.

Used in: Password storage, JWT tokens, blockchain, digital signatures.
