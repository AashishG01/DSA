## 1. What is a Binary Tree?

### 1.1 The Definition

A **Binary Tree** is a hierarchical data structure where each node has **at most 2 children** — a left child and a right child.

```
         1          ← Root
        / \
       2   3        ← Children of 1
      / \   \
     4   5   6      ← Leaves (4, 5, 6) & Internal (2, 3)
```

### 1.2 Key Terminology

| Term | Definition |
|---|---|
| **Root** | The topmost node (no parent) |
| **Leaf** | A node with no children |
| **Internal Node** | A node with at least one child |
| **Edge** | Connection between parent and child |
| **Height** | Longest path from root to any leaf (root height = 0 or 1 depending on convention) |
| **Depth** | Distance from root to a specific node |
| **Level** | All nodes at the same depth |
| **Subtree** | A node and all its descendants |
| **Ancestor** | Any node on the path from a node to the root |
| **Descendant** | Any node in a node's subtree |

### 1.3 Tree Node Structure

```cpp
struct TreeNode {
    int val;
    TreeNode* left;
    TreeNode* right;
    TreeNode(int x) : val(x), left(nullptr), right(nullptr) {}
};
```

### 1.4 Properties & Formulas

| Property | Formula |
|---|---|
| Max nodes at level `l` | 2^l |
| Max nodes in tree of height `h` | 2^(h+1) - 1 |
| Min height for `n` nodes | ⌊log₂(n)⌋ |
| Number of edges | n - 1 |
| Leaf nodes in full BT | internal nodes + 1 |
| NULL pointers in BT with n nodes | n + 1 |

## 2. Types of Binary Trees

### 2.1 Visual Comparison

```
FULL:             COMPLETE:          PERFECT:          DEGENERATE:
    1                1                  1                 1
   / \              / \               / \                 \
  2   3            2   3             2   3                 2
 / \              / \  /            / \ / \                 \
4   5            4  5 6            4  5 6  7                 3
                                                              \
All nodes have   Filled left-to-    All levels        Like a     4
0 or 2 children  right, last may    completely full   linked list
                 be incomplete
```

### 2.2 Type Definitions

| Type | Rule | Properties |
|---|---|---|
| **Full** | Every node has 0 or 2 children | Leaves = Internal + 1 |
| **Complete** | All levels full except last (filled left-to-right) | Can be stored in array, used for heaps |
| **Perfect** | All levels completely filled | n = 2^(h+1) - 1 nodes |
| **Balanced** | Height difference of subtrees ≤ 1 for all nodes | h = O(log n) |
| **Degenerate/Skewed** | Every node has at most 1 child | h = n (like linked list) |

## 3. Tree Traversals — DFS & BFS ⭐⭐

### 3.1 DFS: Inorder (Left → Root → Right)

```cpp
// Recursive
void inorder(TreeNode* root, vector<int>& result) {
    if (!root) return;
    inorder(root->left, result);
    result.push_back(root->val);    // Process root BETWEEN children
    inorder(root->right, result);
}

// Iterative (using stack)
vector<int> inorderIterative(TreeNode* root) {
    vector<int> result;
    stack<TreeNode*> st;
    TreeNode* cur = root;
    
    while (cur || !st.empty()) {
        while (cur) {
            st.push(cur);
            cur = cur->left;       // Go leftmost
        }
        cur = st.top(); st.pop();
        result.push_back(cur->val); // Process
        cur = cur->right;           // Go right
    }
    return result;
}
```

### 3.2 DFS: Preorder (Root → Left → Right) ⭐

```cpp
// Recursive
void preorder(TreeNode* root, vector<int>& result) {
    if (!root) return;
    result.push_back(root->val);    // Process root FIRST
    preorder(root->left, result);
    preorder(root->right, result);
}

// Iterative (using stack)
vector<int> preorderIterative(TreeNode* root) {
    if (!root) return {};
    vector<int> result;
    stack<TreeNode*> st;
    st.push(root);
    
    while (!st.empty()) {
        TreeNode* node = st.top(); st.pop();
        result.push_back(node->val);
        if (node->right) st.push(node->right);  // Right first (LIFO)
        if (node->left) st.push(node->left);     // Left pops first
    }
    return result;
}
```

### 3.3 DFS: Postorder (Left → Right → Root)

```cpp
// Recursive
void postorder(TreeNode* root, vector<int>& result) {
    if (!root) return;
    postorder(root->left, result);
    postorder(root->right, result);
    result.push_back(root->val);    // Process root LAST
}

// Iterative (using two stacks)
vector<int> postorderIterative(TreeNode* root) {
    if (!root) return {};
    vector<int> result;
    stack<TreeNode*> s1, s2;
    s1.push(root);
    
    while (!s1.empty()) {
        TreeNode* node = s1.top(); s1.pop();
        s2.push(node);
        if (node->left) s1.push(node->left);
        if (node->right) s1.push(node->right);
    }
    while (!s2.empty()) {
        result.push_back(s2.top()->val);
        s2.pop();
    }
    return result;
}
```

### 3.4 BFS: Level Order Traversal ⭐

```cpp
vector<vector<int>> levelOrder(TreeNode* root) {
    if (!root) return {};
    vector<vector<int>> result;
    queue<TreeNode*> q;
    q.push(root);
    
    while (!q.empty()) {
        int size = q.size();          // Nodes at current level
        vector<int> level;
        for (int i = 0; i < size; i++) {
            TreeNode* node = q.front(); q.pop();
            level.push_back(node->val);
            if (node->left) q.push(node->left);
            if (node->right) q.push(node->right);
        }
        result.push_back(level);
    }
    return result;
}
// Time: O(n), Space: O(w) where w = max width of tree
```

### 3.5 Traversal Summary

```
Tree:       1
           / \
          2   3
         / \
        4   5

Inorder:    4, 2, 5, 1, 3   (Left → Root → Right)     ← Sorted for BST!
Preorder:   1, 2, 4, 5, 3   (Root → Left → Right)     ← Root is FIRST
Postorder:  4, 5, 2, 3, 1   (Left → Right → Root)     ← Root is LAST
Level:      [1], [2,3], [4,5]                           ← Level by level
```

### 3.6 Morris Traversal — O(1) Space Inorder

```cpp
vector<int> morrisInorder(TreeNode* root) {
    vector<int> result;
    TreeNode* cur = root;
    
    while (cur) {
        if (!cur->left) {
            result.push_back(cur->val);
            cur = cur->right;
        } else {
            // Find inorder predecessor
            TreeNode* pred = cur->left;
            while (pred->right && pred->right != cur)
                pred = pred->right;
            
            if (!pred->right) {
                pred->right = cur;      // Create thread
                cur = cur->left;
            } else {
                pred->right = nullptr;  // Remove thread
                result.push_back(cur->val);
                cur = cur->right;
            }
        }
    }
    return result;
}
// O(n) time, O(1) space — no stack or recursion!
```

## 4. 🧠 How to Think About Tree Problems — Intuition

### 4.1 The TWO Thinking Approaches

> **Approach 1: TOP-DOWN (Preorder thinking)**
> "Pass information from parent to children."
> Process root first, then recurse on children with updated info.
> 
> **Approach 2: BOTTOM-UP (Postorder thinking)** ⭐ Most common
> "Gather results from children, combine at root."
> Recurse on children first, then use their results to compute root's answer.

```
TOP-DOWN example: Max Depth
  "I know my depth. My children's depth = my depth + 1."
  Passed down: current depth.

BOTTOM-UP example: Max Depth
  "I don't know my depth. Ask my children for their heights,
   return max(leftHeight, rightHeight) + 1."
  Returned up: subtree height.
```

### 4.2 The 8 Tree Patterns — Decision Framework

```
┌──────────────────────────────────────────────────────────────────┐
│                   WHICH TREE PATTERN?                            │
├──────────────────────────────────────────────────────────────────┤
│                                                                  │
│  1. HEIGHT / DEPTH / DIAMETER                                    │
│     → "Max depth", "diameter", "balanced check"                  │
│     → Bottom-up: return height from children, combine at root    │
│                                                                  │
│  2. PATH PROBLEMS                                                │
│     → "Path sum", "root-to-leaf", "max path sum"                 │
│     → Top-down (pass sum) or bottom-up (return max path)         │
│                                                                  │
│  3. SYMMETRY / SAME / SUBTREE                                    │
│     → "Mirror?", "Same tree?", "Is subtree?"                     │
│     → Compare two nodes simultaneously                           │
│                                                                  │
│  4. LEVEL ORDER (BFS)                                            │
│     → "Level averages", "zigzag", "right view"                   │
│     → Queue-based BFS, process level by level                    │
│                                                                  │
│  5. LCA & ANCESTORS                                              │
│     → "Lowest Common Ancestor", "distance between nodes"         │
│     → DFS returning target matches                               │
│                                                                  │
│  6. CONSTRUCTION                                                 │
│     → "Build from inorder + preorder"                            │
│     → Use traversal properties to identify root and split        │
│                                                                  │
│  7. SERIALIZE / DESERIALIZE                                      │
│     → "Encode tree to string and back"                           │
│     → Preorder + null markers                                    │
│                                                                  │
│  8. VIEWS & BOUNDARY                                             │
│     → "Left/right/top/bottom view", "boundary traversal"         │
│     → BFS with tracking first/last at each level or column       │
│                                                                  │
└──────────────────────────────────────────────────────────────────┘
```

### 4.3 The Recursive Template for Tree Problems

```cpp
// TEMPLATE: Bottom-up (most common)
ReturnType solve(TreeNode* root) {
    // Base case
    if (!root) return BASE_VALUE;
    
    // Recurse on children
    ReturnType leftResult = solve(root->left);
    ReturnType rightResult = solve(root->right);
    
    // Combine results at current node
    // (possibly update a global answer)
    return COMBINE(leftResult, rightResult, root->val);
}

// TEMPLATE: Top-down
void solve(TreeNode* root, StateType state) {
    if (!root) return;
    
    // Process current node with current state
    PROCESS(root, state);
    
    // Pass updated state to children
    solve(root->left, UPDATE(state));
    solve(root->right, UPDATE(state));
}
```

## 5. Pattern 1: Height / Depth / Diameter ⭐

### 5.1 Maximum Depth of Binary Tree (LC 104) ⭐

```cpp
int maxDepth(TreeNode* root) {
    if (!root) return 0;
    return max(maxDepth(root->left), maxDepth(root->right)) + 1;
}
// Bottom-up: children return their height, parent adds 1.
// Time: O(n), Space: O(h)
```

### 5.2 Minimum Depth (LC 111)

```cpp
int minDepth(TreeNode* root) {
    if (!root) return 0;
    if (!root->left) return minDepth(root->right) + 1;   // no left → go right
    if (!root->right) return minDepth(root->left) + 1;   // no right → go left
    return min(minDepth(root->left), minDepth(root->right)) + 1;
}
// Tricky: a node with only one child is NOT a leaf.
```

### 5.3 Diameter of Binary Tree (LC 543) ⭐⭐

```cpp
int diameterOfBinaryTree(TreeNode* root) {
    int diameter = 0;
    
    function<int(TreeNode*)> height = [&](TreeNode* node) -> int {
        if (!node) return 0;
        int left = height(node->left);
        int right = height(node->right);
        diameter = max(diameter, left + right);  // update global
        return max(left, right) + 1;
    };
    
    height(root);
    return diameter;
}
// Diameter = longest path between ANY two nodes.
// At each node: left_height + right_height = path through this node.
```

### 5.4 Balanced Binary Tree (LC 110)

```cpp
bool isBalanced(TreeNode* root) {
    return checkHeight(root) != -1;
}

int checkHeight(TreeNode* node) {
    if (!node) return 0;
    int left = checkHeight(node->left);
    if (left == -1) return -1;
    int right = checkHeight(node->right);
    if (right == -1) return -1;
    if (abs(left - right) > 1) return -1;  // not balanced
    return max(left, right) + 1;
}
// Return -1 as a signal for "not balanced" — early exit optimization.
```

## 6. Pattern 2: Path Problems ⭐

### 6.1 Path Sum (LC 112)

```cpp
bool hasPathSum(TreeNode* root, int targetSum) {
    if (!root) return false;
    if (!root->left && !root->right) return targetSum == root->val;  // leaf check
    return hasPathSum(root->left, targetSum - root->val) ||
           hasPathSum(root->right, targetSum - root->val);
}
// Top-down: subtract current value, check if leaf sum = 0.
```

### 6.2 Path Sum II — All Root-to-Leaf Paths (LC 113)

```cpp
vector<vector<int>> pathSum(TreeNode* root, int targetSum) {
    vector<vector<int>> result;
    vector<int> path;
    
    function<void(TreeNode*, int)> dfs = [&](TreeNode* node, int remain) {
        if (!node) return;
        path.push_back(node->val);
        
        if (!node->left && !node->right && remain == node->val) {
            result.push_back(path);
        }
        
        dfs(node->left, remain - node->val);
        dfs(node->right, remain - node->val);
        path.pop_back();  // backtrack
    };
    
    dfs(root, targetSum);
    return result;
}
```

### 6.3 Binary Tree Maximum Path Sum (LC 124) ⭐⭐

```cpp
int maxPathSum(TreeNode* root) {
    int maxSum = INT_MIN;
    
    function<int(TreeNode*)> dfs = [&](TreeNode* node) -> int {
        if (!node) return 0;
        
        int left = max(0, dfs(node->left));    // ignore negative paths
        int right = max(0, dfs(node->right));
        
        maxSum = max(maxSum, left + right + node->val);  // path through node
        return max(left, right) + node->val;              // best single path
    };
    
    dfs(root);
    return maxSum;
}
// Key insight: a path can go left-node-right (horseshoe shape),
// but when returning up, we can only choose ONE side.
```

### 6.4 Sum Root to Leaf Numbers (LC 129)

```cpp
int sumNumbers(TreeNode* root) {
    int total = 0;
    
    function<void(TreeNode*, int)> dfs = [&](TreeNode* node, int num) {
        if (!node) return;
        num = num * 10 + node->val;
        if (!node->left && !node->right) { total += num; return; }
        dfs(node->left, num);
        dfs(node->right, num);
    };
    
    dfs(root, 0);
    return total;
}
```

## 7. Pattern 3: Symmetry, Same Tree & Subtree

### 7.1 Same Tree (LC 100)

```cpp
bool isSameTree(TreeNode* p, TreeNode* q) {
    if (!p && !q) return true;
    if (!p || !q) return false;
    return p->val == q->val &&
           isSameTree(p->left, q->left) &&
           isSameTree(p->right, q->right);
}
// Compare two trees simultaneously, node by node.
```

### 7.2 Symmetric Tree (LC 101) ⭐

```cpp
bool isSymmetric(TreeNode* root) {
    return root ? isMirror(root->left, root->right) : true;
}

bool isMirror(TreeNode* a, TreeNode* b) {
    if (!a && !b) return true;
    if (!a || !b) return false;
    return a->val == b->val &&
           isMirror(a->left, b->right) &&   // outer vs outer
           isMirror(a->right, b->left);      // inner vs inner
}
// Symmetric = left subtree mirrors right subtree.
```

### 7.3 Subtree of Another Tree (LC 572)

```cpp
bool isSubtree(TreeNode* root, TreeNode* subRoot) {
    if (!root) return false;
    if (isSameTree(root, subRoot)) return true;
    return isSubtree(root->left, subRoot) || isSubtree(root->right, subRoot);
}
// At each node: is this subtree identical to subRoot?
// Time: O(m × n) worst case
```

### 7.4 Invert Binary Tree (LC 226) ⭐

```cpp
TreeNode* invertTree(TreeNode* root) {
    if (!root) return nullptr;
    swap(root->left, root->right);
    invertTree(root->left);
    invertTree(root->right);
    return root;
}
// Swap children at every node. Simple but classic!
```

## 8. Pattern 4: Level Order (BFS) Problems ⭐

### 8.1 Zigzag Level Order Traversal (LC 103)

```cpp
vector<vector<int>> zigzagLevelOrder(TreeNode* root) {
    if (!root) return {};
    vector<vector<int>> result;
    queue<TreeNode*> q;
    q.push(root);
    bool leftToRight = true;
    
    while (!q.empty()) {
        int size = q.size();
        vector<int> level(size);
        for (int i = 0; i < size; i++) {
            TreeNode* node = q.front(); q.pop();
            int idx = leftToRight ? i : size - 1 - i;
            level[idx] = node->val;
            if (node->left) q.push(node->left);
            if (node->right) q.push(node->right);
        }
        result.push_back(level);
        leftToRight = !leftToRight;
    }
    return result;
}
```

### 8.2 Right Side View (LC 199) ⭐

```cpp
vector<int> rightSideView(TreeNode* root) {
    if (!root) return {};
    vector<int> result;
    queue<TreeNode*> q;
    q.push(root);
    
    while (!q.empty()) {
        int size = q.size();
        for (int i = 0; i < size; i++) {
            TreeNode* node = q.front(); q.pop();
            if (i == size - 1) result.push_back(node->val);  // last in level
            if (node->left) q.push(node->left);
            if (node->right) q.push(node->right);
        }
    }
    return result;
}
// DFS alternative: preorder but visit right first, track depth.
```

### 8.3 Level Order Average (LC 637)

```cpp
vector<double> averageOfLevels(TreeNode* root) {
    vector<double> result;
    queue<TreeNode*> q;
    q.push(root);
    
    while (!q.empty()) {
        int size = q.size();
        double sum = 0;
        for (int i = 0; i < size; i++) {
            TreeNode* node = q.front(); q.pop();
            sum += node->val;
            if (node->left) q.push(node->left);
            if (node->right) q.push(node->right);
        }
        result.push_back(sum / size);
    }
    return result;
}
```

### 8.4 Maximum Width of Binary Tree (LC 662) ⭐

```cpp
int widthOfBinaryTree(TreeNode* root) {
    if (!root) return 0;
    unsigned long long maxWidth = 0;
    queue<pair<TreeNode*, unsigned long long>> q;
    q.push({root, 0});
    
    while (!q.empty()) {
        int size = q.size();
        unsigned long long left = q.front().second;
        unsigned long long right = left;
        for (int i = 0; i < size; i++) {
            auto [node, idx] = q.front(); q.pop();
            right = idx;
            if (node->left) q.push({node->left, 2 * (idx - left)});
            if (node->right) q.push({node->right, 2 * (idx - left) + 1});
        }
        maxWidth = max(maxWidth, right - left + 1);
    }
    return maxWidth;
}
// Use index: left child = 2*i, right child = 2*i+1.
// Subtract offset to prevent overflow.
```

### 8.5 Populating Next Right Pointers (LC 116)

```cpp
Node* connect(Node* root) {
    if (!root) return root;
    queue<Node*> q;
    q.push(root);
    
    while (!q.empty()) {
        int size = q.size();
        for (int i = 0; i < size; i++) {
            Node* node = q.front(); q.pop();
            node->next = (i < size - 1) ? q.front() : nullptr;
            if (node->left) q.push(node->left);
            if (node->right) q.push(node->right);
        }
    }
    return root;
}
```

## 9. Pattern 5: LCA & Ancestor Problems ⭐⭐

### 9.1 Core Intuition

> **LCA (Lowest Common Ancestor):** The deepest node that is an ancestor of BOTH target nodes.
> - If both targets are in left subtree → LCA is in left.
> - If both targets are in right subtree → LCA is in right.
> - If one is in left and one is in right → current node IS the LCA.

### 9.2 LCA of Binary Tree (LC 236) ⭐⭐

```cpp
TreeNode* lowestCommonAncestor(TreeNode* root, TreeNode* p, TreeNode* q) {
    if (!root || root == p || root == q) return root;
    
    TreeNode* left = lowestCommonAncestor(root->left, p, q);
    TreeNode* right = lowestCommonAncestor(root->right, p, q);
    
    if (left && right) return root;  // split point → root is LCA
    return left ? left : right;       // both in one subtree
}
// Time: O(n), Space: O(h)
// This is the MOST important tree problem to master!
```

### 9.3 How LCA Works — Step by Step

```
Tree:         3
            /   \
           5     1
          / \   / \
         6   2 0   8
            / \
           7   4

Find LCA of 5 and 1:
- Root 3: left returns 5, right returns 1 → BOTH non-null → LCA = 3 ✅

Find LCA of 5 and 4:
- Root 3: left returns 5 (found), right returns null → LCA = 5 ✅
  Because 4 is a descendant of 5.

Find LCA of 7 and 4:
- Root 3 → left(5) → left(6) returns null, right(2) → left(7) returns 7, right(4) returns 4
  Both non-null at 2 → LCA = 2 ✅
```

### 9.4 Distance Between Two Nodes

```cpp
int findDistance(TreeNode* root, TreeNode* p, TreeNode* q) {
    TreeNode* lca = lowestCommonAncestor(root, p, q);
    return depth(lca, p, 0) + depth(lca, q, 0);
}

int depth(TreeNode* root, TreeNode* target, int d) {
    if (!root) return -1;
    if (root == target) return d;
    int left = depth(root->left, target, d + 1);
    if (left != -1) return left;
    return depth(root->right, target, d + 1);
}
// Distance = depth(LCA to p) + depth(LCA to q)
```

## 10. Pattern 6: Construction from Traversals ⭐

### 10.1 Core Insight

> **Inorder + Preorder** → Unique binary tree.
> **Inorder + Postorder** → Unique binary tree.
> **Preorder + Postorder** → NOT unique (ambiguous).
>
> Inorder tells you LEFT vs RIGHT. Preorder/Postorder tells you the ROOT.

### 10.2 Build Tree from Inorder + Preorder (LC 105) ⭐⭐

```cpp
TreeNode* buildTree(vector<int>& preorder, vector<int>& inorder) {
    unordered_map<int, int> inMap;
    for (int i = 0; i < inorder.size(); i++) inMap[inorder[i]] = i;
    
    int preIdx = 0;
    return build(preorder, inorder, preIdx, 0, inorder.size() - 1, inMap);
}

TreeNode* build(vector<int>& pre, vector<int>& in, int& preIdx,
                int inStart, int inEnd, unordered_map<int,int>& inMap) {
    if (inStart > inEnd) return nullptr;
    
    int rootVal = pre[preIdx++];
    TreeNode* root = new TreeNode(rootVal);
    int inIdx = inMap[rootVal];  // root's position in inorder
    
    root->left = build(pre, in, preIdx, inStart, inIdx - 1, inMap);
    root->right = build(pre, in, preIdx, inIdx + 1, inEnd, inMap);
    return root;
}
// Preorder: first element = root.
// Inorder: everything left of root = left subtree, right = right subtree.
```

### 10.3 Build Tree from Inorder + Postorder (LC 106)

```cpp
TreeNode* buildTree(vector<int>& inorder, vector<int>& postorder) {
    unordered_map<int, int> inMap;
    for (int i = 0; i < inorder.size(); i++) inMap[inorder[i]] = i;
    
    int postIdx = postorder.size() - 1;
    return build(postorder, inMap, postIdx, 0, inorder.size() - 1);
}

TreeNode* build(vector<int>& post, unordered_map<int,int>& inMap,
                int& postIdx, int inStart, int inEnd) {
    if (inStart > inEnd) return nullptr;
    
    int rootVal = post[postIdx--];
    TreeNode* root = new TreeNode(rootVal);
    int inIdx = inMap[rootVal];
    
    root->right = build(post, inMap, postIdx, inIdx + 1, inEnd);  // RIGHT first!
    root->left = build(post, inMap, postIdx, inStart, inIdx - 1);
    return root;
}
// Postorder: last element = root. Build RIGHT subtree first.
```

### 10.4 Maximum Binary Tree (LC 654)

```cpp
TreeNode* constructMaximumBinaryTree(vector<int>& nums) {
    return build(nums, 0, nums.size() - 1);
}

TreeNode* build(vector<int>& nums, int l, int r) {
    if (l > r) return nullptr;
    int maxIdx = l;
    for (int i = l + 1; i <= r; i++)
        if (nums[i] > nums[maxIdx]) maxIdx = i;
    
    TreeNode* root = new TreeNode(nums[maxIdx]);
    root->left = build(nums, l, maxIdx - 1);
    root->right = build(nums, maxIdx + 1, r);
    return root;
}
```

## 11. Pattern 7: Serialize / Deserialize ⭐

### 11.1 Serialize & Deserialize Binary Tree (LC 297) ⭐⭐

```cpp
class Codec {
public:
    // Preorder serialization with "null" markers
    string serialize(TreeNode* root) {
        if (!root) return "null,";
        return to_string(root->val) + "," +
               serialize(root->left) +
               serialize(root->right);
    }
    
    TreeNode* deserialize(string data) {
        queue<string> tokens;
        stringstream ss(data);
        string token;
        while (getline(ss, token, ','))
            tokens.push(token);
        return buildTree(tokens);
    }
    
private:
    TreeNode* buildTree(queue<string>& tokens) {
        string val = tokens.front(); tokens.pop();
        if (val == "null") return nullptr;
        
        TreeNode* node = new TreeNode(stoi(val));
        node->left = buildTree(tokens);
        node->right = buildTree(tokens);
        return node;
    }
};
// Preorder + null markers uniquely represents any binary tree.
// "1,2,null,null,3,4,null,null,5,null,null"
```

### 11.2 BFS Serialization (Alternative)

```cpp
string serializeBFS(TreeNode* root) {
    if (!root) return "null";
    string result;
    queue<TreeNode*> q;
    q.push(root);
    
    while (!q.empty()) {
        TreeNode* node = q.front(); q.pop();
        if (node) {
            result += to_string(node->val) + ",";
            q.push(node->left);
            q.push(node->right);
        } else {
            result += "null,";
        }
    }
    return result;
}
```

## 12. Pattern 8: Views & Boundary Traversal

### 12.1 Top View of Binary Tree

```cpp
vector<int> topView(TreeNode* root) {
    if (!root) return {};
    map<int, int> topMap;  // horizontal distance → node value
    queue<pair<TreeNode*, int>> q;
    q.push({root, 0});
    
    while (!q.empty()) {
        auto [node, hd] = q.front(); q.pop();
        if (topMap.find(hd) == topMap.end()) {
            topMap[hd] = node->val;  // first node at this horizontal distance
        }
        if (node->left) q.push({node->left, hd - 1});
        if (node->right) q.push({node->right, hd + 1});
    }
    
    vector<int> result;
    for (auto& [hd, val] : topMap) result.push_back(val);
    return result;
}
```

### 12.2 Bottom View of Binary Tree

```cpp
vector<int> bottomView(TreeNode* root) {
    if (!root) return {};
    map<int, int> bottomMap;
    queue<pair<TreeNode*, int>> q;
    q.push({root, 0});
    
    while (!q.empty()) {
        auto [node, hd] = q.front(); q.pop();
        bottomMap[hd] = node->val;  // last node at this horizontal distance
        if (node->left) q.push({node->left, hd - 1});
        if (node->right) q.push({node->right, hd + 1});
    }
    
    vector<int> result;
    for (auto& [hd, val] : bottomMap) result.push_back(val);
    return result;
}
```

### 12.3 Vertical Order Traversal (LC 987) ⭐

```cpp
vector<vector<int>> verticalTraversal(TreeNode* root) {
    if (!root) return {};
    map<int, map<int, multiset<int>>> nodes;  // col → row → values
    queue<tuple<TreeNode*, int, int>> q;
    q.push({root, 0, 0});
    
    while (!q.empty()) {
        auto [node, col, row] = q.front(); q.pop();
        nodes[col][row].insert(node->val);
        if (node->left) q.push({node->left, col - 1, row + 1});
        if (node->right) q.push({node->right, col + 1, row + 1});
    }
    
    vector<vector<int>> result;
    for (auto& [col, rows] : nodes) {
        vector<int> colVals;
        for (auto& [row, vals] : rows)
            for (int v : vals) colVals.push_back(v);
        result.push_back(colVals);
    }
    return result;
}
```

### 12.4 Boundary Traversal

```cpp
vector<int> boundaryTraversal(TreeNode* root) {
    if (!root) return {};
    vector<int> result;
    if (!isLeaf(root)) result.push_back(root->val);
    
    addLeftBoundary(root, result);
    addLeaves(root, result);
    addRightBoundary(root, result);
    return result;
}

bool isLeaf(TreeNode* node) { return node && !node->left && !node->right; }

void addLeftBoundary(TreeNode* root, vector<int>& res) {
    TreeNode* cur = root->left;
    while (cur) {
        if (!isLeaf(cur)) res.push_back(cur->val);
        cur = cur->left ? cur->left : cur->right;
    }
}

void addLeaves(TreeNode* root, vector<int>& res) {
    if (!root) return;
    if (isLeaf(root)) { res.push_back(root->val); return; }
    addLeaves(root->left, res);
    addLeaves(root->right, res);
}

void addRightBoundary(TreeNode* root, vector<int>& res) {
    TreeNode* cur = root->right;
    vector<int> temp;
    while (cur) {
        if (!isLeaf(cur)) temp.push_back(cur->val);
        cur = cur->right ? cur->right : cur->left;
    }
    for (int i = temp.size() - 1; i >= 0; i--) res.push_back(temp[i]);
}
// Left boundary (top-down) + all leaves + right boundary (bottom-up)
```

## 14. Common Mistakes & Debugging

### 14.1 Top 12 Tree Mistakes

| # | Mistake | Fix |
|---|---|---|
| 1 | **Forgetting null check** `root->left` when root is null | Always start with `if (!root) return ...` |
| 2 | **Confusing height vs depth** | Height = bottom-up (leaf=0). Depth = top-down (root=0). |
| 3 | **Minimum depth single-child trap** | A node with only left child is NOT a leaf. Check both children null. |
| 4 | **Modifying tree during traversal** | Use a copy or collect values first, then modify. |
| 5 | **Stack overflow on large trees** | Use iterative traversal (stack/queue) instead of recursion. |
| 6 | **Wrong LCA — checking only direct parent** | LCA uses DFS on both subtrees: if both return non-null → root is LCA. |
| 7 | **Construction: wrong index math** | HashMap for inorder indices. Preorder idx increments, not computed. |
| 8 | **Level order: forgetting `size = q.size()`** | Capture level size BEFORE processing (queue grows during loop). |
| 9 | **Diameter: returning left+right** | Diameter is the global max, but RETURN only max(left,right)+1 up. |
| 10 | **Max path sum: not clamping negatives** | `left = max(0, dfs(left))` — ignore negative paths! |
| 11 | **Assuming binary tree is BST** | BT has no ordering. Don't use BST properties on generic BT. |
| 12 | **Memory leaks** | In C++, remember to `delete` nodes if you replace or remove them. |

### 14.2 Debugging Checklist

```
□ Did I handle the base case (root == nullptr)?
□ Is my approach top-down or bottom-up? Am I consistent?
□ For bottom-up: am I returning meaningful values from children?
□ For top-down: am I passing updated state correctly?
□ Am I updating a global variable where needed (diameter, max path)?
□ For BFS: am I using `int size = q.size()` before the inner loop?
□ Edge cases: empty tree, single node, skewed tree, all same values?
□ Is my recursion depth safe? (deep trees may need iterative approach)
```

## 15. Interview FAQs

#### Q1: What's the difference between DFS and BFS on trees?

**A:** **DFS (Depth-First Search)** goes deep before wide — uses recursion or stack. Three variants: inorder, preorder, postorder. Time & space: O(n) and O(h). **BFS (Breadth-First Search)** goes level by level — uses queue. Time & space: O(n) and O(w) where w = max width. Use DFS for depth/path problems, BFS for level-related problems.

#### Q2: When to use top-down vs bottom-up?

**A:** **Bottom-up** (most common): When you need information from children to compute the answer (height, diameter, LCA). Return values UP the recursion. **Top-down**: When you're passing context DOWN (current depth, path sum remaining, accumulated value). Use when children need parent's info.

#### Q3: How do I identify if DFS or BFS is better?

**A:** 
- **BFS**: "level order", "zigzag", "right view", "width", "next pointers" — anything level-based.
- **DFS**: "path sum", "height", "diameter", "LCA", "serialize" — anything depth/structure-based.
- Often DFS is simpler to code recursively. BFS is simpler with a queue.

#### Q4: Why does inorder + preorder uniquely determine a tree?

**A:** Preorder gives you the root (first element). Inorder splits elements into left and right subtrees relative to root. Combined, you know: (1) what the root is, (2) which elements are in left vs right subtree. Apply recursively.

#### Q5: How does the Max Path Sum differ from regular Path Sum?

**A:** Regular Path Sum: root-to-leaf, path goes only downward. Max Path Sum: ANY node to ANY node, path can go up-down through some "turning point." At each node, the path through it = left + node + right. But when returning to parent, you can only use ONE side (left or right, not both).

#### Q6: What's the time/space complexity of tree problems?

**A:** Almost all tree problems are **O(n)** time (visit every node once). Space is **O(h)** for DFS recursion/stack (h = log n balanced, n for skewed). BFS space is **O(w)** where w = max width (up to n/2 for complete trees).

#### Q7: How do I handle trees with duplicate values?

**A:** Duplicates make construction ambiguous (inorder + preorder may not be unique). For LCA, find nodes by reference (pointer), not value. For "count occurrences," do a full traversal. Always clarify with interviewer if duplicates are possible.

#### Q8: When should I use Morris Traversal?

**A:** When O(1) space is required and you can't use recursion or a stack. Morris modifies the tree temporarily (creates threads) but restores it. It's O(n) time and O(1) space. Use for inorder or preorder when space is critical.

## 16. Real-World Scenarios

### 🔍 Scenario 1: DOM (Document Object Model)

**Problem:** _"How does the browser represent an HTML page?"_

**Answer — DSA:** The DOM is a **tree**. `<html>` is the root, `<head>` and `<body>` are children. CSS selectors traverse this tree (DFS). `document.querySelector` does DFS to find elements. React's virtual DOM does tree diffing (same tree comparison).

Used in: Every web browser, React, Angular, Vue.

---

### 🔍 Scenario 2: File Systems

**Problem:** _"How does `ls -R` list all files recursively?"_

**Answer — DSA:** File systems are **trees**. Root = `/` or `C:\`. Directories = internal nodes. Files = leaves. `ls -R` is a **DFS preorder** traversal. `du` (disk usage) is a **DFS postorder** — compute child sizes first, then add to parent.

Used in: Linux/Windows/Mac file systems, cloud storage (S3 prefixes).

---

### 🔍 Scenario 3: Compiler — Abstract Syntax Trees (AST)

**Problem:** _"How does a compiler understand `(3 + 5) * 2`?"_

**Answer — DSA:** The compiler builds an **AST** (tree). Operators are internal nodes, operands are leaves. Evaluating the expression is a **postorder traversal**: evaluate children first, then apply the operator.

```
      *
     / \
    +   2
   / \
  3   5

Postorder: 3, 5, +, 2, * → ((3+5)*2) = 16
```

Used in: GCC, Clang, V8 (JavaScript), Python interpreter.

---

### 🔍 Scenario 4: XML / JSON Parsing

**Problem:** _"How do you navigate a nested JSON object?"_

**Answer — DSA:** JSON is a **tree**. Objects = nodes with named children. Arrays = nodes with indexed children. `jq` queries use path traversal (DFS). JSON Schema validation walks the tree comparing structure (same tree pattern).

Used in: REST APIs, configuration files, data interchange.

---

### 🔍 Scenario 5: Organization Hierarchy

**Problem:** _"Find the lowest-level manager common to two employees."_

**Answer — DSA:** This is exactly the **LCA (Lowest Common Ancestor)** problem! The org chart is a tree. Finding the reporting chain uses DFS. The LCA is the most specific manager who oversees both employees.

Used in: HR systems, access control, corporate reporting.

---

### 🔍 Scenario 6: Decision Trees (Machine Learning)

**Problem:** _"Classify a new data point based on features."_

**Answer — DSA:** Decision trees are **binary trees**. Each internal node tests a feature. Each edge represents a decision. Leaves are classifications. Prediction = root-to-leaf path. Training builds the tree using entropy/Gini (recursive splitting).

Used in: Random Forests, XGBoost, medical diagnosis, credit scoring.

---

### 🔍 Scenario 7: Network Routing — Spanning Trees

**Problem:** _"How does a network avoid loops?"_

**Answer — DSA:** STP (Spanning Tree Protocol) builds a **tree** from the network graph, ensuring no cycles. BFS from root switch determines the tree structure. This prevents broadcast storms in Ethernet networks.

Used in: Ethernet switches, enterprise networks, STP protocol.

---

### 🔍 Scenario 8: Git — Commit History

**Problem:** _"How does Git track branching and merging?"_

**Answer — DSA:** Git's commit history forms a **DAG** (directed acyclic graph), but individual branches are tree-like paths. Operations like `git log --graph` visualize the tree structure. Merge commits have multiple parents (DAG, not pure tree).

Used in: Git, Mercurial, SVN.

---

### 🔍 Scenario 9: Game AI — Minimax (Chess, Tic-Tac-Toe)

**Problem:** _"How does a chess engine evaluate the best move?"_

**Answer — DSA:** The game tree is a **tree** where each node is a game state and edges are moves. Minimax with alpha-beta pruning is a **DFS postorder** traversal: evaluate leaf positions (base case), propagate scores up (bottom-up).

Used in: Chess engines (Stockfish), Go (AlphaGo), game AI.

---

### 🔍 Scenario 10: Databases — B-Trees & Query Plans

**Problem:** _"How does a SQL query executor evaluate `SELECT ... WHERE ... JOIN ...`?"_

**Answer — DSA:** The query execution plan is a **tree**. Leaf nodes = table scans. Internal nodes = join/filter/sort operators. The optimizer builds this tree. Execution is **postorder**: children provide rows to parent operators.

Used in: MySQL, PostgreSQL, Oracle, SQL Server query optimizers.
