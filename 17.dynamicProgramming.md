## 1. What is Dynamic Programming?

### 1.1 The Definition

> **Dynamic Programming** = solving a problem by breaking it into **overlapping subproblems**, solving each **once**, and **storing the result** to avoid recomputation.

### 1.2 When to Use DP

Two conditions must BOTH be true:
1. **Optimal Substructure** — The optimal solution uses optimal solutions to subproblems.
2. **Overlapping Subproblems** — The same subproblem is solved multiple times.

```
Example: Fibonacci
  fib(5) = fib(4) + fib(3)
  fib(4) = fib(3) + fib(2)    ← fib(3) computed TWICE!

Without DP: O(2^n) — exponential (recomputation)
With DP:    O(n)   — linear (store & reuse)
```

### 1.3 DP vs Greedy vs Divide & Conquer

| | DP | Greedy | Divide & Conquer |
|---|---|---|---|
| Subproblems | Overlapping | No subproblems | Non-overlapping |
| Optimal | Global optimal ✅ | Local optimal (not always global) | Depends |
| Approach | Try all, pick best | Pick best at each step | Recursively solve halves |
| Example | Knapsack, LCS | Activity selection, Huffman | Merge sort, Quick sort |

## 2. 🧠 How to Think About DP — The Framework ⭐⭐

### 2.1 The 5-Step DP Recipe

```
Step 1: DEFINE STATE
  "What do I need to know at each step?"
  → dp[i] = answer for the first i elements / at position i
  → dp[i][j] = answer using rows 0..i, columns 0..j

Step 2: DEFINE TRANSITION (Recurrence)
  "How does dp[i] relate to smaller subproblems?"
  → dp[i] = dp[i-1] + dp[i-2]                     (Fibonacci)
  → dp[i] = max(dp[i-1], dp[i-2] + val[i])        (House Robber)
  → dp[i][j] = dp[i-1][j-1] + 1  if match         (LCS)

Step 3: BASE CASE
  "What's the answer for the smallest inputs?"
  → dp[0] = 0 or 1 (depends on problem)
  → dp[0][j] = ..., dp[i][0] = ...

Step 4: ITERATION ORDER
  "In what order should I fill the DP table?"
  → Left to right? Bottom to top? Diagonal?

Step 5: OPTIMIZE SPACE (if possible)
  "Do I need the full table, or just the last row/few values?"
  → 2D → 1D, 1D → O(1)
```


## 3. Top-Down (Memoization) vs Bottom-Up (Tabulation)

### 3.1 Comparison

| | Top-Down (Memoization) | Bottom-Up (Tabulation) |
|---|---|---|
| Approach | Recursive + cache | Iterative + table |
| Think | Natural (start from answer, break down) | Build from base cases up |
| Code style | `memo[key]` + recursion | `for` loops + `dp[]` array |
| Subproblems | Only solves needed ones | Solves ALL subproblems |
| Stack overflow | Possible for deep recursion | No risk |
| Optimization | Harder to space-optimize | Easier to roll to O(1) |
| When to use | Easier to think, sparse subproblems | Better performance, clear order |

### 3.2 Fibonacci — Both Approaches

```cpp
// Top-Down (Memoization)
int memo[101];
memset(memo, -1, sizeof(memo));

int fib(int n) {
    if (n <= 1) return n;
    if (memo[n] != -1) return memo[n];
    return memo[n] = fib(n-1) + fib(n-2);
}

// Bottom-Up (Tabulation)
int fib(int n) {
    if (n <= 1) return n;
    vector<int> dp(n + 1);
    dp[0] = 0; dp[1] = 1;
    for (int i = 2; i <= n; i++)
        dp[i] = dp[i-1] + dp[i-2];
    return dp[n];
}

// Space-Optimized O(1)
int fib(int n) {
    if (n <= 1) return n;
    int prev2 = 0, prev1 = 1;
    for (int i = 2; i <= n; i++) {
        int curr = prev1 + prev2;
        prev2 = prev1;
        prev1 = curr;
    }
    return prev1;
}
```

## 4. Pattern 1: Fibonacci / Linear DP ⭐

### 4.1 Template

```
dp[i] depends on dp[i-1], dp[i-2], ... (previous states)
Fill left to right.
Often can be space-optimized to O(1).
```

### 4.2 Climbing Stairs (LC 70) ⭐

```cpp
int climbStairs(int n) {
    if (n <= 2) return n;
    int prev2 = 1, prev1 = 2;
    for (int i = 3; i <= n; i++) {
        int curr = prev1 + prev2;
        prev2 = prev1;
        prev1 = curr;
    }
    return prev1;
}
// dp[i] = dp[i-1] + dp[i-2]
// Ways to reach step i = from step i-1 (1 step) + from step i-2 (2 steps)
```

### 4.3 Min Cost Climbing Stairs (LC 746)

```cpp
int minCostClimbingStairs(vector<int>& cost) {
    int n = cost.size();
    int prev2 = 0, prev1 = 0;
    for (int i = 2; i <= n; i++) {
        int curr = min(prev1 + cost[i-1], prev2 + cost[i-2]);
        prev2 = prev1;
        prev1 = curr;
    }
    return prev1;
}
// dp[i] = min(dp[i-1] + cost[i-1], dp[i-2] + cost[i-2])
```

### 4.4 Decode Ways (LC 91) ⭐

```cpp
int numDecodings(string s) {
    if (s[0] == '0') return 0;
    int n = s.size();
    int prev2 = 1, prev1 = 1;
    
    for (int i = 2; i <= n; i++) {
        int curr = 0;
        int oneDigit = s[i-1] - '0';
        int twoDigit = stoi(s.substr(i-2, 2));
        
        if (oneDigit >= 1) curr += prev1;       // single digit valid
        if (twoDigit >= 10 && twoDigit <= 26) curr += prev2;  // two digits valid
        
        prev2 = prev1;
        prev1 = curr;
    }
    return prev1;
}
```

## 5. Pattern 2: House Robber / Take or Skip ⭐

### 5.1 Core Insight

> At each element: **TAKE it** (skip previous) or **SKIP it** (keep previous answer).
> `dp[i] = max(dp[i-1], dp[i-2] + nums[i])`

### 5.2 House Robber (LC 198) ⭐⭐

```cpp
int rob(vector<int>& nums) {
    int prev2 = 0, prev1 = 0;
    for (int num : nums) {
        int curr = max(prev1, prev2 + num);
        prev2 = prev1;
        prev1 = curr;
    }
    return prev1;
}
// Take current + best from 2 ago, or skip current and keep best from 1 ago.
```

### 5.3 House Robber II — Circular (LC 213)

```cpp
int rob(vector<int>& nums) {
    if (nums.size() == 1) return nums[0];
    return max(robRange(nums, 0, nums.size() - 2),
               robRange(nums, 1, nums.size() - 1));
}

int robRange(vector<int>& nums, int start, int end) {
    int prev2 = 0, prev1 = 0;
    for (int i = start; i <= end; i++) {
        int curr = max(prev1, prev2 + nums[i]);
        prev2 = prev1;
        prev1 = curr;
    }
    return prev1;
}
// Circular: either rob [0..n-2] or [1..n-1], take max.
```

### 5.4 Delete and Earn (LC 740) ⭐

```cpp
int deleteAndEarn(vector<int>& nums) {
    int maxVal = *max_element(nums.begin(), nums.end());
    vector<int> points(maxVal + 1, 0);
    for (int n : nums) points[n] += n;
    
    int prev2 = 0, prev1 = 0;
    for (int i = 0; i <= maxVal; i++) {
        int curr = max(prev1, prev2 + points[i]);
        prev2 = prev1;
        prev1 = curr;
    }
    return prev1;
}
// Transform into House Robber: taking value i means deleting i-1 and i+1.
```

## 6. Pattern 3: Counting Ways ⭐

### 6.1 Unique Paths (LC 62) ⭐

```cpp
int uniquePaths(int m, int n) {
    vector<vector<int>> dp(m, vector<int>(n, 1));
    for (int i = 1; i < m; i++)
        for (int j = 1; j < n; j++)
            dp[i][j] = dp[i-1][j] + dp[i][j-1];
    return dp[m-1][n-1];
}
// dp[i][j] = ways to reach (i,j) = from top + from left
```

### 6.2 Unique Paths II — With Obstacles (LC 63)

```cpp
int uniquePathsWithObstacles(vector<vector<int>>& grid) {
    int m = grid.size(), n = grid[0].size();
    if (grid[0][0] || grid[m-1][n-1]) return 0;
    
    vector<vector<long>> dp(m, vector<long>(n, 0));
    dp[0][0] = 1;
    
    for (int i = 0; i < m; i++)
        for (int j = 0; j < n; j++) {
            if (grid[i][j]) { dp[i][j] = 0; continue; }
            if (i > 0) dp[i][j] += dp[i-1][j];
            if (j > 0) dp[i][j] += dp[i][j-1];
        }
    return dp[m-1][n-1];
}
```

## 7. Pattern 4: Longest Common Subsequence (LCS) ⭐⭐

### 7.1 Core Insight

> Compare two strings character by character. If they match, extend the common subsequence. If not, try dropping one character from either string.

### 7.2 LCS (LC 1143) ⭐⭐

```cpp
int longestCommonSubsequence(string text1, string text2) {
    int m = text1.size(), n = text2.size();
    vector<vector<int>> dp(m + 1, vector<int>(n + 1, 0));
    
    for (int i = 1; i <= m; i++) {
        for (int j = 1; j <= n; j++) {
            if (text1[i-1] == text2[j-1])
                dp[i][j] = dp[i-1][j-1] + 1;    // match: extend
            else
                dp[i][j] = max(dp[i-1][j], dp[i][j-1]);  // skip one
        }
    }
    return dp[m][n];
}
// Time: O(m × n), Space: O(m × n)
// dp[i][j] = LCS of text1[0..i-1] and text2[0..j-1]
```

### 7.3 LCS — Space Optimized to O(n)

```cpp
int longestCommonSubsequence(string text1, string text2) {
    int m = text1.size(), n = text2.size();
    vector<int> prev(n + 1, 0), curr(n + 1, 0);
    
    for (int i = 1; i <= m; i++) {
        for (int j = 1; j <= n; j++) {
            if (text1[i-1] == text2[j-1])
                curr[j] = prev[j-1] + 1;
            else
                curr[j] = max(prev[j], curr[j-1]);
        }
        swap(prev, curr);
        fill(curr.begin(), curr.end(), 0);
    }
    return prev[n];
}
// Only need previous row → O(n) space.
```

### 7.4 Longest Common Substring (not subsequence!)

```cpp
int longestCommonSubstring(string s1, string s2) {
    int m = s1.size(), n = s2.size(), maxLen = 0;
    vector<vector<int>> dp(m + 1, vector<int>(n + 1, 0));
    
    for (int i = 1; i <= m; i++)
        for (int j = 1; j <= n; j++) {
            if (s1[i-1] == s2[j-1])
                dp[i][j] = dp[i-1][j-1] + 1;
            else
                dp[i][j] = 0;  // reset! (must be contiguous)
            maxLen = max(maxLen, dp[i][j]);
        }
    return maxLen;
}
// Substring: reset to 0 on mismatch. Subsequence: take max.
```

## 8. Pattern 5: Longest Increasing Subsequence (LIS) ⭐⭐

### 8.1 LIS — O(n²) DP

```cpp
int lengthOfLIS(vector<int>& nums) {
    int n = nums.size();
    vector<int> dp(n, 1);  // every element is a subsequence of length 1
    int maxLen = 1;
    
    for (int i = 1; i < n; i++) {
        for (int j = 0; j < i; j++) {
            if (nums[j] < nums[i])
                dp[i] = max(dp[i], dp[j] + 1);
        }
        maxLen = max(maxLen, dp[i]);
    }
    return maxLen;
}
// dp[i] = length of LIS ending at index i.
```

### 8.2 LIS — O(n log n) with Binary Search ⭐⭐

```cpp
int lengthOfLIS(vector<int>& nums) {
    vector<int> tails;  // tails[i] = smallest tail of all increasing subsequences of length i+1
    
    for (int num : nums) {
        auto it = lower_bound(tails.begin(), tails.end(), num);
        if (it == tails.end())
            tails.push_back(num);    // extend
        else
            *it = num;               // replace to keep tails small
    }
    return tails.size();
}
// Patience sorting! tails is always sorted.
// Time: O(n log n), Space: O(n)
```

### 8.3 Number of LIS (LC 673)

```cpp
int findNumberOfLIS(vector<int>& nums) {
    int n = nums.size(), maxLen = 1;
    vector<int> len(n, 1), cnt(n, 1);
    
    for (int i = 1; i < n; i++) {
        for (int j = 0; j < i; j++) {
            if (nums[j] < nums[i]) {
                if (len[j] + 1 > len[i]) {
                    len[i] = len[j] + 1;
                    cnt[i] = cnt[j];
                } else if (len[j] + 1 == len[i]) {
                    cnt[i] += cnt[j];
                }
            }
        }
        maxLen = max(maxLen, len[i]);
    }
    
    int total = 0;
    for (int i = 0; i < n; i++)
        if (len[i] == maxLen) total += cnt[i];
    return total;
}
```

## 9. Pattern 6: Edit Distance / String DP ⭐

### 9.1 Edit Distance (LC 72) ⭐⭐

```cpp
int minDistance(string word1, string word2) {
    int m = word1.size(), n = word2.size();
    vector<vector<int>> dp(m + 1, vector<int>(n + 1, 0));
    
    for (int i = 0; i <= m; i++) dp[i][0] = i;  // delete all
    for (int j = 0; j <= n; j++) dp[0][j] = j;  // insert all
    
    for (int i = 1; i <= m; i++) {
        for (int j = 1; j <= n; j++) {
            if (word1[i-1] == word2[j-1])
                dp[i][j] = dp[i-1][j-1];           // no operation
            else
                dp[i][j] = 1 + min({dp[i-1][j],     // delete
                                    dp[i][j-1],      // insert
                                    dp[i-1][j-1]});  // replace
        }
    }
    return dp[m][n];
}
// dp[i][j] = min operations to convert word1[0..i-1] to word2[0..j-1]
```

### 9.2 Distinct Subsequences (LC 115)

```cpp
int numDistinct(string s, string t) {
    int m = s.size(), n = t.size();
    vector<vector<unsigned long long>> dp(m + 1, vector<unsigned long long>(n + 1, 0));
    
    for (int i = 0; i <= m; i++) dp[i][0] = 1;  // empty t always matches
    
    for (int i = 1; i <= m; i++)
        for (int j = 1; j <= n; j++) {
            dp[i][j] = dp[i-1][j];  // skip s[i-1]
            if (s[i-1] == t[j-1])
                dp[i][j] += dp[i-1][j-1];  // use s[i-1]
        }
    return dp[m][n];
}
```

### 9.3 Word Break (LC 139) ⭐

```cpp
bool wordBreak(string s, vector<string>& wordDict) {
    unordered_set<string> dict(wordDict.begin(), wordDict.end());
    int n = s.size();
    vector<bool> dp(n + 1, false);
    dp[0] = true;
    
    for (int i = 1; i <= n; i++)
        for (int j = 0; j < i; j++)
            if (dp[j] && dict.count(s.substr(j, i - j))) {
                dp[i] = true;
                break;
            }
    return dp[n];
}
// dp[i] = can s[0..i-1] be segmented into dictionary words?
```

## 10. Pattern 7: Grid DP ⭐

### 10.1 Minimum Path Sum (LC 64) ⭐

```cpp
int minPathSum(vector<vector<int>>& grid) {
    int m = grid.size(), n = grid[0].size();
    
    for (int i = 0; i < m; i++)
        for (int j = 0; j < n; j++) {
            if (i == 0 && j == 0) continue;
            else if (i == 0) grid[i][j] += grid[i][j-1];
            else if (j == 0) grid[i][j] += grid[i-1][j];
            else grid[i][j] += min(grid[i-1][j], grid[i][j-1]);
        }
    return grid[m-1][n-1];
}
// In-place modification. dp[i][j] = min cost to reach (i,j).
```

### 10.2 Triangle (LC 120)

```cpp
int minimumTotal(vector<vector<int>>& triangle) {
    for (int i = triangle.size() - 2; i >= 0; i--)
        for (int j = 0; j <= i; j++)
            triangle[i][j] += min(triangle[i+1][j], triangle[i+1][j+1]);
    return triangle[0][0];
}
// Bottom-up: start from last row, propagate minimum path up.
```

### 10.3 Maximal Square (LC 221) ⭐⭐

```cpp
int maximalSquare(vector<vector<char>>& matrix) {
    int m = matrix.size(), n = matrix[0].size(), maxSide = 0;
    vector<vector<int>> dp(m, vector<int>(n, 0));
    
    for (int i = 0; i < m; i++)
        for (int j = 0; j < n; j++) {
            if (matrix[i][j] == '1') {
                if (i == 0 || j == 0) dp[i][j] = 1;
                else dp[i][j] = min({dp[i-1][j], dp[i][j-1], dp[i-1][j-1]}) + 1;
                maxSide = max(maxSide, dp[i][j]);
            }
        }
    return maxSide * maxSide;
}
// dp[i][j] = side length of largest square with bottom-right corner at (i,j).
// Depends on top, left, and top-left.
```

## 11. Pattern 8: 0/1 Knapsack ⭐⭐

### 11.1 The Classic Problem

> Given `n` items with weights and values, and a knapsack of capacity `W`, maximize value without exceeding weight. Each item used **at most once**.

### 11.2 0/1 Knapsack Template

```cpp
int knapsack01(int W, vector<int>& wt, vector<int>& val, int n) {
    vector<vector<int>> dp(n + 1, vector<int>(W + 1, 0));
    
    for (int i = 1; i <= n; i++) {
        for (int w = 0; w <= W; w++) {
            dp[i][w] = dp[i-1][w];  // don't take item i
            if (wt[i-1] <= w)
                dp[i][w] = max(dp[i][w], dp[i-1][w - wt[i-1]] + val[i-1]);  // take item i
        }
    }
    return dp[n][W];
}
// dp[i][w] = max value using first i items with capacity w.
// Time: O(n × W), Space: O(n × W)
```

### 11.3 Space-Optimized 0/1 Knapsack — O(W)

```cpp
int knapsack01(int W, vector<int>& wt, vector<int>& val, int n) {
    vector<int> dp(W + 1, 0);
    
    for (int i = 0; i < n; i++) {
        for (int w = W; w >= wt[i]; w--) {  // RIGHT to LEFT! (prevents reuse)
            dp[w] = max(dp[w], dp[w - wt[i]] + val[i]);
        }
    }
    return dp[W];
}
// Key: iterate w from W down to wt[i] to ensure each item used once!
```

### 11.4 Target Sum (LC 494) ⭐

```cpp
int findTargetSumWays(vector<int>& nums, int target) {
    int sum = accumulate(nums.begin(), nums.end(), 0);
    if ((sum + target) % 2 != 0 || sum + target < 0) return 0;
    int subsetSum = (sum + target) / 2;
    
    vector<int> dp(subsetSum + 1, 0);
    dp[0] = 1;
    
    for (int num : nums)
        for (int s = subsetSum; s >= num; s--)
            dp[s] += dp[s - num];
    
    return dp[subsetSum];
}
// Transform: find subsets P,N where P-N=target, P+N=sum → P=(sum+target)/2
// Then count subsets with sum = P (0/1 knapsack counting).
```

## 12. Pattern 9: Unbounded Knapsack (Coin Change) ⭐

### 12.1 Key Difference from 0/1

```
0/1 Knapsack:       Each item used AT MOST ONCE
                    → iterate w from RIGHT to LEFT

Unbounded Knapsack: Each item used UNLIMITED times
                    → iterate w from LEFT to RIGHT
```

### 12.2 Coin Change — Min Coins (LC 322) ⭐⭐

```cpp
int coinChange(vector<int>& coins, int amount) {
    vector<int> dp(amount + 1, amount + 1);
    dp[0] = 0;
    
    for (int i = 1; i <= amount; i++) {
        for (int coin : coins) {
            if (coin <= i)
                dp[i] = min(dp[i], dp[i - coin] + 1);
        }
    }
    return dp[amount] > amount ? -1 : dp[amount];
}
// dp[i] = min coins to make amount i.
// Unbounded: same coin can be used multiple times.
```

### 12.3 Coin Change II — Count Ways (LC 518) ⭐

```cpp
int change(int amount, vector<int>& coins) {
    vector<int> dp(amount + 1, 0);
    dp[0] = 1;
    
    for (int coin : coins) {                    // iterate coins in outer loop
        for (int a = coin; a <= amount; a++) {  // left to right (unbounded)
            dp[a] += dp[a - coin];
        }
    }
    return dp[amount];
}
// Coins in outer loop → combinations (not permutations).
// dp[a] = number of ways to make amount a.
```

### 12.4 Unbounded Knapsack — Max Value

```cpp
int unboundedKnapsack(int W, vector<int>& wt, vector<int>& val, int n) {
    vector<int> dp(W + 1, 0);
    
    for (int w = 0; w <= W; w++) {
        for (int i = 0; i < n; i++) {
            if (wt[i] <= w)
                dp[w] = max(dp[w], dp[w - wt[i]] + val[i]);
        }
    }
    return dp[W];
}
// Left to right iteration allows reusing items!
```

### 12.5 Perfect Squares (LC 279) ⭐

```cpp
int numSquares(int n) {
    vector<int> dp(n + 1, INT_MAX);
    dp[0] = 0;
    
    for (int i = 1; i <= n; i++)
        for (int j = 1; j * j <= i; j++)
            dp[i] = min(dp[i], dp[i - j*j] + 1);
    
    return dp[n];
}
// Same as coin change, but "coins" are perfect squares 1, 4, 9, 16, ...
```

## 13. Pattern 10: Partition DP (Subset Sum) ⭐

### 13.1 Subset Sum — Can We Make Target?

```cpp
bool subsetSum(vector<int>& nums, int target) {
    vector<bool> dp(target + 1, false);
    dp[0] = true;
    
    for (int num : nums)
        for (int s = target; s >= num; s--)  // right to left (0/1)
            dp[s] = dp[s] || dp[s - num];
    
    return dp[target];
}
```

### 13.2 Partition Equal Subset Sum (LC 416) ⭐

```cpp
bool canPartition(vector<int>& nums) {
    int sum = accumulate(nums.begin(), nums.end(), 0);
    if (sum % 2 != 0) return false;
    
    int target = sum / 2;
    vector<bool> dp(target + 1, false);
    dp[0] = true;
    
    for (int num : nums)
        for (int s = target; s >= num; s--)
            dp[s] = dp[s] || dp[s - num];
    
    return dp[target];
}
// Partition into 2 equal subsets → find subset with sum = totalSum/2.
```

### 13.3 Last Stone Weight II (LC 1049)

```cpp
int lastStoneWeightII(vector<int>& stones) {
    int sum = accumulate(stones.begin(), stones.end(), 0);
    int target = sum / 2;
    
    vector<bool> dp(target + 1, false);
    dp[0] = true;
    for (int s : stones)
        for (int w = target; w >= s; w--)
            dp[w] = dp[w] || dp[w - s];
    
    for (int w = target; w >= 0; w--)
        if (dp[w]) return sum - 2 * w;
    return 0;
}
// Partition into two groups. Minimize |group1 - group2| = sum - 2*group1.
```

## 14. Pattern 11: Interval / Range DP ⭐

### 14.1 Core Intuition

> **Range DP:** Solve for subproblems on ranges `[i, j]`, combining smaller ranges.
> `dp[i][j]` depends on splitting point `k` within `[i, j]`.

### 14.2 Matrix Chain Multiplication

```cpp
int matrixChainOrder(vector<int>& dims) {
    int n = dims.size() - 1;  // n matrices
    vector<vector<int>> dp(n, vector<int>(n, 0));
    
    for (int len = 2; len <= n; len++) {         // length of chain
        for (int i = 0; i <= n - len; i++) {     // start index
            int j = i + len - 1;                  // end index
            dp[i][j] = INT_MAX;
            for (int k = i; k < j; k++) {        // split point
                int cost = dp[i][k] + dp[k+1][j] + dims[i]*dims[k+1]*dims[j+1];
                dp[i][j] = min(dp[i][j], cost);
            }
        }
    }
    return dp[0][n-1];
}
// Fill diagonally: length 2, then 3, then 4, ...
```

### 14.3 Burst Balloons (LC 312) ⭐⭐

```cpp
int maxCoins(vector<int>& nums) {
    int n = nums.size();
    vector<int> a(n + 2, 1);
    for (int i = 0; i < n; i++) a[i+1] = nums[i];
    
    vector<vector<int>> dp(n + 2, vector<int>(n + 2, 0));
    
    for (int len = 1; len <= n; len++) {
        for (int i = 1; i <= n - len + 1; i++) {
            int j = i + len - 1;
            for (int k = i; k <= j; k++) {
                // k is the LAST balloon to burst in range [i, j]
                dp[i][j] = max(dp[i][j],
                    dp[i][k-1] + a[i-1]*a[k]*a[j+1] + dp[k+1][j]);
            }
        }
    }
    return dp[1][n];
}
// Key insight: think of k as the LAST balloon to burst (not first).
```

### 14.4 Palindrome Partitioning II — Min Cuts (LC 132)

```cpp
int minCut(string s) {
    int n = s.size();
    vector<vector<bool>> isPalin(n, vector<bool>(n, false));
    vector<int> dp(n, 0);
    
    // Precompute palindromes
    for (int i = n-1; i >= 0; i--)
        for (int j = i; j < n; j++)
            isPalin[i][j] = (s[i] == s[j]) && (j - i < 2 || isPalin[i+1][j-1]);
    
    // dp[i] = min cuts for s[0..i]
    for (int i = 0; i < n; i++) {
        if (isPalin[0][i]) { dp[i] = 0; continue; }
        dp[i] = i;  // worst case: cut every character
        for (int j = 1; j <= i; j++)
            if (isPalin[j][i])
                dp[i] = min(dp[i], dp[j-1] + 1);
    }
    return dp[n-1];
}
```

## 15. Pattern 12: DP on Trees ⭐

### 15.1 Core Insight

> Process children first (postorder), then combine at the parent.
> `dp[node]` typically depends on `dp[child1]`, `dp[child2]`, etc.

### 15.2 House Robber III — on Binary Tree (LC 337) ⭐

```cpp
pair<int,int> dfs(TreeNode* node) {
    if (!node) return {0, 0};
    
    auto [leftRob, leftSkip] = dfs(node->left);
    auto [rightRob, rightSkip] = dfs(node->right);
    
    int rob = node->val + leftSkip + rightSkip;     // rob this, skip children
    int skip = max(leftRob, leftSkip) + max(rightRob, rightSkip);  // skip this
    
    return {rob, skip};
}

int rob(TreeNode* root) {
    auto [robRoot, skipRoot] = dfs(root);
    return max(robRoot, skipRoot);
}
// Return pair: {max if robbed, max if skipped} for each node.
```

### 15.3 Diameter of Binary Tree — as DP (LC 543)

```cpp
int diameter = 0;

int dfs(TreeNode* node) {
    if (!node) return 0;
    int left = dfs(node->left);
    int right = dfs(node->right);
    diameter = max(diameter, left + right);  // path through this node
    return max(left, right) + 1;             // height to return
}

int diameterOfBinaryTree(TreeNode* root) {
    dfs(root);
    return diameter;
}
// dp[node] = height. Answer = max(left_height + right_height) across all nodes.
```

## 16. Pattern 13: Bitmask DP ⭐

### 16.1 Core Intuition

> Use a bitmask to represent which items/nodes have been "used."
> State: `dp[mask]` where each bit in mask = "item i is used."
> Useful when n ≤ 20 (since 2^20 = 1M states).

### 16.2 Partition to K Equal Sum Subsets (LC 698)

```cpp
bool canPartitionKSubsets(vector<int>& nums, int k) {
    int sum = accumulate(nums.begin(), nums.end(), 0);
    if (sum % k != 0) return false;
    int target = sum / k, n = nums.size();
    
    vector<int> dp(1 << n, -1);  // -1 = unvisited
    dp[0] = 0;
    sort(nums.begin(), nums.end());
    
    for (int mask = 0; mask < (1 << n); mask++) {
        if (dp[mask] == -1) continue;
        for (int i = 0; i < n; i++) {
            if (mask & (1 << i)) continue;  // already used
            if (dp[mask] + nums[i] > target) break;  // pruning (sorted)
            
            int newMask = mask | (1 << i);
            dp[newMask] = (dp[mask] + nums[i]) % target;
        }
    }
    return dp[(1 << n) - 1] == 0;
}
// dp[mask] = current bucket sum mod target.
// If all items used and sum works out → partition possible.
```

## 17. Pattern 14: State Machine DP (Stock Problems) ⭐⭐

### 17.1 The State Machine Framework

```
Every stock problem has states:
  → HOLD (have stock)     
  → SOLD (just sold)
  → REST (cooldown / no stock)

Transitions:
  REST → REST (do nothing)
  REST → HOLD (buy)
  HOLD → HOLD (do nothing)
  HOLD → SOLD (sell)
  SOLD → REST (cooldown, if applicable)
```

### 17.2 Best Time to Buy and Sell Stock (LC 121) ⭐

```cpp
int maxProfit(vector<int>& prices) {
    int minPrice = INT_MAX, maxProfit = 0;
    for (int price : prices) {
        minPrice = min(minPrice, price);
        maxProfit = max(maxProfit, price - minPrice);
    }
    return maxProfit;
}
// Single transaction. Track min price so far, maximize profit.
```

### 17.3 Best Time — Unlimited Transactions (LC 122)

```cpp
int maxProfit(vector<int>& prices) {
    int profit = 0;
    for (int i = 1; i < prices.size(); i++)
        if (prices[i] > prices[i-1])
            profit += prices[i] - prices[i-1];
    return profit;
}
// Collect every uphill. Greedy works!
```

### 17.4 Best Time — At Most 2 Transactions (LC 123) ⭐

```cpp
int maxProfit(vector<int>& prices) {
    int buy1 = INT_MAX, buy2 = INT_MAX;
    int profit1 = 0, profit2 = 0;
    
    for (int price : prices) {
        buy1 = min(buy1, price);
        profit1 = max(profit1, price - buy1);
        buy2 = min(buy2, price - profit1);  // effective cost of 2nd buy
        profit2 = max(profit2, price - buy2);
    }
    return profit2;
}
```

### 17.5 Best Time — At Most K Transactions (LC 188) ⭐

```cpp
int maxProfit(int k, vector<int>& prices) {
    int n = prices.size();
    if (k >= n / 2) {  // unlimited transactions
        int profit = 0;
        for (int i = 1; i < n; i++)
            profit += max(0, prices[i] - prices[i-1]);
        return profit;
    }
    
    vector<vector<int>> dp(k + 1, vector<int>(n, 0));
    for (int t = 1; t <= k; t++) {
        int maxDiff = -prices[0];
        for (int d = 1; d < n; d++) {
            dp[t][d] = max(dp[t][d-1], prices[d] + maxDiff);
            maxDiff = max(maxDiff, dp[t-1][d] - prices[d]);
        }
    }
    return dp[k][n-1];
}
```

### 17.6 Best Time — With Cooldown (LC 309) ⭐

```cpp
int maxProfit(vector<int>& prices) {
    int n = prices.size();
    if (n < 2) return 0;
    
    int hold = -prices[0], sold = 0, rest = 0;
    
    for (int i = 1; i < n; i++) {
        int newHold = max(hold, rest - prices[i]);   // buy from rest
        int newSold = hold + prices[i];               // sell
        int newRest = max(rest, sold);                // cooldown or stay
        hold = newHold; sold = newSold; rest = newRest;
    }
    return max(sold, rest);
}
// 3 states: hold, sold (just sold), rest.
// After selling, must rest 1 day before buying again.
```

### 17.7 Best Time — With Transaction Fee (LC 714)

```cpp
int maxProfit(vector<int>& prices, int fee) {
    int hold = -prices[0], cash = 0;
    
    for (int i = 1; i < prices.size(); i++) {
        hold = max(hold, cash - prices[i]);
        cash = max(cash, hold + prices[i] - fee);
    }
    return cash;
}
// Same as unlimited, but subtract fee when selling.
```

## 19. Common Mistakes & Debugging

### 19.1 Top 12 DP Mistakes

| # | Mistake | Fix |
|---|---|---|
| 1 | **No base case** | Always set `dp[0]`, `dp[0][0]`, etc. before the loop |
| 2 | **Wrong transition** | Trace through a small example by hand |
| 3 | **Off-by-one in dp array size** | `dp(n+1)` if using 1-indexed, `dp(n)` if 0-indexed |
| 4 | **0/1 Knapsack: left-to-right iteration** | Must iterate **right to left** for 0/1 (prevents reuse)! |
| 5 | **Unbounded Knapsack: right-to-left** | Must iterate **left to right** for unbounded! |
| 6 | **Coin Change II: inner/outer loop swap** | Coins in outer = combinations. Coins in inner = permutations. |
| 7 | **Not returning dp[n] (1-indexed) vs dp[n-1] (0-indexed)** | Know your indexing convention |
| 8 | **Forgetting to initialize dp with INF/0/-1** | `INT_MAX` for min problems, `0` for max, `-1` for memo |
| 9 | **Space optimization breaks access pattern** | Verify you only access `dp[i-1]` and `dp[i]`, not `dp[i+1]` |
| 10 | **Memoization: no cache check** | Always check `if (memo[i] != -1) return memo[i];` FIRST |
| 11 | **Interval DP: wrong fill order** | Must fill by increasing length (len=1, len=2, ...) |
| 12 | **Stock DP: confusing hold/cash/rest states** | Draw the state machine diagram first |

### 19.2 Debugging Checklist

```
□ Can I trace dp values for a small input manually?
□ Base case: does dp[0] / dp[0][0] make sense?
□ Transition: does dp[i] correctly depend on previous values?
□ Iteration order: am I filling the table in the right direction?
□ Final answer: am I returning dp[n] or dp[n-1]?
□ Edge cases: empty input, single element, zero target?
□ Overflow: do I need long long for large values?
□ Can I optimize space? (2D → 1D → O(1))
```

## 20. Interview FAQs

#### Q1: How do I know it's a DP problem?

**A:** Look for these signals: (1) "Count the number of ways" → likely DP. (2) "Find the minimum/maximum" with **choices** → likely DP. (3) You see **overlapping subproblems** — same computations repeated. (4) The brute force is exponential (2^n, n!) → DP might reduce it to polynomial.

#### Q2: Top-down or bottom-up — which should I use in interviews?

**A:** **Top-down** (memoization) is easier to think about — start from the answer and break down. **Bottom-up** (tabulation) is more efficient and easier to space-optimize. In interviews, start with top-down if you're stuck on the order, then mention you could convert to bottom-up.

#### Q3: How do I optimize DP space from 2D to 1D?

**A:** If `dp[i][j]` only depends on `dp[i-1][...]` (previous row), you only need 2 rows → swap them. If it only depends on `dp[i-1][j]` and `dp[i][j-1]`, you need just 1 row with careful iteration direction.

#### Q4: What's the difference between 0/1 and unbounded knapsack?

**A:** **0/1:** Each item used at most once. Iterate weight **right to left** (prevents reusing items from current row). **Unbounded:** Items can be reused. Iterate weight **left to right** (allows reusing from current row).

#### Q5: How do I handle "count combinations" vs "count permutations"?

**A:** **Combinations** (order doesn't matter): Items in outer loop, amount in inner loop. **Permutations** (order matters): Amount in outer loop, items in inner loop. Example: Coin Change II (combinations) vs Combination Sum IV (permutations).

#### Q6: Can Greedy replace DP?

**A:** Only if the **greedy choice property** holds — the locally optimal choice always leads to a globally optimal solution. Greedy works for: activity selection, fractional knapsack, Huffman coding. Greedy FAILS for: 0/1 knapsack, coin change (arbitrary denominations), LCS.

#### Q7: When do I use interval DP vs regular DP?

**A:** Use interval DP when the problem involves **merging** or **splitting ranges**: burst balloons, matrix chain multiplication, palindrome partitioning. The state is `dp[i][j]` for range `[i, j]`, and you try every split point `k`.

#### Q8: DP vs DFS + Memoization — are they the same?

**A:** Yes! DFS + memoization IS top-down DP. The states are the function parameters, the cache is the memo array. The difference is purely implementation: recursive (top-down) vs iterative (bottom-up). They compute the same values.

## 21. Real-World Scenarios

### 🔍 Scenario 1: Google Search — Spell Correction

**Problem:** _"Did you mean: dynamic programming?"_

**Answer — DSA:** **Edit Distance** (Levenshtein distance) measures how many character operations (insert, delete, replace) to transform the query into a dictionary word. The closest words are suggested. DP table computed in O(m × n).

Used in: Google Search, spell checkers, autocorrect.

### 🔍 Scenario 2: Netflix — Recommendation Sequences

**Problem:** _"What should the user watch next based on their history?"_

**Answer — DSA:** **Longest Common Subsequence** between a user's watching history and other users' histories finds similar patterns. Sequence prediction uses DP-based models. Watch-next suggestions use patterns mined from sequential data.

Used in: Netflix, YouTube, Spotify.

---

### 🔍 Scenario 3: Finance — Stock Trading Algorithms

**Problem:** _"Maximize profit from buying and selling stocks with constraints."_

**Answer — DSA:** **State Machine DP** exactly models this: hold/sold/rest states, with transaction limits, cooldown periods, and fees. Algorithmic trading systems use variants of these DP solutions at microsecond scales.

Used in: Quantitative trading, hedge funds, robo-advisors.

---

### 🔍 Scenario 4: Shipping — Resource Allocation

**Problem:** _"Pack the most valuable items into a container with weight limits."_

**Answer — DSA:** **0/1 Knapsack** for single-use items, **Unbounded Knapsack** for unlimited stock items. Multi-dimensional knapsack handles volume + weight constraints. This is the classic operations research problem.

Used in: Amazon logistics, FedEx, UPS, warehouse optimization.

---

### 🔍 Scenario 5: Bioinformatics — DNA Sequence Alignment

**Problem:** _"Align two DNA sequences to find similarities."_

**Answer — DSA:** **Edit Distance / LCS** adapted for biological sequences (with gap penalties). BLAST algorithm uses DP for local sequence alignment. Protein structure prediction relies on DP formulations.

Used in: GenBank, medical diagnostics, drug discovery.

---

### 🔍 Scenario 6: Text Processing — Diff Tools

**Problem:** _"Show what changed between two versions of a file."_

**Answer — DSA:** `git diff` and similar tools use **LCS** to find the longest common parts between two files. The differences (additions/deletions) are the parts NOT in the LCS. Classic DP problem at scale.

Used in: Git, GitHub, diff, patch, merge tools.

### 🔍 Scenario 7: Natural Language — Word Segmentation

**Problem:** _"Break 'ilovecoding' into 'i love coding'."_

**Answer — DSA:** **Word Break** DP (LC 139). `dp[i] = true` if `s[0..i]` can be segmented using the dictionary. Chinese/Japanese text segmentation (no spaces) is a critical NLP DP problem.

Used in: Google Translate, Chinese/Japanese input methods, NLP tokenizers.

### 🔍 Scenario 8: Network — Shortest Path Routing

**Problem:** _"Find the minimum-cost path through a network."_

**Answer — DSA:** **Grid DP / Min Path Sum** for structured networks. **Bellman-Ford** (which IS a DP algorithm) computes shortest paths by relaxing edges V-1 times. Floyd-Warshall is DP for all-pairs shortest paths.

Used in: Network routing (OSPF, BGP), CDNs, packet switching.

### 🔍 Scenario 9: Games — AI Decision Making

**Problem:** _"What's the optimal strategy in a two-player game?"_

**Answer — DSA:** Game theory uses **minimax with DP** (memoized game states). Poker AI uses DP for optimal betting. Chess endgame tablebases are massive DP tables. Even Wordle solvers use DP for information-optimal guesses.

Used in: Chess engines, Go (AlphaGo), Poker AI, game solvers.

### 🔍 Scenario 10: Compilers — Code Optimization

**Problem:** _"Generate the fastest possible machine code."_

**Answer — DSA:** **Register allocation** uses DP (optimal substructure over instruction sequences). **Instruction scheduling** uses DP to minimize pipeline stalls. Expression tree evaluation order (like matrix chain multiplication) minimizes register spills.

Used in: GCC, LLVM, JIT compilers, V8 engine.
