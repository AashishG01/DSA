## 1. What are Stacks & Queues?

### 1.1 Stack — LIFO (Last In, First Out)

```
STACK = A pile of plates
Push → put plate on TOP
Pop  → remove plate from TOP

   ┌─────┐
   │  5  │  ← TOP (last in, first out)
   ├─────┤
   │  3  │
   ├─────┤
   │  8  │
   ├─────┤
   │  1  │  ← BOTTOM
   └─────┘
   
push(5) → 5 goes on top
pop()   → 5 comes off top
peek()  → see 5 without removing
```

**When to use:** The most recent element matters most. Undo operations, backtracking, function calls, bracket matching.

### 1.2 Queue — FIFO (First In, First Out)

```
QUEUE = A line at a restaurant
Enqueue → join at the BACK
Dequeue → served from the FRONT

   FRONT → [1] [8] [3] [5] ← BACK
   
enqueue(5) → 5 joins at back
dequeue()  → 1 leaves from front
front()    → see 1 without removing
```

**When to use:** Order of arrival matters. BFS, level-order traversal, task scheduling.

### 1.3 Deque — Double-Ended Queue

```
   ← push_front / pop_front    push_back / pop_back →
   FRONT → [1] [8] [3] [5] ← BACK
   
Can add/remove from BOTH ends in O(1).
```

**When to use:** Need both stack AND queue behavior. Sliding window max/min.

### 1.4 Quick Comparison

| Property | Stack | Queue | Deque |
|---|---|---|---|
| Order | LIFO | FIFO | Both |
| Insert | Top only | Back only | Both ends |
| Remove | Top only | Front only | Both ends |
| C++ | `stack<int>` | `queue<int>` | `deque<int>` |
| All ops | O(1) | O(1) | O(1) |
| Use for | Undo, DFS, matching | BFS, scheduling | Sliding window, both |

## 2. Stack — Implementation & Operations

### 2.1 Using C++ STL

```cpp
#include <stack>

stack<int> st;

st.push(10);      // push to top
st.push(20);
st.push(30);

st.top();          // 30 (peek, doesn't remove)
st.pop();          // removes 30 (returns void!)
st.top();          // 20
st.size();         // 2
st.empty();        // false
```

> ⚠️ `st.pop()` returns `void` in C++! Use `st.top()` first to get value, THEN `pop()`.

### 2.2 Implement Stack Using Array

```cpp
class ArrayStack {
    int arr[10000];
    int topIdx;
public:
    ArrayStack() : topIdx(-1) {}
    void push(int x) { arr[++topIdx] = x; }
    int pop() { return arr[topIdx--]; }
    int top() { return arr[topIdx]; }
    bool empty() { return topIdx == -1; }
    int size() { return topIdx + 1; }
};
```

### 2.3 Implement Stack Using Linked List

```cpp
class LinkedStack {
    struct Node {
        int data;
        Node* next;
        Node(int v) : data(v), next(nullptr) {}
    };
    Node* topNode;
public:
    LinkedStack() : topNode(nullptr) {}
    void push(int x) {
        Node* n = new Node(x);
        n->next = topNode;
        topNode = n;
    }
    int pop() {
        int val = topNode->data;
        Node* temp = topNode;
        topNode = topNode->next;
        delete temp;
        return val;
    }
    int top() { return topNode->data; }
    bool empty() { return topNode == nullptr; }
};
```

## 3. Queue, Deque & Priority Queue

### 3.1 Queue Using STL

```cpp
#include <queue>

queue<int> q;
q.push(10);       // enqueue at back
q.push(20);
q.front();         // 10
q.back();          // 20
q.pop();           // removes 10 (from front)
q.size();          // 1
q.empty();         // false
```

### 3.2 Circular Queue (Array-Based)

```cpp
class CircularQueue {
    int arr[1000];
    int front, rear, sz, cap;
public:
    CircularQueue(int k) : front(0), rear(-1), sz(0), cap(k) {}
    
    bool enqueue(int x) {
        if (sz == cap) return false;
        rear = (rear + 1) % cap;
        arr[rear] = x;
        sz++;
        return true;
    }
    
    int dequeue() {
        if (sz == 0) return -1;
        int val = arr[front];
        front = (front + 1) % cap;
        sz--;
        return val;
    }
    
    int getFront() { return sz == 0 ? -1 : arr[front]; }
    bool isEmpty() { return sz == 0; }
    bool isFull() { return sz == cap; }
};
```

### 3.3 Deque Using STL

```cpp
#include <deque>

deque<int> dq;
dq.push_back(10);     // add to back
dq.push_front(5);     // add to front
dq.front();            // 5
dq.back();             // 10
dq.pop_front();        // removes 5
dq.pop_back();         // removes 10
dq[i];                 // random access O(1)!
```

### 3.4 Priority Queue (Min-Heap / Max-Heap)

```cpp
#include <queue>

// Max-Heap (default)
priority_queue<int> maxPQ;
maxPQ.push(3); maxPQ.push(1); maxPQ.push(5);
maxPQ.top();   // 5 (largest)

// Min-Heap
priority_queue<int, vector<int>, greater<int>> minPQ;
minPQ.push(3); minPQ.push(1); minPQ.push(5);
minPQ.top();   // 1 (smallest)

// Custom comparator (e.g., sort by second element)
auto cmp = [](pair<int,int>& a, pair<int,int>& b) {
    return a.second > b.second;  // min-heap by second
};
priority_queue<pair<int,int>, vector<pair<int,int>>, decltype(cmp)> pq(cmp);
```

| Operation | Time |
|---|---|
| push | O(log n) |
| pop | O(log n) |
| top | O(1) |
| size | O(1) |

## 4. 🧠 How to Think About Stack Problems — Intuition

> This is the most important section. Understanding WHEN and WHY to use a stack is more important than memorizing solutions.

### 4.1 The ONE Question to Ask Yourself

> **"Does the most RECENT unseen/unresolved element matter?"**
>
> If YES → Use a **Stack**.

Examples:
- Bracket matching: the most recent **open** bracket must match the current close bracket.
- Undo: the most **recent** action is undone first.
- Next Greater Element: for each element, the most **recently seen** larger element matters.
- Function calls: the most **recently called** function returns first.

### 4.2 The 6 Stack Patterns — Decision Framework

```
┌─────────────────────────────────────────────────────────────────┐
│                  WHEN TO USE A STACK?                            │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  1. MATCHING/BALANCING                                          │
│     → "Is this balanced?" Brackets, HTML tags, palindromes      │
│     → Push opening, pop on closing, check match                 │
│                                                                 │
│  2. MONOTONIC STACK (Next Greater / Next Smaller)               │
│     → "What's the next larger/smaller element?"                 │
│     → Maintain increasing/decreasing stack                      │
│     → Also: Stock span, histogram, temperatures                 │
│                                                                 │
│  3. HISTORY/UNDO                                                │
│     → "Go back to previous state"                               │
│     → Browser back, undo, backspace, simplify path              │
│                                                                 │
│  4. EXPRESSION EVALUATION                                       │
│     → "Calculate 3 + 4 * 2"                                     │
│     → Operator precedence, infix/postfix/prefix                 │
│                                                                 │
│  5. DFS / RECURSION SIMULATION                                  │
│     → Convert recursive DFS to iterative using explicit stack   │
│                                                                 │
│  6. DESIGN                                                      │
│     → Min Stack, Max Stack, Stack using Queues                  │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 4.3 When to Use a QUEUE Instead

```
Stack (LIFO):  Most RECENT first    →  DFS, undo, matching
Queue (FIFO):  FIRST come first    →  BFS, level-order, scheduling

Ask: "Does ORDER OF ARRIVAL matter?"
  YES → Queue (BFS, scheduling, first-come-first-served)
  NO, most RECENT matters → Stack (DFS, undo, matching)
```

### 4.4 The Approach Template for Stack Problems

```
Step 1: Read the problem. Ask: "Does the most recent element matter?"
Step 2: Identify the pattern (matching, monotonic, history, expression, DFS)
Step 3: Decide what to STORE in the stack:
        - The element itself? (brackets, values)
        - The INDEX? (monotonic stack for distances/spans)
        - Both? (sometimes need (value, index) or (value, count))
Step 4: Decide WHEN to push and WHEN to pop:
        - Push: when we see a new "open" thing (bracket, unresolved element)
        - Pop: when we find the "close" / "answer" for the top element
Step 5: Handle what's LEFT in the stack:
        - Unmatched brackets? → invalid
        - Remaining elements? → they have no "next greater" → answer is -1
```

## 5. Pattern 1: Matching & Balancing

### 5.1 Valid Parentheses (LC 20) ⭐

**Intuition:** Every closing bracket must match the MOST RECENT unmatched opening bracket. That's LIFO = Stack.

```cpp
bool isValid(string s) {
    stack<char> st;
    for (char c : s) {
        if (c == '(' || c == '{' || c == '[') {
            st.push(c);
        } else {
            if (st.empty()) return false;
            char top = st.top();
            if ((c == ')' && top == '(') ||
                (c == '}' && top == '{') ||
                (c == ']' && top == '[')) {
                st.pop();
            } else return false;
        }
    }
    return st.empty();
}
// Time: O(n), Space: O(n)
```

### 5.2 Minimum Add to Make Parentheses Valid (LC 921)

```cpp
int minAddToMakeValid(string s) {
    int open = 0, close = 0;  // unmatched counts
    for (char c : s) {
        if (c == '(') open++;
        else {
            if (open > 0) open--;  // matched!
            else close++;          // unmatched ')'
        }
    }
    return open + close;  // total unmatched
}
// No stack needed! Just counting is enough.
```

### 5.3 Longest Valid Parentheses (LC 32) — HARD

```cpp
int longestValidParentheses(string s) {
    stack<int> st;
    st.push(-1);  // base for length calculation
    int ans = 0;
    
    for (int i = 0; i < s.length(); i++) {
        if (s[i] == '(') {
            st.push(i);
        } else {
            st.pop();
            if (st.empty()) {
                st.push(i);  // new base
            } else {
                ans = max(ans, i - st.top());
            }
        }
    }
    return ans;
}
// Key insight: Push INDEX, not char. Length = current_index - stack_top.
```

### 5.4 Remove Outermost Parentheses (LC 1021)

```cpp
string removeOuterParentheses(string s) {
    string result;
    int depth = 0;
    for (char c : s) {
        if (c == '(') {
            if (depth > 0) result += c;  // skip outermost '('
            depth++;
        } else {
            depth--;
            if (depth > 0) result += c;  // skip outermost ')'
        }
    }
    return result;
}
```

## 6. Pattern 2: Monotonic Stack — Next Greater/Smaller ⭐⭐

> **This is the most asked stack pattern in interviews.** Master this.

### 6.1 The Core Intuition

**Problem:** For each element, find the **Next Greater Element** (NGE) — the first element to its right that is larger.

```
Array:  [4, 5, 2, 10, 8]
NGE:    [5, 10, 10, -1, -1]

For 4: look right → 5 is greater → NGE = 5
For 5: look right → 2 (no), 10 (yes!) → NGE = 10
For 2: look right → 10 is greater → NGE = 10
For 10: nothing greater to right → NGE = -1
For 8: nothing greater to right → NGE = -1
```

**Brute force:** For each element, scan all elements to its right → O(n²).

**Stack intuition:** Think of elements "waiting" for their answer. Elements are pushed onto the stack. When a bigger element comes, it "resolves" all waiting elements that are smaller than it.

```
Processing right to left:
i=4: stack=[], no greater → ans[4]=-1, push 8.  stack=[8]
i=3: stack=[8], 8 < 10 → pop. stack=[], no greater → ans[3]=-1, push 10.  stack=[10]
i=2: stack=[10], 10 > 2 → ans[2]=10, push 2.  stack=[10, 2]
i=1: stack=[10, 2], 2 < 5 → pop. 10 > 5 → ans[1]=10, push 5.  stack=[10, 5]
i=0: stack=[10, 5], 5 > 4 → ans[0]=5, push 4.  stack=[10, 5, 4]
```

### 6.2 The Monotonic Stack Template — MEMORIZE THIS

```cpp
// Next Greater Element (right side)
vector<int> nextGreater(vector<int>& arr) {
    int n = arr.size();
    vector<int> ans(n, -1);
    stack<int> st;  // stores INDICES
    
    for (int i = n - 1; i >= 0; i--) {
        // Pop elements SMALLER than current (they can never be NGE for anyone)
        while (!st.empty() && arr[st.top()] <= arr[i]) {
            st.pop();
        }
        // If stack not empty, top is the NGE
        if (!st.empty()) ans[i] = arr[st.top()];
        
        // Push current index
        st.push(i);
    }
    return ans;
}
// Time: O(n) — each element pushed and popped at most once
// Space: O(n)
```

### 6.3 All 4 Variations

| Variation | Stack Type | Traverse Direction | Pop Condition |
|---|---|---|---|
| **Next Greater (Right)** | Decreasing | Right → Left | Pop if `arr[top] <= arr[i]` |
| **Next Smaller (Right)** | Increasing | Right → Left | Pop if `arr[top] >= arr[i]` |
| **Previous Greater (Left)** | Decreasing | Left → Right | Pop if `arr[top] <= arr[i]` |
| **Previous Smaller (Left)** | Increasing | Left → Right | Pop if `arr[top] >= arr[i]` |

```cpp
// Next Smaller Element (right side)
vector<int> nextSmaller(vector<int>& arr) {
    int n = arr.size();
    vector<int> ans(n, -1);
    stack<int> st;
    
    for (int i = n - 1; i >= 0; i--) {
        while (!st.empty() && arr[st.top()] >= arr[i]) st.pop();
        if (!st.empty()) ans[i] = arr[st.top()];
        st.push(i);
    }
    return ans;
}

// Previous Smaller Element (left side)
vector<int> prevSmaller(vector<int>& arr) {
    int n = arr.size();
    vector<int> ans(n, -1);
    stack<int> st;
    
    for (int i = 0; i < n; i++) {  // left to right!
        while (!st.empty() && arr[st.top()] >= arr[i]) st.pop();
        if (!st.empty()) ans[i] = arr[st.top()];
        st.push(i);
    }
    return ans;
}
```

### 6.4 Daily Temperatures (LC 739) ⭐

**Problem:** For each day, how many days until a warmer temperature?

**Intuition:** This is "Next Greater Element" but return the DISTANCE, not the value.

```cpp
vector<int> dailyTemperatures(vector<int>& temperatures) {
    int n = temperatures.size();
    vector<int> ans(n, 0);
    stack<int> st;  // indices of days waiting for warmer
    
    for (int i = 0; i < n; i++) {
        // Current temp resolves all colder days on stack
        while (!st.empty() && temperatures[st.top()] < temperatures[i]) {
            int idx = st.top(); st.pop();
            ans[idx] = i - idx;  // distance
        }
        st.push(i);
    }
    return ans;
}
// Left to right, popping resolved elements
```

### 6.5 Stock Span Problem (LC 901)

**Problem:** For each day, how many consecutive days (including today) the price was ≤ today's price?

**Intuition:** This is "Previous Greater Element" — find the nearest day with a HIGHER price.

```cpp
class StockSpanner {
    stack<pair<int, int>> st;  // (price, span)
public:
    int next(int price) {
        int span = 1;
        while (!st.empty() && st.top().first <= price) {
            span += st.top().second;
            st.pop();
        }
        st.push({price, span});
        return span;
    }
};
```

### 6.6 Largest Rectangle in Histogram (LC 84) — HARD ⭐⭐

**Problem:** Find the area of the largest rectangle that fits within the histogram bars.

**Intuition:** For each bar, the rectangle it can form extends left until hitting a shorter bar and right until hitting a shorter bar. So we need **Previous Smaller** and **Next Smaller** for each bar.

```cpp
int largestRectangleArea(vector<int>& heights) {
    int n = heights.size();
    stack<int> st;
    int maxArea = 0;
    
    for (int i = 0; i <= n; i++) {
        int h = (i == n) ? 0 : heights[i];
        while (!st.empty() && heights[st.top()] > h) {
            int height = heights[st.top()]; st.pop();
            int width = st.empty() ? i : (i - st.top() - 1);
            maxArea = max(maxArea, height * width);
        }
        st.push(i);
    }
    return maxArea;
}
// Time: O(n), Space: O(n)
// Key: When we pop a bar, it means we found its left and right boundaries.
//      Left boundary = new stack top. Right boundary = current index.
```

### 6.7 Maximal Rectangle in Binary Matrix (LC 85) — HARD

**Intuition:** Build histograms row by row. Apply "Largest Rectangle in Histogram" for each row.

```cpp
int maximalRectangle(vector<vector<char>>& matrix) {
    if (matrix.empty()) return 0;
    int n = matrix[0].size();
    vector<int> heights(n, 0);
    int maxArea = 0;
    
    for (auto& row : matrix) {
        for (int j = 0; j < n; j++) {
            heights[j] = (row[j] == '1') ? heights[j] + 1 : 0;
        }
        maxArea = max(maxArea, largestRectangleArea(heights));
    }
    return maxArea;
}
```

## 7. Pattern 3: Stack as History/Undo

### 7.1 Core Intuition

> When a problem says "go back", "undo", "remove last", "backspace", or "simplify" → Stack.
> The stack stores the **valid state so far**. We build it element by element, and sometimes pop to "undo" the last decision.

### 7.2 Simplify Path (LC 71)

```cpp
string simplifyPath(string path) {
    stack<string> st;
    stringstream ss(path);
    string token;
    
    while (getline(ss, token, '/')) {
        if (token == "" || token == ".") continue;
        if (token == "..") {
            if (!st.empty()) st.pop();  // go up one directory
        } else {
            st.push(token);
        }
    }
    
    string result;
    while (!st.empty()) {
        result = "/" + st.top() + result;
        st.pop();
    }
    return result.empty() ? "/" : result;
}
```

### 7.3 Remove All Adjacent Duplicates (LC 1047)

```cpp
string removeDuplicates(string s) {
    string st;  // use string as stack (more efficient)
    for (char c : s) {
        if (!st.empty() && st.back() == c) {
            st.pop_back();  // undo!
        } else {
            st.push_back(c);
        }
    }
    return st;
}
// "abbaca" → "ca"
```

### 7.4 Removing Stars From a String (LC 2390)

```cpp
string removeStars(string s) {
    string st;
    for (char c : s) {
        if (c == '*') st.pop_back();  // star removes previous char
        else st.push_back(c);
    }
    return st;
}
```

### 7.5 Asteroid Collision (LC 735) ⭐

**Intuition:** Asteroids move right (+) or left (-). Collision happens when a right-mover meets a left-mover. Stack holds surviving asteroids.

```cpp
vector<int> asteroidCollision(vector<int>& asteroids) {
    vector<int> st;
    for (int a : asteroids) {
        bool alive = true;
        while (alive && !st.empty() && a < 0 && st.back() > 0) {
            // collision: right-moving (top) vs left-moving (a)
            if (st.back() < -a) {
                st.pop_back();      // top destroyed
            } else if (st.back() == -a) {
                st.pop_back();      // both destroyed
                alive = false;
            } else {
                alive = false;      // a destroyed
            }
        }
        if (alive) st.push_back(a);
    }
    return st;
}
```

### 7.6 Min Stack — O(1) getMin (LC 155) ⭐

```cpp
class MinStack {
    stack<long long> st;
    long long minVal;
public:
    void push(int x) {
        if (st.empty()) {
            st.push(0);
            minVal = x;
        } else {
            st.push((long long)x - minVal);  // store difference
            if (x < minVal) minVal = x;
        }
    }
    
    void pop() {
        long long top = st.top(); st.pop();
        if (top < 0) minVal -= top;  // restore previous min
    }
    
    int top() {
        long long topVal = st.top();
        return (topVal < 0) ? minVal : minVal + topVal;
    }
    
    int getMin() { return minVal; }
};
// All operations O(1) time, O(1) extra space per element

// Alternative: store (value, currentMin) pairs
class MinStack2 {
    stack<pair<int,int>> st;  // (value, min_at_this_point)
public:
    void push(int x) {
        int curMin = st.empty() ? x : min(x, st.top().second);
        st.push({x, curMin});
    }
    void pop() { st.pop(); }
    int top() { return st.top().first; }
    int getMin() { return st.top().second; }
};
```

## 8. Pattern 4: Stack for Expression Evaluation

### 8.1 When to Use

> Any problem involving **mathematical expressions**, **operator precedence**, or **converting between infix/postfix/prefix** → Use a stack.

### 8.2 Basic Calculator II (LC 227) ⭐

**Problem:** Evaluate `"3+2*2"` → 7. Supports `+`, `-`, `*`, `/`. No parentheses.

**Intuition:** `*` and `/` have higher precedence. Process them immediately. `+` and `-` can wait.

```cpp
int calculate(string s) {
    stack<int> st;
    int num = 0;
    char op = '+';  // default operator before first number
    
    for (int i = 0; i < s.length(); i++) {
        char c = s[i];
        if (isdigit(c)) {
            num = num * 10 + (c - '0');
        }
        if ((!isdigit(c) && c != ' ') || i == s.length() - 1) {
            if (op == '+') st.push(num);
            else if (op == '-') st.push(-num);
            else if (op == '*') { int top = st.top(); st.pop(); st.push(top * num); }
            else if (op == '/') { int top = st.top(); st.pop(); st.push(top / num); }
            op = c;
            num = 0;
        }
    }
    
    int result = 0;
    while (!st.empty()) { result += st.top(); st.pop(); }
    return result;
}
// Time: O(n), Space: O(n)
// Key: Push numbers. For * and /, pop-compute-push immediately.
```

### 8.3 Basic Calculator (LC 224) — HARD

**Problem:** Evaluate `"(1+(4+5+2)-3)+(6+8)"`. Supports `+`, `-`, `(`, `)`.

```cpp
int calculate(string s) {
    stack<int> st;
    int result = 0, num = 0, sign = 1;
    
    for (char c : s) {
        if (isdigit(c)) {
            num = num * 10 + (c - '0');
        } else if (c == '+') {
            result += sign * num;
            num = 0;
            sign = 1;
        } else if (c == '-') {
            result += sign * num;
            num = 0;
            sign = -1;
        } else if (c == '(') {
            st.push(result);   // save current result
            st.push(sign);     // save current sign
            result = 0;
            sign = 1;
        } else if (c == ')') {
            result += sign * num;
            num = 0;
            result *= st.top(); st.pop();  // apply saved sign
            result += st.top(); st.pop();  // add saved result
        }
    }
    return result + sign * num;
}
```

### 8.4 Evaluate Reverse Polish Notation — Postfix (LC 150)

```cpp
int evalRPN(vector<string>& tokens) {
    stack<int> st;
    for (string& t : tokens) {
        if (t == "+" || t == "-" || t == "*" || t == "/") {
            int b = st.top(); st.pop();  // second operand
            int a = st.top(); st.pop();  // first operand
            if (t == "+") st.push(a + b);
            else if (t == "-") st.push(a - b);
            else if (t == "*") st.push(a * b);
            else st.push(a / b);
        } else {
            st.push(stoi(t));
        }
    }
    return st.top();
}
// ["2","1","+","3","*"] → (2+1)*3 = 9
```

### 8.5 Infix → Postfix Conversion (Shunting Yard)

```cpp
string infixToPostfix(string s) {
    stack<char> opStack;
    string result;
    auto precedence = [](char c) {
        if (c == '+' || c == '-') return 1;
        if (c == '*' || c == '/') return 2;
        return 0;
    };
    
    for (char c : s) {
        if (isalnum(c)) {
            result += c;
        } else if (c == '(') {
            opStack.push(c);
        } else if (c == ')') {
            while (opStack.top() != '(') {
                result += opStack.top();
                opStack.pop();
            }
            opStack.pop();  // remove '('
        } else {  // operator
            while (!opStack.empty() && precedence(opStack.top()) >= precedence(c)) {
                result += opStack.top();
                opStack.pop();
            }
            opStack.push(c);
        }
    }
    while (!opStack.empty()) {
        result += opStack.top();
        opStack.pop();
    }
    return result;
}
// "a+b*c" → "abc*+"
```

## 9. Pattern 5: Stack ↔ Queue Conversions

### 9.1 Implement Queue using Stacks (LC 232)

**Intuition:** Reverse the order with two stacks. One for input, one for output.

```cpp
class MyQueue {
    stack<int> inStack, outStack;
    
    void transfer() {
        while (!inStack.empty()) {
            outStack.push(inStack.top());
            inStack.pop();
        }
    }
    
public:
    void push(int x) { inStack.push(x); }
    
    int pop() {
        if (outStack.empty()) transfer();
        int val = outStack.top();
        outStack.pop();
        return val;
    }
    
    int peek() {
        if (outStack.empty()) transfer();
        return outStack.top();
    }
    
    bool empty() { return inStack.empty() && outStack.empty(); }
};
// Amortized O(1) per operation!
// Each element is moved at most once from inStack to outStack.
```

### 9.2 Implement Stack using Queues (LC 225)

```cpp
class MyStack {
    queue<int> q;
public:
    void push(int x) {
        q.push(x);
        // Rotate so the new element is at front
        for (int i = 0; i < q.size() - 1; i++) {
            q.push(q.front());
            q.pop();
        }
    }
    
    int pop() {
        int val = q.front();
        q.pop();
        return val;
    }
    
    int top() { return q.front(); }
    bool empty() { return q.empty(); }
};
// push: O(n), pop: O(1)
```

## 10. Pattern 6: Monotonic Deque (Sliding Window Max/Min)

### 10.1 When to Use

> **"Find the maximum (or minimum) in every window of size K"** → Monotonic Deque.

This was covered in the Sliding Window chapter (LC 239), but here's the deeper intuition:

### 10.2 Why a Deque and Not a Stack?

```
Stack:  can only push/pop from ONE end   → can't remove expired elements from bottom
Deque:  push/pop from BOTH ends          → remove expired from front, maintain order from back
```

The deque maintains a **decreasing** order (for max) or **increasing** order (for min):
- **Back:** Pop smaller elements (they'll never be the answer while the current, larger element exists).
- **Front:** Pop expired elements (indices outside the window).

### 10.3 Sliding Window Maximum (LC 239) — Recap

```cpp
vector<int> maxSlidingWindow(vector<int>& nums, int k) {
    deque<int> dq;  // stores indices, front = max
    vector<int> ans;
    
    for (int i = 0; i < nums.size(); i++) {
        // Remove expired elements from front
        while (!dq.empty() && dq.front() < i - k + 1) dq.pop_front();
        
        // Remove smaller elements from back (they're useless now)
        while (!dq.empty() && nums[dq.back()] < nums[i]) dq.pop_back();
        
        dq.push_back(i);
        
        if (i >= k - 1) ans.push_back(nums[dq.front()]);
    }
    return ans;
}
```

### 10.4 Sliding Window Minimum

```cpp
// Just flip the comparison: remove LARGER elements from back
vector<int> minSlidingWindow(vector<int>& nums, int k) {
    deque<int> dq;
    vector<int> ans;
    
    for (int i = 0; i < nums.size(); i++) {
        while (!dq.empty() && dq.front() < i - k + 1) dq.pop_front();
        while (!dq.empty() && nums[dq.back()] > nums[i]) dq.pop_back();
        dq.push_back(i);
        if (i >= k - 1) ans.push_back(nums[dq.front()]);
    }
    return ans;
}
```

## 11. Queue-Based Problems (BFS & Level Order)

### 11.1 When to Use a Queue

> **"Process things in order of arrival"** or **"Level by level"** or **"Shortest path in unweighted graph"** → Queue (BFS).

### 11.2 Rotten Oranges (LC 994) — Multi-Source BFS ⭐

**Intuition:** All rotten oranges spread simultaneously (like ripples). Start BFS from ALL rotten oranges at once.

```cpp
int orangesRotting(vector<vector<int>>& grid) {
    int m = grid.size(), n = grid[0].size();
    queue<pair<int,int>> q;
    int fresh = 0;
    
    // Add all rotten oranges to queue
    for (int i = 0; i < m; i++)
        for (int j = 0; j < n; j++) {
            if (grid[i][j] == 2) q.push({i, j});
            else if (grid[i][j] == 1) fresh++;
        }
    
    if (fresh == 0) return 0;
    
    int minutes = 0;
    int dx[] = {0,0,1,-1}, dy[] = {1,-1,0,0};
    
    while (!q.empty()) {
        int size = q.size();
        bool rotted = false;
        for (int k = 0; k < size; k++) {
            auto [x, y] = q.front(); q.pop();
            for (int d = 0; d < 4; d++) {
                int nx = x + dx[d], ny = y + dy[d];
                if (nx >= 0 && nx < m && ny >= 0 && ny < n && grid[nx][ny] == 1) {
                    grid[nx][ny] = 2;
                    q.push({nx, ny});
                    fresh--;
                    rotted = true;
                }
            }
        }
        if (rotted) minutes++;
    }
    return fresh == 0 ? minutes : -1;
}
```

### 11.3 Design Hit Counter (LC 362)

```cpp
class HitCounter {
    queue<int> hits;
public:
    void hit(int timestamp) { hits.push(timestamp); }
    
    int getHits(int timestamp) {
        while (!hits.empty() && hits.front() <= timestamp - 300) {
            hits.pop();  // remove hits older than 5 minutes
        }
        return hits.size();
    }
};
// Sliding window using a queue!
```

### 11.4 Number of Recent Calls (LC 933)

```cpp
class RecentCounter {
    queue<int> q;
public:
    int ping(int t) {
        q.push(t);
        while (q.front() < t - 3000) q.pop();
        return q.size();
    }
};
```

## 13. Common Mistakes & Debugging

### 13.1 Top 12 Stack/Queue Mistakes

| # | Mistake | Fix |
|---|---|---|
| 1 | **Calling `top()` or `pop()` on empty stack** | Always check `st.empty()` first! |
| 2 | **C++ `pop()` returns void** | Use `st.top()` to get value, THEN `st.pop()` |
| 3 | **Forgetting what to store: value vs index** | Index for distance/span. Value for matching/evaluation. |
| 4 | **Wrong monotonic direction** | Next Greater → decreasing stack. Next Smaller → increasing stack. |
| 5 | **Not processing remaining stack elements** | After the loop, elements left in stack have no answer (= -1 or need special handling) |
| 6 | **Queue BFS: not processing level by level** | Use `int size = q.size()` at start of each level |
| 7 | **Circular queue: wrong modular arithmetic** | `rear = (rear + 1) % capacity`, NOT `rear++` |
| 8 | **Priority queue: wrong comparator direction** | `greater<int>` = min-heap (confusing!). `less<int>` = max-heap. |
| 9 | **Not handling edge cases** | Empty input, single element, all same elements |
| 10 | **Using stack for BFS or queue for DFS** | Stack = DFS. Queue = BFS. Don't mix them up! |
| 11 | **Monotonic stack: strict vs non-strict** | `<=` vs `<` in pop condition depends on duplicates |
| 12 | **Histogram problem: forgetting sentinel** | Push index `n` with height 0 to flush remaining stack |

### 13.2 Debugging Checklist

```
□ Am I checking empty() before top()/pop()/front()?
□ Am I storing the right thing (value vs index)?
□ Is my monotonic stack direction correct? (decreasing for NGE, increasing for NSE)
□ After the main loop, did I handle remaining stack elements?
□ For BFS: am I processing each level with a size loop?
□ For expression: am I handling operator precedence correctly?
□ Edge: what happens with empty input? single element?
```

## 14. Interview FAQs

#### Q1: How do I decide between Stack, Queue, and Deque?

**A:**
- **Stack** → most RECENT element matters (LIFO): matching, monotonic, undo, DFS, expressions.
- **Queue** → ORDER OF ARRIVAL matters (FIFO): BFS, level-order, scheduling.
- **Deque** → need to add/remove from BOTH ends: sliding window max/min, palindrome checks.

#### Q2: What is a Monotonic Stack and when do I use it?

**A:** A monotonic stack maintains elements in strictly increasing or decreasing order. Use it when you need to find the "next greater/smaller" or "previous greater/smaller" element for every element in an array. Classic problems: daily temperatures, stock span, histogram, trapping rain water.

#### Q3: Why is the Monotonic Stack O(n) even with a nested loop?

**A:** Each element is pushed onto the stack exactly once and popped at most once. So total push + pop operations = 2n = O(n). The inner while loop doesn't multiply — it shares work across all iterations.

#### Q4: How does "Queue using Two Stacks" achieve amortized O(1)?

**A:** Each element is moved from `inStack` to `outStack` exactly once. Over n operations, total transfers = n. So amortized cost per operation = n/n = O(1). Worst-case single operation can be O(n), but averaged out it's O(1).

#### Q5: What's the difference between Monotonic Stack and Monotonic Deque?

**A:** 
- **Monotonic Stack** can only add/remove from one end (top). Used for NGE/NSE where you process elements once.
- **Monotonic Deque** can add/remove from both ends. Used for **sliding window** problems where elements expire from the front (out of window range) AND smaller/larger elements are removed from the back.

#### Q6: How do I evaluate expressions with nested parentheses?

**A:** Use a stack to save the current state when you see `(`. When you see `)`, pop the saved state and combine with the result inside the parentheses. Alternatively, convert to postfix (no parentheses needed) using the Shunting Yard algorithm, then evaluate postfix with a single stack.

#### Q7: When should I store indices vs values in the stack?

**A:** Store **indices** when you need to compute distances, time differences, or spans (e.g., Daily Temperatures, Histogram). Store **values** when you need the actual elements for computation (e.g., expression evaluation, bracket matching). Sometimes store **both** as pairs.

#### Q8: What is the "Next Greater Element in Circular Array" trick?

**A:** Iterate through the array twice (`2*n` iterations) using `i % n` to simulate the circular nature. Push indices, and the monotonic stack naturally handles the wrap-around.

```cpp
vector<int> nextGreaterCircular(vector<int>& nums) {
    int n = nums.size();
    vector<int> ans(n, -1);
    stack<int> st;
    for (int i = 2 * n - 1; i >= 0; i--) {
        while (!st.empty() && nums[st.top()] <= nums[i % n]) st.pop();
        if (i < n && !st.empty()) ans[i] = nums[st.top()];
        st.push(i % n);
    }
    return ans;
}
```

## 15. Real-World Scenarios

### 🔍 Scenario 1: Browser Back/Forward (Stack)

**Problem:** _"Implement the back and forward buttons."_

**Answer:** Two stacks: `backStack` and `forwardStack`. Navigate to new page → push current to `backStack`, clear `forwardStack`. Back → push current to `forwardStack`, pop from `backStack`. Forward → push current to `backStack`, pop from `forwardStack`.

Used in: Every web browser.

---

### 🔍 Scenario 2: Undo/Redo in Text Editors (Stack)

**Problem:** _"Implement Ctrl+Z (undo) and Ctrl+Y (redo)."_

**Answer:** Two stacks: `undoStack` stores actions. Undo → pop from `undoStack`, push to `redoStack`. New action → push to `undoStack`, clear `redoStack`.

Used in: VS Code, Google Docs, Photoshop, every editor.

---

### 🔍 Scenario 3: Function Call Stack (Stack)

**Problem:** _"How does recursion work?"_

**Answer:** Every function call pushes a **stack frame** (local variables, return address) onto the **call stack**. When a function returns, its frame is popped. Stack overflow = too many frames (deep recursion).

Used in: Every programming language runtime.

---

### 🔍 Scenario 4: Syntax Checking in Compilers (Stack)

**Problem:** _"Check if `{{[()]}}` has balanced brackets."_

**Answer:** Push opening brackets, pop on closing. If mismatch or stack not empty at end → syntax error. Compilers do this for `{}`, `()`, `<>` in code.

Used in: Every compiler, linter, IDE syntax highlighting.

---

### 🔍 Scenario 5: Print Queue / Task Scheduling (Queue)

**Problem:** _"Multiple people send documents to a shared printer."_

**Answer:** FIFO Queue. Documents are printed in the order they were submitted. Priority queue variant: urgent documents jump ahead.

Used in: OS process scheduler, print spooler, message queues (RabbitMQ, Kafka).

---

### 🔍 Scenario 6: BFS — Shortest Path in Maps (Queue)

**Problem:** _"Find the shortest path between two locations on a grid."_

**Answer:** BFS using a Queue explores all paths of length 1, then length 2, etc. First time you reach the destination = shortest path.

Used in: Google Maps, GPS navigation, maze solving, network routing.

---

### 🔍 Scenario 7: Customer Service Hotline (Queue)

**Problem:** _"Calls are answered in the order received."_

**Answer:** FIFO Queue. Callers wait in queue. When an agent is free, the longest-waiting caller is served. Priority queue: VIP customers served first.

Used in: Call centers, support ticket systems, hospital triage.

---

### 🔍 Scenario 8: Stock Trading — Real-Time Analytics (Monotonic Stack)

**Problem:** _"For each day, how many consecutive days was the price less than or equal to today?"_

**Answer:** Stock Span Problem. Monotonic decreasing stack. Each day, pop all days with lower prices and accumulate their spans.

Used in: Trading platforms, financial dashboards, technical analysis.

---

### 🔍 Scenario 9: Autocomplete — Recent Searches (Stack/Deque)

**Problem:** _"Show the 5 most recent searches."_

**Answer:** Use a deque (bounded size). New search → push to front. If size > 5, pop from back. Display front to back.

Used in: Google search, browser address bar, IDE command palette.

---

### 🔍 Scenario 10: Message Queues — Microservices (Queue)

**Problem:** _"Service A sends events that Service B must process in order."_

**Answer:** Message Queue (FIFO). Producer (Service A) enqueues messages. Consumer (Service B) dequeues and processes them. Guarantees ordering and decouples services.

```
Service A → [msg1, msg2, msg3] → Service B
              (Queue)
```

Used in: RabbitMQ, Apache Kafka, AWS SQS, Redis Streams.
