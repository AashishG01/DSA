## 1. What is a Heap?

### 1.1 The Definition

A **Heap** is a **complete binary tree** that satisfies the **heap property**:
- **Max-Heap:** Every parent ≥ its children. Root = maximum.
- **Min-Heap:** Every parent ≤ its children. Root = minimum.

```
MAX-HEAP:                 MIN-HEAP:
        50                        1
       /  \                      / \
     30    40                   3    2
    /  \  /                    / \  /
   10  20 35                  5   4 6

Root = max (50)             Root = min (1)
Parent ≥ Children           Parent ≤ Children
```

### 1.2 Key Properties

| Property | Detail |
|---|---|
| **Shape** | Complete binary tree (filled level by level, left to right) |
| **Heap Property** | Parent vs children ordering (max or min) |
| **Height** | O(log n) — because complete binary tree |
| **Root** | Always the max (max-heap) or min (min-heap) |
| **NOT sorted** | Left child is NOT necessarily < right child |
| **Storage** | Array-based (no pointers needed!) |

### 1.3 Why Heaps Matter

> **The fundamental question a heap answers:**
> "What is the current maximum/minimum element, and can I efficiently add and remove elements?"

| Operation | Sorted Array | Unsorted Array | **Heap** |
|---|---|---|---|
| Find Max/Min | O(1) | O(n) | **O(1)** ✅ |
| Insert | O(n) | O(1) | **O(log n)** ✅ |
| Delete Max/Min | O(1) | O(n) | **O(log n)** ✅ |
| Build from array | O(n log n) | O(1) | **O(n)** ✅ |

### 1.4 Heap vs BST vs Sorted Array

| Feature | Heap | BST | Sorted Array |
|---|---|---|---|
| Find Min/Max | O(1) ✅ | O(log n) | O(1) |
| Find Arbitrary | O(n) | O(log n) ✅ | O(log n) ✅ |
| Insert | O(log n) | O(log n) | O(n) |
| Delete Min/Max | O(log n) ✅ | O(log n) | O(1) / O(n) |
| Delete Arbitrary | O(n) | O(log n) ✅ | O(n) |
| **Use when** | Need min/max repeatedly | Need all operations balanced | Data is static |

## 2. Heap Implementation from Scratch

### 2.1 Array Representation

A heap is stored as an array. For index `i` (0-indexed):

```
Parent:      (i - 1) / 2
Left Child:  2 * i + 1
Right Child: 2 * i + 2

Array:  [50, 30, 40, 10, 20, 35]
Tree:
        50(0)
       /     \
    30(1)    40(2)
   /    \    /
 10(3) 20(4) 35(5)
```

### 2.2 Core Operations — Min-Heap

```cpp
class MinHeap {
    vector<int> heap;
    
    void heapifyUp(int i) {
        while (i > 0) {
            int parent = (i - 1) / 2;
            if (heap[i] < heap[parent]) {
                swap(heap[i], heap[parent]);
                i = parent;
            } else break;
        }
    }
    
    void heapifyDown(int i) {
        int n = heap.size();
        while (true) {
            int smallest = i;
            int left = 2 * i + 1;
            int right = 2 * i + 2;
            
            if (left < n && heap[left] < heap[smallest]) smallest = left;
            if (right < n && heap[right] < heap[smallest]) smallest = right;
            
            if (smallest != i) {
                swap(heap[i], heap[smallest]);
                i = smallest;
            } else break;
        }
    }
    
public:
    // INSERT — O(log n)
    void push(int val) {
        heap.push_back(val);
        heapifyUp(heap.size() - 1);
    }
    
    // GET MIN — O(1)
    int top() {
        return heap[0];
    }
    
    // REMOVE MIN — O(log n)
    void pop() {
        heap[0] = heap.back();
        heap.pop_back();
        if (!heap.empty()) heapifyDown(0);
    }
    
    int size() { return heap.size(); }
    bool empty() { return heap.empty(); }
};
```

### 2.3 How Insert Works (Heapify Up)

```
Insert 5 into Min-Heap [1, 3, 2, 7, 4, 6]:

Step 1: Add at end → [1, 3, 2, 7, 4, 6, 5]
Step 2: Compare with parent: 5 < parent(2)? No → DONE

Insert 0 into [1, 3, 2, 7, 4, 6]:

Step 1: Add at end → [1, 3, 2, 7, 4, 6, 0]
Step 2: Compare with parent: 0 < 2? Yes → swap → [1, 3, 0, 7, 4, 6, 2]
Step 3: Compare with parent: 0 < 1? Yes → swap → [0, 3, 1, 7, 4, 6, 2]
Step 4: At root → DONE
```

### 2.4 How Delete Works (Heapify Down)

```
Remove min from [1, 3, 2, 7, 4, 6]:

Step 1: Replace root with last → [6, 3, 2, 7, 4]
Step 2: Heapify down:
        6 → children are 3, 2. Smallest child = 2 → swap → [2, 3, 6, 7, 4]
        6 → children don't exist → DONE

Result: [2, 3, 6, 7, 4]
```

### 2.5 Build Heap from Array — O(n) ✅

```cpp
void buildHeap(vector<int>& arr) {
    int n = arr.size();
    // Start from last internal node and heapify down
    for (int i = n / 2 - 1; i >= 0; i--) {
        heapifyDown(arr, i, n);
    }
}
// Why O(n) not O(n log n)?
// Most nodes are near leaves (height 0-1).
// Total work = Σ(nodes at height h × h) = O(n)
```

## 3. C++ Priority Queue — STL Reference

### 3.1 Basic Usage

```cpp
#include <queue>

// MAX-HEAP (default)
priority_queue<int> maxPQ;
maxPQ.push(3); maxPQ.push(1); maxPQ.push(5);
maxPQ.top();   // 5 (largest)
maxPQ.pop();   // removes 5

// MIN-HEAP
priority_queue<int, vector<int>, greater<int>> minPQ;
minPQ.push(3); minPQ.push(1); minPQ.push(5);
minPQ.top();   // 1 (smallest)
minPQ.pop();   // removes 1
```

### 3.2 Custom Types in Priority Queue

```cpp
// For pairs: sorts by first element (then second)
priority_queue<pair<int,int>> maxPQ;             // max by first
priority_queue<pair<int,int>, vector<pair<int,int>>, greater<pair<int,int>>> minPQ; // min by first

// Custom comparator
auto cmp = [](const pair<int,int>& a, const pair<int,int>& b) {
    return a.second > b.second;  // min-heap by second element
};
priority_queue<pair<int,int>, vector<pair<int,int>>, decltype(cmp)> pq(cmp);

// Struct with comparator
struct Task {
    int priority, id;
};
auto taskCmp = [](const Task& a, const Task& b) {
    return a.priority < b.priority;  // max-heap by priority
};
priority_queue<Task, vector<Task>, decltype(taskCmp)> taskPQ(taskCmp);
```

### 3.3 STL Operations Summary

| Operation | Code | Time |
|---|---|---|
| Insert | `pq.push(x)` | O(log n) |
| Get top | `pq.top()` | O(1) |
| Remove top | `pq.pop()` | O(log n) |
| Size | `pq.size()` | O(1) |
| Empty check | `pq.empty()` | O(1) |
| Build from vector | `priority_queue<int> pq(v.begin(), v.end())` | O(n) |

> ⚠️ C++ `priority_queue` has **NO** `erase()`, `find()`, or `decrease_key()`. For those, use `set<>` or `multiset<>` instead.

## 4. 🧠 How to Think About Heap Problems — Intuition

### 4.1 The ONE Question to Ask

> **"Do I need to repeatedly find the minimum or maximum from a changing collection?"**
>
> If YES → Use a **Heap / Priority Queue**.

### 4.2 The 6 Heap Patterns — Decision Framework

```
┌──────────────────────────────────────────────────────────────────┐
│                   WHEN TO USE A HEAP?                            │
├──────────────────────────────────────────────────────────────────┤
│                                                                  │
│  1. TOP K ELEMENTS                                               │
│     → "Find K largest/smallest/most frequent"                    │
│     → Use OPPOSITE heap: K largest → min-heap of size K          │
│     → O(n log k)                                                 │
│                                                                  │
│  2. K-WAY MERGE                                                  │
│     → "Merge K sorted lists/arrays"                              │
│     → Min-heap of size K (one element from each list)            │
│     → O(n log k)                                                 │
│                                                                  │
│  3. TWO HEAPS (Median)                                           │
│     → "Find running median" or "balance two halves"              │
│     → Max-heap (left half) + Min-heap (right half)               │
│     → O(log n) per insert, O(1) median                           │
│                                                                  │
│  4. STREAMING / LAZY DELETION                                    │
│     → Process elements one by one, maintain best K               │
│     → Keep heap, mark deleted lazily                              │
│                                                                  │
│  5. HEAP + GREEDY                                                │
│     → "Always pick the best/smallest/largest next option"        │
│     → Greedy choice + heap to maintain candidates                │
│     → E.g., meeting rooms, task scheduler, reorganize string     │
│                                                                  │
│  6. K CLOSEST / DISTANCE-BASED                                   │
│     → "Find K closest points/elements"                           │
│     → Max-heap of size K (evict farthest when full)              │
│                                                                  │
└──────────────────────────────────────────────────────────────────┘
```

### 4.3 The "Opposite Heap" Trick — Most Important Insight

> **To find K LARGEST → use a MIN-heap of size K.**
> **To find K SMALLEST → use a MAX-heap of size K.**

Why? The heap of size K acts as a "gatekeeper":
- Min-heap: the top is the SMALLEST of the K largest. Anything smaller gets rejected.
- This is O(n log K) instead of O(n log n).

```
Find 3 largest from [4, 1, 7, 3, 8, 5]:

Min-heap (capacity 3):
  Push 4: [4]
  Push 1: [1, 4]
  Push 7: [1, 4, 7]        ← full
  Push 3: 3 > top(1)? Yes → pop 1, push 3: [3, 4, 7]
  Push 8: 8 > top(3)? Yes → pop 3, push 8: [4, 7, 8]
  Push 5: 5 > top(4)? Yes → pop 4, push 5: [5, 7, 8]

Result: {5, 7, 8} — the 3 largest! Top (5) = Kth largest.
```

### 4.4 Heap vs Sort vs QuickSelect — When to Use What

| Approach | Time | Use When |
|---|---|---|
| **Sort** | O(n log n) | Need ALL elements sorted, or K ≈ n |
| **Heap (size K)** | O(n log K) | K << n, need top K, streaming data |
| **QuickSelect** | O(n) avg | Need ONLY the Kth element (not all K) |
| **Counting Sort** | O(n + range) | Small value range (e.g., frequencies) |

## 5. Pattern 1: Top K Elements

### 5.1 Kth Largest Element in Array (LC 215) ⭐

```cpp
// Method 1: Min-Heap of size K — O(n log k)
int findKthLargest(vector<int>& nums, int k) {
    priority_queue<int, vector<int>, greater<int>> minPQ;  // min-heap
    
    for (int num : nums) {
        minPQ.push(num);
        if (minPQ.size() > k) minPQ.pop();  // evict smallest
    }
    return minPQ.top();  // Kth largest = smallest of K largest
}

// Method 2: QuickSelect — O(n) average (for reference)
int quickSelect(vector<int>& nums, int l, int r, int k) {
    int pivot = nums[r], p = l;
    for (int i = l; i < r; i++)
        if (nums[i] <= pivot) swap(nums[i], nums[p++]);
    swap(nums[p], nums[r]);
    
    if (p == k) return nums[p];
    if (p < k) return quickSelect(nums, p + 1, r, k);
    return quickSelect(nums, l, p - 1, k);
}
int findKthLargest2(vector<int>& nums, int k) {
    return quickSelect(nums, 0, nums.size() - 1, nums.size() - k);
}
```

### 5.2 Top K Frequent Elements (LC 347) ⭐

```cpp
vector<int> topKFrequent(vector<int>& nums, int k) {
    unordered_map<int, int> freq;
    for (int n : nums) freq[n]++;
    
    // Min-heap by frequency
    auto cmp = [](pair<int,int>& a, pair<int,int>& b) {
        return a.second > b.second;  // min-heap by freq
    };
    priority_queue<pair<int,int>, vector<pair<int,int>>, decltype(cmp)> pq(cmp);
    
    for (auto& [num, f] : freq) {
        pq.push({num, f});
        if (pq.size() > k) pq.pop();
    }
    
    vector<int> result;
    while (!pq.empty()) {
        result.push_back(pq.top().first);
        pq.pop();
    }
    return result;
}
// Time: O(n log k), Space: O(n)
```

### 5.3 K Closest Points to Origin (LC 973) ⭐

```cpp
vector<vector<int>> kClosest(vector<vector<int>>& points, int k) {
    // Max-heap: evict FARTHEST when size > k
    auto cmp = [](vector<int>& a, vector<int>& b) {
        return a[0]*a[0] + a[1]*a[1] < b[0]*b[0] + b[1]*b[1];  // max-heap by distance
    };
    priority_queue<vector<int>, vector<vector<int>>, decltype(cmp)> pq(cmp);
    
    for (auto& p : points) {
        pq.push(p);
        if (pq.size() > k) pq.pop();
    }
    
    vector<vector<int>> result;
    while (!pq.empty()) {
        result.push_back(pq.top());
        pq.pop();
    }
    return result;
}
// K closest → max-heap of size K → evict farthest
```

### 5.4 Sort Characters by Frequency (LC 451)

```cpp
string frequencySort(string s) {
    unordered_map<char, int> freq;
    for (char c : s) freq[c]++;
    
    priority_queue<pair<int,char>> pq;  // max-heap by freq
    for (auto& [c, f] : freq) pq.push({f, c});
    
    string result;
    while (!pq.empty()) {
        auto [f, c] = pq.top(); pq.pop();
        result += string(f, c);
    }
    return result;
}
```

## 6. Pattern 2: K-Way Merge (Merge K Sorted)

### 6.1 Core Intuition

When you have **K sorted** lists/arrays and need to merge them:
- Put the first element of each list into a min-heap.
- Pop the smallest, output it, and push the next element from that list.
- Repeat until all elements are processed.

```
List 1: [1, 4, 7]
List 2: [2, 5, 8]
List 3: [3, 6, 9]

Min-Heap: {1, 2, 3}  → pop 1 (from list 1), push 4 → {2, 3, 4}
          {2, 3, 4}  → pop 2 (from list 2), push 5 → {3, 4, 5}
          {3, 4, 5}  → pop 3 (from list 3), push 6 → {4, 5, 6}
          ...

Result: [1, 2, 3, 4, 5, 6, 7, 8, 9]
```

### 6.2 Merge K Sorted Lists (LC 23) ⭐⭐

```cpp
struct CompareNode {
    bool operator()(ListNode* a, ListNode* b) {
        return a->val > b->val;  // min-heap
    }
};

ListNode* mergeKLists(vector<ListNode*>& lists) {
    priority_queue<ListNode*, vector<ListNode*>, CompareNode> pq;
    
    // Push head of each list
    for (auto* list : lists)
        if (list) pq.push(list);
    
    ListNode dummy(0);
    ListNode* tail = &dummy;
    
    while (!pq.empty()) {
        ListNode* node = pq.top(); pq.pop();
        tail->next = node;
        tail = tail->next;
        if (node->next) pq.push(node->next);
    }
    return dummy.next;
}
// Time: O(n log k) where n = total elements, k = number of lists
// Space: O(k)
```

### 6.3 Kth Smallest Element in Sorted Matrix (LC 378)

```cpp
int kthSmallest(vector<vector<int>>& matrix, int k) {
    int n = matrix.size();
    // (value, row, col) — min-heap
    priority_queue<tuple<int,int,int>, vector<tuple<int,int,int>>, greater<>> pq;
    
    // Push first element of each row
    for (int i = 0; i < min(n, k); i++)
        pq.push({matrix[i][0], i, 0});
    
    int result = 0;
    for (int i = 0; i < k; i++) {
        auto [val, row, col] = pq.top(); pq.pop();
        result = val;
        if (col + 1 < n)
            pq.push({matrix[row][col+1], row, col+1});
    }
    return result;
}
// K-way merge: each row is a sorted list
```

### 6.4 Smallest Range Covering Elements from K Lists (LC 632) — HARD

```cpp
vector<int> smallestRange(vector<vector<int>>& nums) {
    // Min-heap: (value, list_index, element_index)
    priority_queue<tuple<int,int,int>, vector<tuple<int,int,int>>, greater<>> pq;
    int curMax = INT_MIN;
    
    for (int i = 0; i < nums.size(); i++) {
        pq.push({nums[i][0], i, 0});
        curMax = max(curMax, nums[i][0]);
    }
    
    vector<int> ans = {get<0>(pq.top()), curMax};
    
    while (true) {
        auto [val, li, ei] = pq.top(); pq.pop();
        if (ei + 1 >= nums[li].size()) break;
        
        pq.push({nums[li][ei+1], li, ei+1});
        curMax = max(curMax, nums[li][ei+1]);
        
        if (curMax - get<0>(pq.top()) < ans[1] - ans[0]) {
            ans = {get<0>(pq.top()), curMax};
        }
    }
    return ans;
}
```

## 7. Pattern 3: Two Heaps (Median Tracking) ⭐⭐

### 7.1 Core Intuition

To find the **median** of a stream, split elements into two halves:
- **Max-heap (left half):** Stores the smaller half. Top = largest of the small.
- **Min-heap (right half):** Stores the larger half. Top = smallest of the large.

```
Numbers seen: [5, 2, 8, 1, 4, 7, 3]

Max-Heap (left):  Min-Heap (right):
    4                  5
   / \                / \
  2   3              7   8
 /
1

Median = top of left (4) or average of (4, 5) depending on count.

Rule: |left.size() - right.size()| ≤ 1
```

### 7.2 Find Median from Data Stream (LC 295) ⭐⭐

```cpp
class MedianFinder {
    priority_queue<int> maxHeap;                                    // left half
    priority_queue<int, vector<int>, greater<int>> minHeap;         // right half
    
public:
    void addNum(int num) {
        // Step 1: Add to appropriate heap
        if (maxHeap.empty() || num <= maxHeap.top()) {
            maxHeap.push(num);
        } else {
            minHeap.push(num);
        }
        
        // Step 2: Balance — maxHeap can have at most 1 extra
        if (maxHeap.size() > minHeap.size() + 1) {
            minHeap.push(maxHeap.top());
            maxHeap.pop();
        } else if (minHeap.size() > maxHeap.size()) {
            maxHeap.push(minHeap.top());
            minHeap.pop();
        }
    }
    
    double findMedian() {
        if (maxHeap.size() > minHeap.size()) {
            return maxHeap.top();
        }
        return (maxHeap.top() + minHeap.top()) / 2.0;
    }
};
// addNum: O(log n), findMedian: O(1)
```

### 7.3 When to Use Two Heaps

| Problem | Left (Max-Heap) | Right (Min-Heap) |
|---|---|---|
| Running median | Smaller half | Larger half |
| Maximize capital | Capital ≤ budget | Profit of doable projects |
| Sliding window median | Values ≤ median | Values > median |

## 8. Pattern 4: Lazy Deletion & Streaming

### 8.1 The Problem

C++ `priority_queue` has NO `erase()` or `decrease_key()`. When you need to remove or update elements, use **lazy deletion**: mark elements as invalid and skip them when they appear at the top.

### 8.2 Lazy Deletion Template

```cpp
// Example: Track max element but sometimes delete elements
priority_queue<int> pq;
unordered_map<int, int> deleted;  // element → delete count

void add(int x) { pq.push(x); }

void remove(int x) { deleted[x]++; }  // mark for lazy deletion

int getMax() {
    while (!pq.empty() && deleted[pq.top()] > 0) {
        deleted[pq.top()]--;
        pq.pop();  // discard lazily deleted element
    }
    return pq.empty() ? -1 : pq.top();
}
```

### 8.3 When Lazy Deletion is Needed

| Problem | Why |
|---|---|
| Sliding Window Median | Remove elements leaving the window |
| Dijkstra's Algorithm | Relaxed nodes still in the heap |
| IPO (LC 502) | Projects become available as capital grows |

## 9. Pattern 5: Heap + Greedy

### 9.1 Core Intuition

> **"Greedily pick the best available option at each step"**
> → Sort or pre-process, then use heap to efficiently find the best candidate.

### 9.2 Meeting Rooms II — Min Rooms Needed (LC 253) ⭐

**Problem:** Given meeting intervals, find the minimum number of rooms required.

**Intuition:** Sort by start time. Use a min-heap to track the end time of each active room. If a new meeting starts after the earliest ending room, reuse that room.

```cpp
int minMeetingRooms(vector<vector<int>>& intervals) {
    sort(intervals.begin(), intervals.end());
    priority_queue<int, vector<int>, greater<int>> pq;  // min-heap of end times
    
    for (auto& interval : intervals) {
        // If earliest room is free, reuse it
        if (!pq.empty() && pq.top() <= interval[0]) {
            pq.pop();
        }
        pq.push(interval[1]);  // assign room with this end time
    }
    return pq.size();
}
// Time: O(n log n), Space: O(n)
```

### 9.3 Task Scheduler (LC 621) ⭐

**Problem:** Tasks `A-Z` with cooldown `n`. Minimize total time.

**Intuition:** Always execute the most frequent task first (greedy). Use a max-heap.

```cpp
int leastInterval(vector<char>& tasks, int n) {
    vector<int> freq(26, 0);
    for (char c : tasks) freq[c - 'A']++;
    
    priority_queue<int> pq;  // max-heap of frequencies
    for (int f : freq) if (f > 0) pq.push(f);
    
    int time = 0;
    while (!pq.empty()) {
        vector<int> temp;
        int cycle = n + 1;  // each cycle is n+1 slots
        
        while (cycle > 0 && !pq.empty()) {
            temp.push_back(pq.top() - 1);  // execute task
            pq.pop();
            cycle--;
            time++;
        }
        
        for (int t : temp) if (t > 0) pq.push(t);
        
        if (!pq.empty()) time += cycle;  // idle slots
    }
    return time;
}
```

### 9.4 Reorganize String (LC 767)

**Problem:** Rearrange string so no two adjacent characters are the same.

```cpp
string reorganizeString(string s) {
    unordered_map<char, int> freq;
    for (char c : s) freq[c]++;
    
    priority_queue<pair<int,char>> pq;
    for (auto& [c, f] : freq) pq.push({f, c});
    
    string result;
    while (pq.size() >= 2) {
        auto [f1, c1] = pq.top(); pq.pop();
        auto [f2, c2] = pq.top(); pq.pop();
        result += c1;
        result += c2;
        if (f1 - 1 > 0) pq.push({f1 - 1, c1});
        if (f2 - 1 > 0) pq.push({f2 - 1, c2});
    }
    if (!pq.empty()) {
        if (pq.top().first > 1) return "";  // impossible
        result += pq.top().second;
    }
    return result;
}
```

### 9.5 IPO — Maximize Capital (LC 502) ⭐

**Problem:** You have initial capital `w`. Select at most `k` projects. Each project has a cost and profit. Maximize final capital.

**Intuition:** Greedy — always pick the most profitable project you can afford. Two heaps:
1. Min-heap sorted by capital (to find affordable projects).
2. Max-heap sorted by profit (to pick the best affordable project).

```cpp
int findMaximizedCapital(int k, int w, vector<int>& profits, vector<int>& capital) {
    int n = profits.size();
    vector<pair<int,int>> projects(n);
    for (int i = 0; i < n; i++) projects[i] = {capital[i], profits[i]};
    sort(projects.begin(), projects.end());  // sort by capital needed
    
    priority_queue<int> pq;  // max-heap of available profits
    int idx = 0;
    
    for (int i = 0; i < k; i++) {
        // Unlock all affordable projects
        while (idx < n && projects[idx].first <= w) {
            pq.push(projects[idx].second);
            idx++;
        }
        if (pq.empty()) break;
        w += pq.top(); pq.pop();  // do the most profitable
    }
    return w;
}
```

### 9.6 Minimum Cost to Connect Sticks / Ropes (LC 1167)

```cpp
int connectSticks(vector<int>& sticks) {
    priority_queue<int, vector<int>, greater<int>> pq(sticks.begin(), sticks.end());
    int cost = 0;
    
    while (pq.size() > 1) {
        int a = pq.top(); pq.pop();
        int b = pq.top(); pq.pop();
        cost += a + b;
        pq.push(a + b);
    }
    return cost;
}
// Greedy: always merge the two smallest sticks first
```

## 10. Pattern 6: K Closest / Sliding Window Median

### 10.1 Find K Pairs with Smallest Sums (LC 373)

```cpp
vector<vector<int>> kSmallestPairs(vector<int>& nums1, vector<int>& nums2, int k) {
    auto cmp = [&](pair<int,int>& a, pair<int,int>& b) {
        return nums1[a.first] + nums2[a.second] > nums1[b.first] + nums2[b.second];
    };
    priority_queue<pair<int,int>, vector<pair<int,int>>, decltype(cmp)> pq(cmp);
    
    for (int i = 0; i < min((int)nums1.size(), k); i++)
        pq.push({i, 0});
    
    vector<vector<int>> result;
    while (k-- > 0 && !pq.empty()) {
        auto [i, j] = pq.top(); pq.pop();
        result.push_back({nums1[i], nums2[j]});
        if (j + 1 < nums2.size()) pq.push({i, j + 1});
    }
    return result;
}
```

### 10.2 Last Stone Weight (LC 1046)

```cpp
int lastStoneWeight(vector<int>& stones) {
    priority_queue<int> pq(stones.begin(), stones.end());  // max-heap
    
    while (pq.size() > 1) {
        int a = pq.top(); pq.pop();
        int b = pq.top(); pq.pop();
        if (a != b) pq.push(a - b);
    }
    return pq.empty() ? 0 : pq.top();
}
```

## 11. Heap Sort

### 11.1 Algorithm

```
1. Build a max-heap from the array — O(n)
2. Repeatedly swap root (max) with last element, reduce heap size, heapify — O(n log n)
```

```cpp
void heapify(vector<int>& arr, int n, int i) {
    int largest = i;
    int left = 2 * i + 1, right = 2 * i + 2;
    if (left < n && arr[left] > arr[largest]) largest = left;
    if (right < n && arr[right] > arr[largest]) largest = right;
    if (largest != i) {
        swap(arr[i], arr[largest]);
        heapify(arr, n, largest);
    }
}

void heapSort(vector<int>& arr) {
    int n = arr.size();
    
    // Build max-heap
    for (int i = n / 2 - 1; i >= 0; i--)
        heapify(arr, n, i);
    
    // Extract elements one by one
    for (int i = n - 1; i > 0; i--) {
        swap(arr[0], arr[i]);    // move max to end
        heapify(arr, i, 0);      // heapify reduced heap
    }
}
// Time: O(n log n), Space: O(1) — IN-PLACE!
// NOT stable. Worse cache performance than quicksort/mergesort.
```

### 11.2 Heap Sort vs Other Sorts

| Sort | Time (Avg) | Time (Worst) | Space | Stable? | Use Case |
|---|---|---|---|---|---|
| Quick Sort | O(n log n) | O(n²) | O(log n) | No | General, best avg |
| Merge Sort | O(n log n) | O(n log n) | O(n) | Yes | Linked lists, stability needed |
| **Heap Sort** | O(n log n) | O(n log n) | **O(1)** | No | Memory-critical, guaranteed worst |
| Counting | O(n+k) | O(n+k) | O(k) | Yes | Small range integers |

## 13. Common Mistakes & Debugging

### 13.1 Top 10 Heap Mistakes

| # | Mistake | Fix |
|---|---|---|
| 1 | **Using max-heap for K smallest, min-heap for K largest** | Use OPPOSITE: K largest → min-heap, K smallest → max-heap |
| 2 | **Forgetting `greater<int>` for min-heap** | `priority_queue<int, vector<int>, greater<int>>` |
| 3 | **Wrong comparator direction** | `return a > b` = min-heap, `return a < b` = max-heap (confusing!) |
| 4 | **`pop()` returns void in C++** | Use `top()` first, then `pop()` |
| 5 | **Calling `top()` on empty heap** | Always check `empty()` first |
| 6 | **Not handling duplicates** | Heaps allow duplicates by default. Use a counted map if needed. |
| 7 | **Expecting sorted output from heap** | Heap is NOT sorted. Only root is guaranteed min/max. |
| 8 | **Using heap when set/multiset is better** | If you need `erase()` or `find()`, use `multiset` |
| 9 | **Heap of K size creates K+1 then pops** | Push first, then check `if (pq.size() > k) pq.pop()` |
| 10 | **Forgetting tuple/pair ordering** | Pairs compare by first, then second. Use custom comparator if needed. |

### 13.2 Debugging Checklist

```
□ Is my heap min or max? (Default C++ = max-heap)
□ Am I using the OPPOSITE heap for Top K problems?
□ Am I checking empty() before top()/pop()?
□ Is my comparator returning the right direction?
□ For K-way merge: am I pushing the next element from the same list?
□ For Two Heaps: are they balanced (size difference ≤ 1)?
□ Am I handling edge cases: k=0, k=n, empty input?
□ For lazy deletion: am I cleaning lazily deleted elements from top?
```

## 14. Interview FAQs

#### Q1: Heap vs BST — when to use which?

**A:** Use a **Heap** when you only need repeated access to the min or max element (and don't need arbitrary lookups/deletions). Use a **BST** (or `set`/`map` in C++) when you need to find, insert, delete, and iterate over arbitrary elements in order. Heap is simpler and faster for its narrow use case.

#### Q2: Why does C++ default to max-heap?

**A:** Historical convention. To get a min-heap, use `priority_queue<int, vector<int>, greater<int>>`. The `greater<int>` comparator reverses the ordering.

#### Q3: Can I decrease-key in C++ priority_queue?

**A:** No. C++ `priority_queue` doesn't support `decrease_key`. Workarounds:
1. **Lazy deletion:** Push the new value, mark the old one as invalid.
2. **Use `set<>` or `multiset<>`** which support `erase()` and `insert()`.
3. **Indexed priority queue** (custom implementation).

#### Q4: What's the time complexity of building a heap?

**A:** **O(n)**, not O(n log n). This is because most nodes are near the leaves and require very little heapify-down work. The math: Σ(nodes at height h × h) = O(n). Floyd's algorithm starts from the last internal node and heapifies down.

#### Q5: When should I use QuickSelect instead of a Heap?

**A:** Use **QuickSelect** when you need ONLY the Kth element (not all K elements) and you have all data upfront (not streaming). QuickSelect is O(n) average vs Heap's O(n log k). But QuickSelect modifies the array and has O(n²) worst case.

#### Q6: How does the Two Heaps pattern work for median?

**A:** Split elements into two halves. Max-heap stores the smaller half (its top = largest of the small). Min-heap stores the larger half (its top = smallest of the large). Keep sizes balanced (differ by at most 1). Median = larger heap's top, or average of both tops.

#### Q7: What's lazy deletion and when do I need it?

**A:** Lazy deletion = mark an element as "deleted" without actually removing it from the heap. When it appears at the top during `pop()`, skip it. Need it when: 1) C++ priority_queue has no `erase()`, 2) Sliding window problems where elements expire, 3) Dijkstra-like algorithms with relaxed nodes.

#### Q8: Heap vs `multiset` for sliding window problems?

**A:** `multiset` supports O(log n) `erase()` and O(1) `begin()`/`rbegin()` (min/max). For sliding window median/max/min, `multiset` is often simpler than two heaps + lazy deletion. Trade-off: `multiset` has higher constant factor and more memory overhead.

## 15. Real-World Scenarios

### 🔍 Scenario 1: Task Scheduling in Operating Systems

**Problem:** _"The OS needs to run the highest-priority process next."_

**Answer — DSA:** Max-Heap (priority queue). Each process has a priority. The scheduler always picks the process with the highest priority (heap top) in O(1). Insert/remove is O(log n).

Used in: Linux CFS scheduler, Windows thread scheduling, real-time OS.

---

### 🔍 Scenario 2: Dijkstra's Algorithm — Shortest Paths

**Problem:** _"Find the shortest path from A to all other nodes in a weighted graph."_

**Answer — DSA:** Min-Heap. Always process the node with the smallest tentative distance. When a shorter path is found, push the updated distance (lazy deletion for old entries).

Used in: Google Maps, GPS navigation, network routing (OSPF, BGP).

---

### 🔍 Scenario 3: Hospital Emergency Room — Triage

**Problem:** _"Patients with life-threatening conditions must be treated first, regardless of arrival time."_

**Answer — DSA:** Max-Heap by severity. New patient → push with severity score. Doctor available → pop the most severe case. O(log n) per operation.

Used in: ER triage systems, ambulance dispatch, disaster response.

---

### 🔍 Scenario 4: Autocomplete / Search Suggestions

**Problem:** _"Show the top 10 most relevant search suggestions."_

**Answer — DSA:** Min-heap of size 10 (Top K pattern). As you type, score all matching queries. Keep the top 10 in the heap. Evict the lowest-scoring suggestion when a better one arrives.

Used in: Google Search, Amazon product search, IDE autocomplete.

---

### 🔍 Scenario 5: Media Player — Streaming Data Median

**Problem:** _"Show the median bitrate of the last N data points for adaptive streaming."_

**Answer — DSA:** Two Heaps (max-heap + min-heap) for running median. As each data point arrives, insert into the appropriate heap, balance, and read the median in O(1).

Used in: Netflix adaptive bitrate, Spotify audio quality, live video streaming.

---

### 🔍 Scenario 6: Load Balancer — Least Connections

**Problem:** _"Route each new request to the server with fewest active connections."_

**Answer — DSA:** Min-Heap by connection count. Pop the least-loaded server, assign the request, increment its count, and push it back. O(log n) per request.

Used in: Nginx, HAProxy, AWS ELB, cloud load balancers.

---

### 🔍 Scenario 7: Merge K Sorted Log Files

**Problem:** _"You have K sorted log files from different servers. Merge them into one sorted log."_

**Answer — DSA:** K-Way Merge with min-heap. Push one entry from each file. Pop the smallest, output it, push the next entry from that file. O(n log K).

Used in: Distributed logging (ELK Stack), external sorting, MapReduce merge phase.

---

### 🔍 Scenario 8: Uber/Lyft — Nearest Driver

**Problem:** _"Find the 5 nearest available drivers to a rider."_

**Answer — DSA:** Max-heap of size 5 by distance. For each driver, calculate distance. If closer than the farthest in the heap, swap. Result = 5 closest drivers.

Used in: Uber, Lyft, DoorDash, all location-based matching services.

---

### 🔍 Scenario 9: Event-Driven Simulation

**Problem:** _"Process events in chronological order, where new events can be generated."_

**Answer — DSA:** Min-Heap by timestamp. Pop the next event (earliest), process it (may generate new future events), push new events. Always processing in time order.

Used in: Game engines, network simulators, stock market order matching.

---

### 🔍 Scenario 10: Huffman Encoding — Data Compression

**Problem:** _"Compress data by assigning shorter codes to more frequent characters."_

**Answer — DSA:** Min-Heap. Build a frequency map, push all characters. Repeatedly pop two smallest, merge into a new node, push back. Builds an optimal prefix-free binary tree.

```
Input: {a:5, b:9, c:12, d:13, e:16, f:45}
Min-Heap merges until one tree remains.
Result: Variable-length codes based on tree depth.
```

Used in: ZIP, GZIP, JPEG, MP3, all compression algorithms using Huffman coding.
