## 1. What is Recursion?

### 1.1 The Definition

**Recursion** is when a function **calls itself** to solve a smaller version of the same problem.

> **Analogy:** Imagine you're in a line of people and want to know your position. You ask the person in front: "What's your position?" They ask the person in front of them. This continues until the first person says "I'm #1." Then the answer comes back: "I'm #2," "I'm #3," ... until it reaches you.

### 1.2 The Three Essential Parts

Every recursive function MUST have:

```
1. BASE CASE     → When to STOP (prevents infinite recursion)
2. RECURSIVE CASE → Call itself with a SMALLER problem
3. PROGRESS       → Each call must move TOWARD the base case
```

```cpp
void printNumbers(int n) {
    if (n == 0) return;        // 1. BASE CASE: stop when n = 0
    cout << n << " ";          // 3. PROGRESS: do something
    printNumbers(n - 1);       // 2. RECURSIVE CASE: smaller problem (n-1)
}
// printNumbers(5) → 5 4 3 2 1
```

### 1.3 Why Recursion Matters

| Reason | Explanation |
|---|---|
| **Trees & Graphs** | Almost impossible to traverse without recursion |
| **Divide & Conquer** | Merge Sort, Quick Sort, Binary Search |
| **Dynamic Programming** | Every DP solution starts as a recursive solution |
| **Backtracking** | Permutations, combinations, N-Queens, Sudoku |
| **Elegant code** | Some problems are 5 lines recursive vs 50 lines iterative |
| **Interview essential** | ~40% of interview problems involve recursion |

## 2. How Recursion Works — The Call Stack

### 2.1 What is the Call Stack?

Every function call creates a **stack frame** containing:
- Local variables
- Parameters
- Return address (where to go back)

Recursive calls **stack up** until the base case is hit, then **unwind** (return) back.

### 2.2 Visual Walkthrough — Factorial

```cpp
int factorial(int n) {
    if (n <= 1) return 1;        // base case
    return n * factorial(n - 1); // recursive case
}
```

```
factorial(4) is called
├── 4 * factorial(3)          ← push frame
│   ├── 3 * factorial(2)      ← push frame
│   │   ├── 2 * factorial(1)  ← push frame
│   │   │   └── return 1      ← BASE CASE HIT! Start unwinding
│   │   └── return 2 * 1 = 2  ← pop frame
│   └── return 3 * 2 = 6      ← pop frame
└── return 4 * 6 = 24         ← pop frame

Answer: 24
```

### 2.3 The Call Stack in Memory

```
┌─────────────────────────┐
│ factorial(1)  → return 1│  ← TOP (most recent call)
├─────────────────────────┤
│ factorial(2)  → waiting │
├─────────────────────────┤
│ factorial(3)  → waiting │
├─────────────────────────┤
│ factorial(4)  → waiting │
├─────────────────────────┤
│ main()                  │  ← BOTTOM
└─────────────────────────┘

Stack grows UP. When factorial(1) returns, it pops.
Then factorial(2) can compute its result, and so on.
```

### 2.4 Stack Overflow

```
void infinite() {
    infinite();  // no base case! → STACK OVERFLOW
}
```

- Each call uses memory (~1-8 KB per frame)
- Default stack size: ~1-8 MB (depends on OS)
- ~10,000-100,000 frames before crash
- **Rule:** If recursion depth > 10,000, consider converting to iteration

### 2.5 Time & Space Complexity of Recursion

| What | How to Find |
|---|---|
| **Time** | Number of recursive calls × work per call |
| **Space** | Maximum depth of call stack (+ any auxiliary data) |

```
factorial(n):
  Calls: n → Time: O(n)
  Max depth: n → Space: O(n)

fibonacci(n) naive:
  Calls: ~2ⁿ → Time: O(2ⁿ)
  Max depth: n → Space: O(n)  (only one branch active at a time)

binary_search(n):
  Calls: log n → Time: O(log n)
  Max depth: log n → Space: O(log n)
```

## 3. Types of Recursion

### 3.1 Linear Recursion

One recursive call per function.

```cpp
int factorial(int n) {
    if (n <= 1) return 1;
    return n * factorial(n - 1);  // ONE call
}
// Time: O(n), Space: O(n)
```

### 3.2 Tail Recursion

The recursive call is the **LAST** operation — nothing happens after it returns.

```cpp
// NOT tail recursive (multiplication happens AFTER the recursive call returns)
int factorial(int n) {
    if (n <= 1) return 1;
    return n * factorial(n - 1);  // must multiply after return
}

// TAIL recursive (result accumulated in parameter)
int factorialTail(int n, int acc = 1) {
    if (n <= 1) return acc;
    return factorialTail(n - 1, n * acc);  // nothing after this call
}
```

> **Why tail recursion matters:** Some compilers optimize tail recursion into a loop (Tail Call Optimization / TCO), using O(1) space instead of O(n). C++ compilers MAY do this with `-O2` flag, but it's not guaranteed.

### 3.3 Binary / Tree Recursion

Two or more recursive calls per function — creates a **tree** of calls.

```cpp
int fibonacci(int n) {
    if (n <= 1) return n;
    return fibonacci(n - 1) + fibonacci(n - 2);  // TWO calls
}
// Creates a binary tree of calls
// Time: O(2ⁿ) — exponential!
// Space: O(n) — max depth of tree
```

```
                    fib(5)
                   /      \
              fib(4)        fib(3)
             /     \        /     \
         fib(3)   fib(2)  fib(2)  fib(1)
        /    \    /    \   /    \
    fib(2) fib(1) fib(1) fib(0) fib(1) fib(0)
    /    \
fib(1) fib(0)

So much repeated work! fib(3) computed twice, fib(2) computed 3 times...
This is why we need memoization / DP.
```

### 3.4 Mutual Recursion

Two functions call each other.

```cpp
bool isEven(int n);
bool isOdd(int n);

bool isEven(int n) {
    if (n == 0) return true;
    return isOdd(n - 1);
}
bool isOdd(int n) {
    if (n == 0) return false;
    return isEven(n - 1);
}
```

### 3.5 Nested Recursion

The recursive call is inside the argument of another recursive call.

```cpp
int ackermann(int m, int n) {
    if (m == 0) return n + 1;
    if (n == 0) return ackermann(m - 1, 1);
    return ackermann(m - 1, ackermann(m, n - 1));  // nested!
}
// Grows EXTREMELY fast. ackermann(4, 2) has > 10^19728 digits!
```

## 4. Recursion vs Iteration

### 4.1 Comparison Table

| Aspect | Recursion | Iteration |
|---|---|---|
| **Approach** | Function calls itself | Loop repeats |
| **Termination** | Base case | Loop condition |
| **Memory** | O(depth) stack space | O(1) (usually) |
| **Speed** | Slower (function call overhead) | Faster (no overhead) |
| **Readability** | Cleaner for trees, graphs, backtracking | Cleaner for simple loops |
| **Risk** | Stack overflow | None |
| **Conversion** | Any recursion → iteration (with explicit stack) | Not all loops → clean recursion |

### 4.2 When to Use Which

| Use Recursion When | Use Iteration When |
|---|---|
| Tree / graph traversal | Simple counting / summing |
| Divide and conquer | Array traversal |
| Backtracking (permutations, subsets) | Known number of iterations |
| Problem has recursive structure | Deep recursion (> 10K depth) |
| Clarity > performance | Performance is critical |

### 4.3 Converting Recursion to Iteration

```cpp
// RECURSIVE DFS
void dfs(int node, vector<vector<int>>& adj, vector<bool>& visited) {
    visited[node] = true;
    for (int neighbor : adj[node]) {
        if (!visited[neighbor]) dfs(neighbor, adj, visited);
    }
}

// ITERATIVE DFS (using explicit stack)
void dfsIterative(int start, vector<vector<int>>& adj, vector<bool>& visited) {
    stack<int> st;
    st.push(start);
    while (!st.empty()) {
        int node = st.top(); st.pop();
        if (visited[node]) continue;
        visited[node] = true;
        for (int neighbor : adj[node]) {
            if (!visited[neighbor]) st.push(neighbor);
        }
    }
}
```

> **Rule of thumb:** Replace the call stack with an explicit `stack<>` or `queue<>`.

## 5. How to Think Recursively — The 3-Step Framework

> This is the **MOST IMPORTANT** section. Most students fail at recursion because they try to trace every call. Instead, use this framework.

### 5.1 The "Leap of Faith" Method

```
Step 1: DEFINE what the function does (in plain English)
Step 2: FIND the base case (simplest input)
Step 3: ASSUME the function works for smaller inputs (LEAP OF FAITH)
         → Use the result to solve the current input
```

### 5.2 Example — Sum of Array

```
Step 1: DEFINE: "sum(arr, n) returns the sum of first n elements"

Step 2: BASE CASE: If n == 0, return 0 (sum of empty array)

Step 3: LEAP OF FAITH: 
        "I trust that sum(arr, n-1) correctly gives me the sum of first n-1 elements"
        So: sum(arr, n) = arr[n-1] + sum(arr, n-1)
```

```cpp
int sum(int arr[], int n) {
    if (n == 0) return 0;                  // base case
    return arr[n - 1] + sum(arr, n - 1);   // leap of faith
}
```

### 5.3 Example — Reverse a String

```
Step 1: DEFINE: "reverse(s, l, r) reverses s from index l to r"

Step 2: BASE CASE: If l >= r, stop (0 or 1 chars left)

Step 3: LEAP OF FAITH:
        "Swap s[l] and s[r], then trust that reverse(s, l+1, r-1) 
         correctly reverses the middle portion"
```

```cpp
void reverse(string& s, int l, int r) {
    if (l >= r) return;             // base case
    swap(s[l], s[r]);               // do work
    reverse(s, l + 1, r - 1);      // leap of faith
}
```

### 5.4 Example — Check if Array is Sorted

```
Step 1: DEFINE: "isSorted(arr, n) returns true if first n elements are sorted"

Step 2: BASE CASE: If n <= 1, return true (0 or 1 elements always sorted)

Step 3: LEAP OF FAITH:
        "If arr[n-1] >= arr[n-2] AND isSorted(arr, n-1) is true, 
         then the whole thing is sorted"
```

```cpp
bool isSorted(int arr[], int n) {
    if (n <= 1) return true;
    return arr[n - 1] >= arr[n - 2] && isSorted(arr, n - 1);
}
```

### 5.5 🧠 The Golden Rules of Recursive Thinking

1. **Don't trace.** Trust your function works for smaller inputs.
2. **Focus on ONE call.** What does THIS call need to do?
3. **Base case first.** Always write the stopping condition first.
4. **Identify the subproblem.** How is the smaller version different?
5. **Combine.** How do you use the answer of the smaller problem?

## 6. Fundamental Recursion Problems (Building Blocks)

### 6.1 Print Numbers 1 to N (and N to 1)

```cpp
// Print 1 to N (print AFTER recursive call)
void print1toN(int n) {
    if (n == 0) return;
    print1toN(n - 1);  // go deeper first
    cout << n << " ";   // print on the way BACK (unwinding)
}
// print1toN(5) → 1 2 3 4 5

// Print N to 1 (print BEFORE recursive call)
void printNto1(int n) {
    if (n == 0) return;
    cout << n << " ";   // print on the way DOWN
    printNto1(n - 1);
}
// printNto1(5) → 5 4 3 2 1
```

> 🧠 **Key insight:** "Before recursive call" = work done going DOWN. "After recursive call" = work done coming BACK UP.

### 6.2 Factorial

```cpp
int factorial(int n) {
    if (n <= 1) return 1;
    return n * factorial(n - 1);
}
// Time: O(n), Space: O(n)
```

### 6.3 Fibonacci

```cpp
// Naive — O(2ⁿ) time, O(n) space
int fib(int n) {
    if (n <= 1) return n;
    return fib(n - 1) + fib(n - 2);
}

// Memoized — O(n) time, O(n) space
int fib(int n, vector<int>& dp) {
    if (n <= 1) return n;
    if (dp[n] != -1) return dp[n];
    return dp[n] = fib(n - 1, dp) + fib(n - 2, dp);
}
```

### 6.4 Power — x^n

```cpp
// Naive — O(n)
double power(double x, int n) {
    if (n == 0) return 1;
    if (n < 0) return 1.0 / power(x, -n);
    return x * power(x, n - 1);
}

// Fast Power (Binary Exponentiation) — O(log n) ✅
double fastPower(double x, long long n) {
    if (n == 0) return 1;
    if (n < 0) { x = 1 / x; n = -n; }
    if (n % 2 == 0) {
        double half = fastPower(x, n / 2);
        return half * half;            // x^n = (x^(n/2))²
    } else {
        return x * fastPower(x, n - 1); // x^n = x × x^(n-1)
    }
}
```

> 🧠 **Intuition:** x¹⁰ = (x⁵)² = (x × x⁴)² = (x × (x²)²)². Instead of 10 multiplications, only 4!

### 6.5 Sum of Digits

```cpp
int sumOfDigits(int n) {
    if (n == 0) return 0;
    return (n % 10) + sumOfDigits(n / 10);
}
// sumOfDigits(1234) = 4 + sumOfDigits(123) = 4 + 3 + 2 + 1 = 10
```

### 6.6 Count Zeros in a Number

```cpp
int countZeros(int n) {
    if (n < 10) return (n == 0) ? 1 : 0;
    return (n % 10 == 0 ? 1 : 0) + countZeros(n / 10);
}
```

### 6.7 GCD (Euclidean Algorithm)

```cpp
int gcd(int a, int b) {
    if (b == 0) return a;
    return gcd(b, a % b);
}
// Time: O(log(min(a,b))), Space: O(log(min(a,b)))
```

### 6.8 Tower of Hanoi

```cpp
void hanoi(int n, char from, char to, char aux) {
    if (n == 0) return;
    hanoi(n - 1, from, aux, to);           // move n-1 from source to aux
    cout << "Move disk " << n << " from " << from << " to " << to << endl;
    hanoi(n - 1, aux, to, from);           // move n-1 from aux to destination
}
// hanoi(3, 'A', 'C', 'B');
// Time: O(2ⁿ), Space: O(n)
// Number of moves = 2ⁿ - 1

// Intuition:
// To move n disks from A to C:
// 1. Move top n-1 disks from A to B (using C as helper)
// 2. Move the nth (largest) disk from A to C
// 3. Move n-1 disks from B to C (using A as helper)
```

## 7. Recursion on Arrays & Strings

### 7.1 Linear Search (Recursive)

```cpp
int search(int arr[], int n, int target) {
    if (n == 0) return -1;
    if (arr[n - 1] == target) return n - 1;
    return search(arr, n - 1, target);
}
```

### 7.2 Binary Search (Recursive)

```cpp
int binarySearch(int arr[], int lo, int hi, int target) {
    if (lo > hi) return -1;
    int mid = lo + (hi - lo) / 2;
    if (arr[mid] == target) return mid;
    if (arr[mid] < target) return binarySearch(arr, mid + 1, hi, target);
    return binarySearch(arr, lo, mid - 1, target);
}
// Time: O(log n), Space: O(log n) stack
```

### 7.3 Check Palindrome

```cpp
bool isPalindrome(string& s, int l, int r) {
    if (l >= r) return true;
    if (s[l] != s[r]) return false;
    return isPalindrome(s, l + 1, r - 1);
}
```

### 7.4 Reverse an Array

```cpp
void reverseArray(int arr[], int l, int r) {
    if (l >= r) return;
    swap(arr[l], arr[r]);
    reverseArray(arr, l + 1, r - 1);
}
```

### 7.5 Count Occurrences

```cpp
int countOccurrences(int arr[], int n, int target) {
    if (n == 0) return 0;
    return (arr[n - 1] == target ? 1 : 0) + countOccurrences(arr, n - 1, target);
}
```

### 7.6 Remove Character from String

```cpp
string removeChar(string s, char c) {
    if (s.empty()) return "";
    string rest = removeChar(s.substr(1), c);
    if (s[0] == c) return rest;
    return s[0] + rest;
}
// removeChar("hello", 'l') → "heo"
```

### 7.7 Replace Character in String

```cpp
string replaceChar(string s, char old, char newC) {
    if (s.empty()) return "";
    string rest = replaceChar(s.substr(1), old, newC);
    return (s[0] == old ? newC : s[0]) + rest;
}
```

### 7.8 Merge Sort — The Classic Divide & Conquer

```cpp
void merge(vector<int>& arr, int l, int m, int r) {
    vector<int> temp;
    int i = l, j = m + 1;
    while (i <= m && j <= r) {
        if (arr[i] <= arr[j]) temp.push_back(arr[i++]);
        else temp.push_back(arr[j++]);
    }
    while (i <= m) temp.push_back(arr[i++]);
    while (j <= r) temp.push_back(arr[j++]);
    for (int k = l; k <= r; k++) arr[k] = temp[k - l];
}

void mergeSort(vector<int>& arr, int l, int r) {
    if (l >= r) return;            // base case: 0 or 1 element
    int mid = l + (r - l) / 2;
    mergeSort(arr, l, mid);        // sort left half
    mergeSort(arr, mid + 1, r);    // sort right half
    merge(arr, l, mid, r);         // merge sorted halves
}
// Time: O(n log n), Space: O(n)

// Recurrence: T(n) = 2T(n/2) + O(n)
// By Master Theorem: a=2, b=2, d=1 → log₂(2) = 1 = d → O(n log n)
```

### 7.9 Quick Sort

```cpp
int partition(vector<int>& arr, int lo, int hi) {
    int pivot = arr[hi];
    int i = lo - 1;
    for (int j = lo; j < hi; j++) {
        if (arr[j] < pivot) {
            i++;
            swap(arr[i], arr[j]);
        }
    }
    swap(arr[i + 1], arr[hi]);
    return i + 1;
}

void quickSort(vector<int>& arr, int lo, int hi) {
    if (lo >= hi) return;
    int p = partition(arr, lo, hi);
    quickSort(arr, lo, p - 1);
    quickSort(arr, p + 1, hi);
}
// Time: O(n log n) average, O(n²) worst
// Space: O(log n) average stack depth
```

## 8. Recursion on Subsequences — The Power Pattern

> This is where recursion gets POWERFUL. Understanding the "include/exclude" pattern unlocks backtracking, subsets, combinations, and DP.

### 8.1 The Include/Exclude Pattern

For each element, you have two choices:
1. **Include** it in the current result
2. **Exclude** (skip) it

This creates a **binary decision tree** with 2ⁿ leaves (all possible subsequences).

```
Array: [1, 2, 3]

                    f(0, [])
                   /         \
          Include 1           Exclude 1
          f(1, [1])           f(1, [])
         /       \           /       \
    Inc 2      Exc 2     Inc 2    Exc 2
  f(2,[1,2])  f(2,[1]) f(2,[2])  f(2,[])
   /    \     /    \    /    \    /    \
 I3   E3   I3   E3  I3   E3  I3   E3
[1,2,3] [1,2] [1,3] [1] [2,3] [2] [3] []

Subsets: [], [1], [2], [3], [1,2], [1,3], [2,3], [1,2,3]
```

### 8.2 Generate All Subsequences / Subsets

```cpp
void generateSubsets(vector<int>& arr, int i, vector<int>& current, 
                     vector<vector<int>>& result) {
    if (i == arr.size()) {
        result.push_back(current);  // base case: processed all elements
        return;
    }
    
    // Choice 1: Include arr[i]
    current.push_back(arr[i]);
    generateSubsets(arr, i + 1, current, result);
    current.pop_back();  // BACKTRACK! undo the include
    
    // Choice 2: Exclude arr[i]
    generateSubsets(arr, i + 1, current, result);
}

// Usage:
vector<int> arr = {1, 2, 3};
vector<int> current;
vector<vector<int>> result;
generateSubsets(arr, 0, current, result);
// result = [[], [3], [2], [2,3], [1], [1,3], [1,2], [1,2,3]]
// Time: O(2ⁿ × n), Space: O(n) recursion depth
```

### 8.3 Print All Subsequences with Given Sum

```cpp
void subsequencesWithSum(vector<int>& arr, int i, vector<int>& current, 
                         int currentSum, int targetSum) {
    if (i == arr.size()) {
        if (currentSum == targetSum) {
            for (int x : current) cout << x << " ";
            cout << endl;
        }
        return;
    }
    
    // Include
    current.push_back(arr[i]);
    subsequencesWithSum(arr, i + 1, current, currentSum + arr[i], targetSum);
    current.pop_back();
    
    // Exclude
    subsequencesWithSum(arr, i + 1, current, currentSum, targetSum);
}
```

### 8.4 Count Subsequences with Given Sum

```cpp
int countSubsequences(vector<int>& arr, int i, int currentSum, int targetSum) {
    if (i == arr.size()) {
        return (currentSum == targetSum) ? 1 : 0;
    }
    
    // Include + Exclude
    return countSubsequences(arr, i + 1, currentSum + arr[i], targetSum)
         + countSubsequences(arr, i + 1, currentSum, targetSum);
}
```

### 8.5 Check if ANY Subsequence with Given Sum Exists

```cpp
bool anySubsequence(vector<int>& arr, int i, int currentSum, int targetSum) {
    if (i == arr.size()) return currentSum == targetSum;
    
    // If include gives true, no need to check exclude (early termination)
    if (anySubsequence(arr, i + 1, currentSum + arr[i], targetSum))
        return true;
    return anySubsequence(arr, i + 1, currentSum, targetSum);
}
```

### 8.6 Generate All Subsequences of a String

```cpp
void stringSubsequences(string& s, int i, string current) {
    if (i == s.length()) {
        cout << "\"" << current << "\"" << endl;
        return;
    }
    
    // Include s[i]
    stringSubsequences(s, i + 1, current + s[i]);
    
    // Exclude s[i]
    stringSubsequences(s, i + 1, current);
}
// stringSubsequences("abc", 0, "") 
// → "abc", "ab", "ac", "a", "bc", "b", "c", ""
```

### 8.7 🧠 The Include/Exclude Template (MEMORIZE THIS)

```cpp
void solve(input, index, currentState, result) {
    // BASE CASE
    if (index == input.size()) {
        if (isValid(currentState)) result.add(currentState);
        return;
    }
    
    // INCLUDE current element
    addToCurrent(currentState, input[index]);
    solve(input, index + 1, currentState, result);
    removeFromCurrent(currentState, input[index]);  // BACKTRACK
    
    // EXCLUDE current element
    solve(input, index + 1, currentState, result);
}
```

> This single template solves: subsets, subsequences, combinations, subsequence sum, string subsequences, and is the foundation of all backtracking.

| Fundamentals | Factorial, Fibonacci, Power, GCD, Tower of Hanoi |
| Arrays & Strings | Search, reverse, palindrome, merge sort, quick sort |
| Subsequences | Include/Exclude pattern → 2ⁿ subsets, sum conditions |

## 9. Backtracking — Complete Guide

### 9.1 What is Backtracking?

**Backtracking** = recursion + **undoing choices** that don't lead to a solution.

> **Analogy:** You're in a maze. At each fork, you choose a path. If you hit a dead end, you **go back** to the last fork and try a different path. That "going back" is backtracking.

```
Backtracking = Try → Check → Undo

1. TRY a choice (make a move)
2. RECURSE with that choice
3. CHECK if it leads to a solution
4. If not, UNDO the choice (backtrack) and try the next option
```

### 9.2 The Backtracking Template — MEMORIZE THIS

```cpp
void backtrack(state, choices, result) {
    // BASE CASE: found a complete solution
    if (isComplete(state)) {
        result.add(state);
        return;
    }
    
    for (each choice in choices) {
        if (isValid(choice)) {      // PRUNE: skip invalid choices
            makeChoice(state);       // TRY
            backtrack(state, remaining_choices, result);  // RECURSE
            undoChoice(state);       // UNDO (backtrack)
        }
    }
}
```

### 9.3 Permutations — All Orderings

```cpp
// Generate all permutations of [1,2,...,n]
void permutations(vector<int>& arr, int idx, vector<vector<int>>& result) {
    if (idx == arr.size()) {
        result.push_back(arr);  // found a complete permutation
        return;
    }
    for (int i = idx; i < arr.size(); i++) {
        swap(arr[idx], arr[i]);              // TRY: place arr[i] at position idx
        permutations(arr, idx + 1, result);  // RECURSE
        swap(arr[idx], arr[i]);              // UNDO
    }
}
// Time: O(n! × n), Space: O(n)

// Alternative: Using a visited array
void permute(vector<int>& arr, vector<bool>& used, vector<int>& current,
             vector<vector<int>>& result) {
    if (current.size() == arr.size()) {
        result.push_back(current);
        return;
    }
    for (int i = 0; i < arr.size(); i++) {
        if (used[i]) continue;          // skip already used
        used[i] = true;                 // TRY
        current.push_back(arr[i]);
        permute(arr, used, current, result);  // RECURSE
        current.pop_back();             // UNDO
        used[i] = false;
    }
}
```

**Handling duplicates (Permutations II):**
```cpp
void permuteUnique(vector<int>& arr, vector<bool>& used, vector<int>& current,
                   vector<vector<int>>& result) {
    if (current.size() == arr.size()) {
        result.push_back(current);
        return;
    }
    for (int i = 0; i < arr.size(); i++) {
        if (used[i]) continue;
        // Skip duplicates: if same value as previous AND previous is unused (was backtracked)
        if (i > 0 && arr[i] == arr[i-1] && !used[i-1]) continue;
        used[i] = true;
        current.push_back(arr[i]);
        permuteUnique(arr, used, current, result);
        current.pop_back();
        used[i] = false;
    }
}
// IMPORTANT: sort arr first!
```

### 9.4 Combinations — Choose r from n

```cpp
// Generate all C(n, r) combinations
void combine(int n, int r, int start, vector<int>& current,
             vector<vector<int>>& result) {
    if (current.size() == r) {
        result.push_back(current);
        return;
    }
    for (int i = start; i <= n; i++) {
        current.push_back(i);                // TRY
        combine(n, r, i + 1, current, result); // RECURSE (i+1 to avoid reuse)
        current.pop_back();                   // UNDO
    }
}
// combine(4, 2, ...) → [1,2], [1,3], [1,4], [2,3], [2,4], [3,4]
```

### 9.5 Combination Sum — Unlimited Use of Elements

```cpp
// Find all combinations that sum to target (can reuse elements)
void combinationSum(vector<int>& candidates, int target, int start,
                    vector<int>& current, vector<vector<int>>& result) {
    if (target == 0) {
        result.push_back(current);
        return;
    }
    if (target < 0) return;  // PRUNE
    
    for (int i = start; i < candidates.size(); i++) {
        current.push_back(candidates[i]);
        combinationSum(candidates, target - candidates[i], i, current, result); // i, not i+1!
        current.pop_back();
    }
}

// Combination Sum II (each element used at most once, has duplicates):
void combinationSum2(vector<int>& candidates, int target, int start,
                     vector<int>& current, vector<vector<int>>& result) {
    if (target == 0) {
        result.push_back(current);
        return;
    }
    for (int i = start; i < candidates.size(); i++) {
        if (i > start && candidates[i] == candidates[i-1]) continue; // skip dups
        if (candidates[i] > target) break; // PRUNE (requires sorted input)
        current.push_back(candidates[i]);
        combinationSum2(candidates, target - candidates[i], i + 1, current, result);
        current.pop_back();
    }
}
// IMPORTANT: sort candidates first!
```

### 9.6 Subsets (Power Set)

```cpp
// Method 1: Include/Exclude (from Section 8)
void subsets(vector<int>& arr, int i, vector<int>& current,
             vector<vector<int>>& result) {
    if (i == arr.size()) {
        result.push_back(current);
        return;
    }
    current.push_back(arr[i]);           // Include
    subsets(arr, i + 1, current, result);
    current.pop_back();                   // Backtrack
    subsets(arr, i + 1, current, result); // Exclude
}

// Method 2: Iterative (for-loop based, useful for Subsets II)
void subsets2(vector<int>& arr, int start, vector<int>& current,
              vector<vector<int>>& result) {
    result.push_back(current);  // every state is a valid subset
    for (int i = start; i < arr.size(); i++) {
        current.push_back(arr[i]);
        subsets2(arr, i + 1, current, result);
        current.pop_back();
    }
}

// Subsets II (with duplicates — skip same-level duplicates)
void subsetsWithDup(vector<int>& arr, int start, vector<int>& current,
                    vector<vector<int>>& result) {
    result.push_back(current);
    for (int i = start; i < arr.size(); i++) {
        if (i > start && arr[i] == arr[i-1]) continue; // skip dups at same level
        current.push_back(arr[i]);
        subsetsWithDup(arr, i + 1, current, result);
        current.pop_back();
    }
}
// IMPORTANT: sort arr first!
```

### 9.7 N-Queens — The Classic Backtracking Problem

```cpp
// Place N queens on an N×N board so no two attack each other
bool isSafe(vector<string>& board, int row, int col, int n) {
    // Check column above
    for (int i = 0; i < row; i++)
        if (board[i][col] == 'Q') return false;
    
    // Check upper-left diagonal
    for (int i = row-1, j = col-1; i >= 0 && j >= 0; i--, j--)
        if (board[i][j] == 'Q') return false;
    
    // Check upper-right diagonal
    for (int i = row-1, j = col+1; i >= 0 && j < n; i--, j++)
        if (board[i][j] == 'Q') return false;
    
    return true;
}

void solveNQueens(vector<string>& board, int row, int n,
                  vector<vector<string>>& result) {
    if (row == n) {
        result.push_back(board);  // all queens placed!
        return;
    }
    for (int col = 0; col < n; col++) {
        if (isSafe(board, row, col, n)) {
            board[row][col] = 'Q';              // TRY
            solveNQueens(board, row + 1, n, result); // RECURSE
            board[row][col] = '.';              // UNDO
        }
    }
}
// Time: O(n!), Space: O(n²)
```

### 9.8 Sudoku Solver

```cpp
bool isValid(vector<vector<char>>& board, int row, int col, char c) {
    for (int i = 0; i < 9; i++) {
        if (board[row][i] == c) return false;  // check row
        if (board[i][col] == c) return false;  // check column
        // check 3×3 box
        int boxRow = 3 * (row / 3) + i / 3;
        int boxCol = 3 * (col / 3) + i % 3;
        if (board[boxRow][boxCol] == c) return false;
    }
    return true;
}

bool solveSudoku(vector<vector<char>>& board) {
    for (int row = 0; row < 9; row++) {
        for (int col = 0; col < 9; col++) {
            if (board[row][col] == '.') {
                for (char c = '1'; c <= '9'; c++) {
                    if (isValid(board, row, col, c)) {
                        board[row][col] = c;         // TRY
                        if (solveSudoku(board))       // RECURSE
                            return true;              // found solution!
                        board[row][col] = '.';        // UNDO
                    }
                }
                return false;  // no valid number → backtrack
            }
        }
    }
    return true;  // all cells filled
}
```

### 9.9 Palindrome Partitioning

```cpp
// Partition string so every substring is a palindrome
bool isPalindrome(string& s, int l, int r) {
    while (l < r) { if (s[l++] != s[r--]) return false; }
    return true;
}

void partitionPalindrome(string& s, int start, vector<string>& current,
                         vector<vector<string>>& result) {
    if (start == s.length()) {
        result.push_back(current);
        return;
    }
    for (int end = start; end < s.length(); end++) {
        if (isPalindrome(s, start, end)) {
            current.push_back(s.substr(start, end - start + 1)); // TRY
            partitionPalindrome(s, end + 1, current, result);     // RECURSE
            current.pop_back();                                    // UNDO
        }
    }
}
// "aab" → [["a","a","b"], ["aa","b"]]
```

### 9.10 Word Search in Grid

```cpp
bool wordSearch(vector<vector<char>>& board, string& word, int i, int j, int k) {
    if (k == word.length()) return true;  // found entire word
    if (i < 0 || i >= board.size() || j < 0 || j >= board[0].size()) return false;
    if (board[i][j] != word[k]) return false;
    
    char temp = board[i][j];
    board[i][j] = '#';  // mark visited
    
    bool found = wordSearch(board, word, i+1, j, k+1)
              || wordSearch(board, word, i-1, j, k+1)
              || wordSearch(board, word, i, j+1, k+1)
              || wordSearch(board, word, i, j-1, k+1);
    
    board[i][j] = temp;  // UNDO (backtrack)
    return found;
}
```

### 9.11 🧠 Backtracking vs Subsequences — When to Use Which Pattern

| Pattern | Template | Use When |
|---|---|---|
| **Include/Exclude** | Binary choice per element | Subsets, subsequences, subset sum |
| **For-loop backtracking** | Try each option at current position | Permutations, combinations, N-Queens, Sudoku |

```
Include/Exclude:              For-loop:
      idx                       idx
     /   \                    / | | \
  include  exclude          c1  c2 c3 c4
   idx+1   idx+1           idx+1 for each
```

## 10. Divide and Conquer

### 10.1 What is Divide & Conquer?

```
1. DIVIDE the problem into smaller subproblems
2. CONQUER each subproblem recursively
3. COMBINE the results
```

### 10.2 Classic Divide & Conquer Problems

| Problem | Divide | Conquer | Combine | Time |
|---|---|---|---|---|
| Merge Sort | Split array in half | Sort each half | Merge sorted halves | O(n log n) |
| Quick Sort | Partition around pivot | Sort each partition | Already combined | O(n log n) avg |
| Binary Search | Pick middle | Search one half | Return result | O(log n) |
| Max Subarray (Kadane's D&C) | Split in half | Max in each half | Max crossing middle | O(n log n) |
| Count Inversions | Split in half | Count in each half | Count cross inversions | O(n log n) |
| Closest Pair of Points | Split by x | Closest in each half | Check strip | O(n log n) |
| Strassen's Matrix Mult | Split into 4 sub-matrices | 7 recursive mults | Combine sub-results | O(n^2.81) |
| Power (x^n) | Split exponent | Compute half power | Square the result | O(log n) |

### 10.3 Count Inversions (using Merge Sort)

```cpp
// An inversion is a pair (i, j) where i < j but arr[i] > arr[j]
long long mergeCount(vector<int>& arr, int l, int m, int r) {
    vector<int> temp;
    int i = l, j = m + 1;
    long long inversions = 0;
    
    while (i <= m && j <= r) {
        if (arr[i] <= arr[j]) {
            temp.push_back(arr[i++]);
        } else {
            inversions += (m - i + 1);  // all remaining left elements form inversions
            temp.push_back(arr[j++]);
        }
    }
    while (i <= m) temp.push_back(arr[i++]);
    while (j <= r) temp.push_back(arr[j++]);
    for (int k = l; k <= r; k++) arr[k] = temp[k - l];
    return inversions;
}

long long countInversions(vector<int>& arr, int l, int r) {
    if (l >= r) return 0;
    int mid = l + (r - l) / 2;
    long long count = 0;
    count += countInversions(arr, l, mid);
    count += countInversions(arr, mid + 1, r);
    count += mergeCount(arr, l, mid, r);
    return count;
}
// Time: O(n log n)
```

## 11. Recursion to Dynamic Programming — The Bridge

### 11.1 Why Recursion Leads to DP

```
Plain Recursion → overlapping subproblems → repeated work → SLOW (O(2ⁿ))
     ↓ add memoization
Memoized Recursion (Top-Down DP) → store results → NO repeated work → FAST (O(n))
     ↓ convert to iteration
Tabulation (Bottom-Up DP) → fill table iteratively → FAST + less stack space
```

### 11.2 The 3-Step Conversion

#### Step 1: Write the Recursive Solution

```cpp
// Fibonacci — plain recursion O(2ⁿ)
int fib(int n) {
    if (n <= 1) return n;
    return fib(n - 1) + fib(n - 2);
}
```

#### Step 2: Add Memoization (Top-Down DP)

```cpp
// Fibonacci — memoized O(n) time, O(n) space
int fib(int n, vector<int>& dp) {
    if (n <= 1) return n;
    if (dp[n] != -1) return dp[n];        // already computed? return it
    return dp[n] = fib(n-1, dp) + fib(n-2, dp);  // store before returning
}
// Initialize: vector<int> dp(n+1, -1);
```

#### Step 3: Convert to Tabulation (Bottom-Up DP)

```cpp
// Fibonacci — tabulation O(n) time, O(n) space
int fib(int n) {
    if (n <= 1) return n;
    vector<int> dp(n + 1);
    dp[0] = 0; dp[1] = 1;
    for (int i = 2; i <= n; i++)
        dp[i] = dp[i-1] + dp[i-2];
    return dp[n];
}

// Space-optimized — O(1) space
int fib(int n) {
    if (n <= 1) return n;
    int prev2 = 0, prev1 = 1;
    for (int i = 2; i <= n; i++) {
        int curr = prev1 + prev2;
        prev2 = prev1;
        prev1 = curr;
    }
    return prev1;
}
```

### 11.3 Classic Recursion → DP Examples

| Problem | Recursive | Memoized | Tabulated |
|---|---|---|---|
| Fibonacci | O(2ⁿ) | O(n) | O(n) → O(1) space |
| 0/1 Knapsack | O(2ⁿ) | O(n×W) | O(n×W) → O(W) space |
| Longest Common Subsequence | O(2^(m+n)) | O(m×n) | O(m×n) → O(n) space |
| Coin Change | O(2ⁿ) | O(n×amount) | O(n×amount) |
| Climbing Stairs | O(2ⁿ) | O(n) | O(n) → O(1) space |
| Edit Distance | O(3^(m+n)) | O(m×n) | O(m×n) |

### 11.4 How to Identify DP Problems

Ask yourself these two questions:
1. **Overlapping subproblems?** → Same function called with same arguments multiple times?
2. **Optimal substructure?** → Can optimal solution be built from optimal solutions of subproblems?

If BOTH → Use DP. If only optimal substructure (no overlap) → Divide & Conquer.

## 13. Common Mistakes & Debugging Recursion

### 13.1 The Top 10 Recursion Mistakes

| # | Mistake | Fix |
|---|---|---|
| 1 | **No base case** | Always write base case FIRST before any other logic |
| 2 | **Base case doesn't cover all terminal states** | Think: what's the smallest valid AND invalid input? |
| 3 | **Not making progress toward base case** | Ensure each recursive call reduces the problem |
| 4 | **Forgetting to return** | `return recursiveCall(...)` not just `recursiveCall(...)` |
| 5 | **Not backtracking (undo step)** | After recursive call, undo the change (pop, unmark, swap back) |
| 6 | **Wrong index increment** | `i+1` for no reuse, `i` for reuse allowed |
| 7 | **Stack overflow** | Limit depth, convert to iteration, or use tail recursion |
| 8 | **Tracing all calls instead of trusting** | Use "Leap of Faith" — focus on ONE level |
| 9 | **Modifying shared state without restoration** | Pass by value, or undo changes after recursion |
| 10 | **Not sorting before duplicate-skipping** | Subsets II, Permutations II, Combination Sum II ALL need sorted input |

### 13.2 How to Debug Recursive Functions

```cpp
// Add depth-based indentation to visualize the call tree
void debug(int n, int depth = 0) {
    string indent(depth * 2, ' ');
    cout << indent << "Call: n=" << n << endl;
    
    if (n <= 0) {
        cout << indent << "Base case, return" << endl;
        return;
    }
    
    debug(n - 1, depth + 1);
    cout << indent << "Return from n=" << n << endl;
}
```

### 13.3 Recursion Debugging Checklist

```
□ Does the base case handle ALL stopping conditions?
□ Does each recursive call move TOWARD the base case?
□ Am I returning the recursive result (not just calling)?
□ Am I undoing modifications after backtracking?
□ Is the function producing correct results for n=0, 1, 2?
□ Am I passing the right index (i vs i+1)?
□ Is the stack depth manageable (< 10K)?
□ For duplicate handling: did I sort first?
```

## 14. Interview Tips & FAQs

### 💡 Top Interview Tips for Recursion

1. **Always start with the base case** — write it first, think about it first.
2. **Use the "Leap of Faith"** — don't trace all calls. Focus on one level.
3. **Draw the recursion tree** — for binary recursion, sketch 2-3 levels to see the pattern.
4. **Identify the choice** — what decisions are you making at each step?
5. **Think about state** — what information does each call need? Pass it as parameters.
6. **Backtracking = recursion + undo** — if you modify something, undo it after returning.
7. **Sort first for duplicate problems** — Subsets II, Permutations II, Combination Sum II.
8. **Know when to switch to DP** — if you see overlapping subproblems, memoize.

### 🎯 Frequently Asked Questions

#### Q1: What's the difference between recursion and backtracking?

**A:** All backtracking is recursion, but not all recursion is backtracking. **Recursion** = function calls itself. **Backtracking** = recursion + making a choice → trying it → undoing it if it doesn't work. Backtracking explores a decision tree and prunes invalid branches.

#### Q2: How do you convert any recursion to iteration?

**A:** Use an explicit stack (for DFS-like recursion) or queue (for BFS-like). Push function state (parameters) onto the stack. Pop and process. This simulates the call stack manually.

#### Q3: What's the time complexity of generating all subsets?

**A:** O(2ⁿ × n). There are 2ⁿ subsets, and copying each subset takes O(n) in the worst case. This is optimal — you can't generate 2ⁿ subsets faster than O(2ⁿ).

#### Q4: What's the time complexity of generating all permutations?

**A:** O(n! × n). There are n! permutations, and each takes O(n) to copy/process.

#### Q5: When should I use memoization vs tabulation?

**A:** **Memoization (top-down):** easier to write (just add a cache to recursion), solves only needed subproblems, but uses stack space. **Tabulation (bottom-up):** iterative, better for space optimization, solves all subproblems. Start with memoization in interviews (faster to code), then mention tabulation as optimization.

#### Q6: How do I handle duplicates in backtracking?

**A:** Sort the input first. Then in the for-loop, skip `arr[i]` if `arr[i] == arr[i-1]` AND `i > start` (for combinations/subsets) or `!used[i-1]` (for permutations). The key insight: prevent the same value from being chosen twice at the same recursion level.

#### Q7: What's the difference between subsets (include/exclude) and combinations (for-loop)?

**A:** Both can generate subsets. **Include/exclude** makes a binary choice per element (include or skip), naturally giving 2ⁿ subsets. **For-loop** picks elements in order with a start index, naturally extending to combinations of fixed size. Use include/exclude for "all subsets" and for-loop for "subsets/combinations of size K."

#### Q8: What is pruning in backtracking?

**A:** Pruning means skipping branches of the recursion tree that can't possibly lead to valid solutions. Examples: In N-Queens, don't try columns already attacked. In Combination Sum, if current sum exceeds target, stop. Pruning can dramatically reduce time from O(n!) to something much smaller.

```sql
    UNION ALL
    SELECT e.id, e.name, e.manager_id
    FROM employees e JOIN managers m ON e.id = m.manager_id
)
SELECT * FROM managers;
```

Used in: Org charts, category trees, bill of materials, social network graphs.

### 🔍 Scenario 5: Compiler — Abstract Syntax Tree (AST)

**Problem:** _"Evaluate the expression `(3 + 4) * (2 - 1)`."_

```
        ×
       / \
      +    -
     / \  / \
    3   4 2   1
```

```cpp
int evaluate(Node* node) {
    if (node->isNumber) return node->value;  // base case (leaf)
    int left = evaluate(node->left);          // recurse left
    int right = evaluate(node->right);        // recurse right
    return applyOp(node->op, left, right);   // combine
}
```

Used in: Every compiler, interpreter, calculator, formula engine.

**Answer:** History forms a tree (not a stack). Each state has children (actions taken from that point). Traversal is recursive — DFS to find the desired state.

### 🔍 Scenario 7: Autocomplete with Trie

```cpp
void findWords(TrieNode* node, string prefix, vector<string>& results) {
    if (node->isEnd) results.push_back(prefix);
    for (auto& [ch, child] : node->children) {
        findWords(child, prefix + ch, results);  // recurse through trie
    }
}
```

Used in: Search engines, IDEs, mobile keyboards.

### 🔍 Scenario 8: Merge Sort in Database Systems

Databases use **external merge sort** for sorting data larger than RAM:
1. Split data into chunks that fit in memory
2. Sort each chunk (recursive merge sort)
3. Merge sorted chunks from disk

Used in: MySQL `ORDER BY`, PostgreSQL sort, MapReduce's shuffle phase.

```cpp
void drawTree(Point start, float length, float angle, int depth) {
```

### 🔍 Scenario 10: Network Packet Routing — Graph DFS/BFS

**Problem:** _"Find if two nodes in a network can communicate."_

```cpp
bool canReach(int src, int dest, vector<vector<int>>& adj, vector<bool>& visited) {
    if (src == dest) return true;
    visited[src] = true;
    for (int neighbor : adj[src]) {
        if (!visited[neighbor] && canReach(neighbor, dest, adj, visited))
            return true;
    }
    return false;
}
```

Used in: Network routing, social network analysis, dependency resolution.
