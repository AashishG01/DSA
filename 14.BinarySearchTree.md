## 1. What is a Binary Search Tree?

### 1.1 The Definition

A **Binary Search Tree (BST)** is a binary tree where for EVERY node:
- All values in the **left subtree** are **less than** the node's value.
- All values in the **right subtree** are **greater than** the node's value.
- Both left and right subtrees are also BSTs.

```
        8
       / \
      3   10
     / \    \
    1   6    14
       / \   /
      4   7 13

Valid BST:
✅ 3 < 8 (left of 8), 10 > 8 (right of 8)
✅ 1 < 3 (left of 3), 6 > 3 (right of 3)
✅ 4 < 6 (left of 6), 7 > 6 (right of 6)
✅ 13 < 14 (left of 14)
```

### 1.2 The BST Property — Why It Matters

> **The BST property turns a tree into a "sorted dictionary."**
> You can search, insert, and delete in **O(h)** time, where h = height.

| Property | Impact |
|---|---|
| Left < Root < Right | Enables binary search on a tree structure |
| Inorder traversal = sorted | Walking left→root→right gives sorted output |
| Each comparison eliminates half | Like binary search on arrays, but dynamic |

### 1.3 BST Node Structure

```cpp
struct TreeNode {
    int val;
    TreeNode* left;
    TreeNode* right;
    TreeNode(int x) : val(x), left(nullptr), right(nullptr) {}
};
```

### 1.4 BST vs Binary Tree vs Heap

| Feature | Binary Tree | **BST** | Heap |
|---|---|---|---|
| Ordering | None | Left < Root < Right | Parent vs children |
| Search | O(n) | **O(h)** ✅ | O(n) |
| Insert | O(1) at leaf | **O(h)** | O(log n) |
| Delete | O(n) | **O(h)** | O(log n) |
| Find Min/Max | O(n) | **O(h)** ✅ | O(1) |
| Sorted order | O(n log n) | **O(n) inorder** ✅ | Not possible |
| Shape | Any | Any (ideally balanced) | Complete |
| h (height) | O(n) worst | O(n) worst, O(log n) balanced | O(log n) always |

### 1.5 Height Matters — Balanced vs Skewed

```
BALANCED BST (h = log n):     SKEWED BST (h = n):
        4                      1
       / \                      \
      2   6                      2
     / \ / \                      \
    1  3 5  7                      3
                                    \
                                     4
                                      \
                                       5

O(log n) operations           O(n) operations = linked list!
```

> **Self-balancing BSTs** (AVL, Red-Black) guarantee h = O(log n).

## 2. BST Operations — CRUD

### 2.1 Search — O(h)

```cpp
TreeNode* search(TreeNode* root, int target) {
    if (!root || root->val == target) return root;
    
    if (target < root->val) return search(root->left, target);
    else return search(root->right, target);
}

// Iterative version (preferred — no stack overflow)
TreeNode* searchIterative(TreeNode* root, int target) {
    while (root && root->val != target) {
        root = (target < root->val) ? root->left : root->right;
    }
    return root;
}
```

### 2.2 Insert — O(h)

```cpp
TreeNode* insert(TreeNode* root, int val) {
    if (!root) return new TreeNode(val);
    
    if (val < root->val) root->left = insert(root->left, val);
    else if (val > root->val) root->right = insert(root->right, val);
    // val == root->val → duplicate, do nothing (or handle as needed)
    
    return root;
}
```

### 2.3 Delete — O(h) ⭐

Deletion has 3 cases:

```
Case 1: Node has NO children (leaf) → just remove it.
Case 2: Node has ONE child → replace node with its child.
Case 3: Node has TWO children → replace node with its:
        - INORDER SUCCESSOR (smallest in right subtree), OR
        - INORDER PREDECESSOR (largest in left subtree)
        Then delete the successor/predecessor.
```

```cpp
TreeNode* findMin(TreeNode* node) {
    while (node->left) node = node->left;
    return node;
}

TreeNode* deleteNode(TreeNode* root, int key) {
    if (!root) return root;
    
    if (key < root->val) {
        root->left = deleteNode(root->left, key);
    } else if (key > root->val) {
        root->right = deleteNode(root->right, key);
    } else {
        // Found the node to delete
        
        // Case 1 & 2: 0 or 1 child
        if (!root->left) {
            TreeNode* temp = root->right;
            delete root;
            return temp;
        }
        if (!root->right) {
            TreeNode* temp = root->left;
            delete root;
            return temp;
        }
        
        // Case 3: 2 children — find inorder successor
        TreeNode* successor = findMin(root->right);
        root->val = successor->val;
        root->right = deleteNode(root->right, successor->val);
    }
    return root;
}
```

### 2.4 Find Min / Find Max — O(h)

```cpp
int findMin(TreeNode* root) {
    while (root->left) root = root->left;  // go leftmost
    return root->val;
}

int findMax(TreeNode* root) {
    while (root->right) root = root->right;  // go rightmost
    return root->val;
}
```

## 3. 🧠 How to Think About BST Problems — Intuition

### 3.1 The TWO Key Properties to Exploit

> **Property 1: Inorder traversal of a BST = sorted array.**
> If a problem involves "sorted order" + "tree" → think inorder.
>
> **Property 2: BST search = binary search on a tree.**
> At each node, you KNOW which subtree to explore. Go left OR right, never both.

### 3.2 The 6 BST Patterns — Decision Framework

```
┌──────────────────────────────────────────────────────────────────┐
│                   WHEN DO BST PROPERTIES HELP?                   │
├──────────────────────────────────────────────────────────────────┤
│                                                                  │
│  1. INORDER = SORTED                                             │
│     → "Kth smallest", "sorted order", "convert to sorted list"   │
│     → Do inorder traversal, process in sorted order              │
│                                                                  │
│  2. BST SEARCH (Go Left or Right)                                │
│     → "Search", "insert", "closest value"                        │
│     → At each node, discard one subtree                          │
│                                                                  │
│  3. VALIDATE & BOUND PASSING                                     │
│     → "Is this a valid BST?"                                     │
│     → Pass (min, max) bounds down the recursion                  │
│                                                                  │
│  4. LCA (Lowest Common Ancestor)                                 │
│     → "Find LCA of two nodes"                                    │
│     → BST LCA is much easier than BT LCA                        │
│     → If both < root → go left, both > root → go right          │
│                                                                  │
│  5. CONSTRUCTION                                                 │
│     → "Build BST from preorder / sorted array"                   │
│     → Use bounds or divide-and-conquer on sorted data            │
│                                                                  │
│  6. BST ↔ OTHER STRUCTURES                                      │
│     → "Convert BST to sorted DLL / linked list"                  │
│     → Inorder traversal with pointer rewiring                    │
│                                                                  │
└──────────────────────────────────────────────────────────────────┘
```

### 3.3 BST vs Binary Tree — When Does BST Property Help?

```
Question: Is the problem asking about a GENERIC binary tree, or a BST?

Generic BT approach → DFS/BFS on both subtrees → often O(n)
BST approach       → Exploit ordering → often O(h) = O(log n)

Example: Find LCA
  BT:  DFS both subtrees, O(n)
  BST: Compare values, go left/right, O(h)  ← FASTER!

Example: Kth Smallest
  BT:  No shortcut, O(n)
  BST: Inorder traversal stops at k, O(h + k)  ← FASTER!
```

### 3.4 The Recursive Template for BST Problems

```cpp
void solve(TreeNode* root, /* extra params */) {
    if (!root) return;  // base case
    
    // OPTION A: Use BST property to go left OR right (O(h))
    if (target < root->val) solve(root->left, ...);
    else solve(root->right, ...);
    
    // OPTION B: Do inorder for sorted processing (O(n))
    solve(root->left, ...);   // process left
    /* process root */         // process current
    solve(root->right, ...);  // process right
}
```

## 4. Pattern 1: Inorder Traversal = Sorted Order ⭐

### 4.1 Core Insight

> **Inorder traversal (left → root → right) of a BST visits nodes in ascending order.**
> This is the single most powerful BST property.

```
BST:        5
           / \
          3   7
         / \ / \
        2  4 6  8

Inorder: 2, 3, 4, 5, 6, 7, 8  ← SORTED!
```

### 4.2 Kth Smallest Element in BST (LC 230) ⭐

```cpp
int kthSmallest(TreeNode* root, int k) {
    int count = 0, result = 0;
    
    function<void(TreeNode*)> inorder = [&](TreeNode* node) {
        if (!node || count >= k) return;
        inorder(node->left);
        count++;
        if (count == k) { result = node->val; return; }
        inorder(node->right);
    };
    
    inorder(root);
    return result;
}
// Time: O(h + k), Space: O(h)
```

**Iterative version (stack-based):**

```cpp
int kthSmallest(TreeNode* root, int k) {
    stack<TreeNode*> st;
    TreeNode* cur = root;
    
    while (cur || !st.empty()) {
        while (cur) {
            st.push(cur);
            cur = cur->left;  // go leftmost
        }
        cur = st.top(); st.pop();
        k--;
        if (k == 0) return cur->val;
        cur = cur->right;
    }
    return -1;
}
```

### 4.3 Inorder Successor in BST (LC 285)

```cpp
TreeNode* inorderSuccessor(TreeNode* root, TreeNode* p) {
    TreeNode* successor = nullptr;
    while (root) {
        if (p->val < root->val) {
            successor = root;       // candidate (might be the answer)
            root = root->left;
        } else {
            root = root->right;     // p is in left, successor must be right
        }
    }
    return successor;
}
// Time: O(h) — no full inorder needed!
// Key insight: Successor is the LEFTMOST node in right subtree,
//              OR the nearest ancestor for which p is in its left subtree.
```

### 4.4 BST Iterator (LC 173) ⭐

Implement `next()` and `hasNext()` for inorder traversal.

```cpp
class BSTIterator {
    stack<TreeNode*> st;
    
    void pushLeft(TreeNode* node) {
        while (node) {
            st.push(node);
            node = node->left;
        }
    }
    
public:
    BSTIterator(TreeNode* root) { pushLeft(root); }
    
    int next() {
        TreeNode* node = st.top(); st.pop();
        pushLeft(node->right);
        return node->val;
    }
    
    bool hasNext() { return !st.empty(); }
};
// next(): O(1) amortized, O(h) space
// Same as iterative inorder but split across calls
```

### 4.5 Two Sum in BST (LC 653)

```cpp
bool findTarget(TreeNode* root, int k) {
    // Method 1: Inorder → sorted array → two pointers
    vector<int> sorted;
    function<void(TreeNode*)> inorder = [&](TreeNode* n) {
        if (!n) return;
        inorder(n->left);
        sorted.push_back(n->val);
        inorder(n->right);
    };
    inorder(root);
    
    int l = 0, r = sorted.size() - 1;
    while (l < r) {
        int sum = sorted[l] + sorted[r];
        if (sum == k) return true;
        if (sum < k) l++;
        else r--;
    }
    return false;
}
// Method 2: Use two BST iterators (forward + reverse) for O(h) space
```

## 5. Pattern 2: BST Search Property (Go Left or Right)

### 5.1 Core Insight

> At every node, the BST property tells you EXACTLY where to go:
> - Target < current → go LEFT
> - Target > current → go RIGHT
> - Target == current → FOUND
>
> You never need to search both subtrees. This is what makes BST O(h), not O(n).

### 5.2 Search in BST (LC 700)

```cpp
TreeNode* searchBST(TreeNode* root, int val) {
    while (root) {
        if (val == root->val) return root;
        root = (val < root->val) ? root->left : root->right;
    }
    return nullptr;
}
// O(h) time, O(1) space
```

### 5.3 Insert into BST (LC 701)

```cpp
TreeNode* insertIntoBST(TreeNode* root, int val) {
    if (!root) return new TreeNode(val);
    if (val < root->val) root->left = insertIntoBST(root->left, val);
    else root->right = insertIntoBST(root->right, val);
    return root;
}
```

### 5.4 Delete from BST (LC 450)

```cpp
// Already covered in Section 2.3 — the 3 cases.
// Case 3 (two children) uses inorder successor.
```

### 5.5 Closest Binary Search Tree Value (LC 270)

```cpp
int closestValue(TreeNode* root, double target) {
    int closest = root->val;
    while (root) {
        if (abs(root->val - target) < abs(closest - target)) {
            closest = root->val;
        }
        root = (target < root->val) ? root->left : root->right;
    }
    return closest;
}
// O(h) time — BST search, updating closest along the way
```

## 6. Pattern 3: Validate & Bound Passing ⭐

### 6.1 Core Insight

> To validate a BST, it's NOT enough to check `left < root < right` for each node.
> You must ensure ALL nodes in the left subtree are less, and ALL in the right are greater.
> Solution: Pass **bounds (min, max)** down the recursion.

```
        5
       / \
      1   6
         / \
        3   7    ← INVALID! 3 is in right subtree of 5, but 3 < 5.

Checking only parent-child: 3 < 6 ✅ — WRONG answer!
Checking with bounds: 3 must be > 5 (right of 5) but 3 < 5 ❌ — CORRECT!
```

### 6.2 Validate Binary Search Tree (LC 98) ⭐⭐

```cpp
bool isValidBST(TreeNode* root) {
    return validate(root, LONG_MIN, LONG_MAX);
}

bool validate(TreeNode* node, long minVal, long maxVal) {
    if (!node) return true;
    
    if (node->val <= minVal || node->val >= maxVal) return false;
    
    return validate(node->left, minVal, node->val) &&     // left must be < node
           validate(node->right, node->val, maxVal);      // right must be > node
}
// Time: O(n), Space: O(h)
```

**Alternative: Inorder should be strictly increasing.**

```cpp
bool isValidBST(TreeNode* root) {
    TreeNode* prev = nullptr;
    return inorderCheck(root, prev);
}

bool inorderCheck(TreeNode* node, TreeNode*& prev) {
    if (!node) return true;
    if (!inorderCheck(node->left, prev)) return false;
    if (prev && node->val <= prev->val) return false;
    prev = node;
    return inorderCheck(node->right, prev);
}
```

### 6.3 Range Sum of BST (LC 938)

```cpp
int rangeSumBST(TreeNode* root, int low, int high) {
    if (!root) return 0;
    
    int sum = 0;
    if (root->val >= low && root->val <= high) sum += root->val;
    
    if (root->val > low) sum += rangeSumBST(root->left, low, high);
    if (root->val < high) sum += rangeSumBST(root->right, low, high);
    
    return sum;
}
// BST property: prune branches outside [low, high]
```

### 6.4 Trim BST (LC 669)

```cpp
TreeNode* trimBST(TreeNode* root, int low, int high) {
    if (!root) return nullptr;
    
    if (root->val < low) return trimBST(root->right, low, high);   // discard left
    if (root->val > high) return trimBST(root->left, low, high);  // discard right
    
    root->left = trimBST(root->left, low, high);
    root->right = trimBST(root->right, low, high);
    return root;
}
// Beautifully concise — BST pruning using bounds
```

## 7. Pattern 4: LCA and Ancestor Problems

### 7.1 Core Insight

> In a BST, finding the LCA is O(h) instead of O(n) because:
> - If both nodes are smaller → LCA is in left subtree.
> - If both nodes are larger → LCA is in right subtree.
> - If one is left and one is right (or one equals root) → root IS the LCA.

### 7.2 LCA of BST (LC 235) ⭐

```cpp
TreeNode* lowestCommonAncestor(TreeNode* root, TreeNode* p, TreeNode* q) {
    while (root) {
        if (p->val < root->val && q->val < root->val) {
            root = root->left;   // both in left subtree
        } else if (p->val > root->val && q->val > root->val) {
            root = root->right;  // both in right subtree
        } else {
            return root;         // split point = LCA
        }
    }
    return root;
}
// Time: O(h), Space: O(1)
// Compare with BT LCA which is O(n) with DFS on both subtrees!
```

### 7.3 BST LCA vs Binary Tree LCA

| | BST LCA | Binary Tree LCA |
|---|---|---|
| Time | O(h) | O(n) |
| Approach | Compare values, go left/right | DFS both subtrees |
| Key insight | BST ordering eliminates subtrees | Must search everywhere |

## 8. Pattern 5: BST Construction from Traversals

### 8.1 Convert Sorted Array to BST (LC 108) ⭐

**Intuition:** Middle element = root (for height-balanced BST). Recurse on left and right halves.

```cpp
TreeNode* sortedArrayToBST(vector<int>& nums) {
    return build(nums, 0, nums.size() - 1);
}

TreeNode* build(vector<int>& nums, int left, int right) {
    if (left > right) return nullptr;
    
    int mid = left + (right - left) / 2;
    TreeNode* root = new TreeNode(nums[mid]);
    root->left = build(nums, left, mid - 1);
    root->right = build(nums, mid + 1, right);
    return root;
}
// Time: O(n), Space: O(log n) — perfectly balanced!
```

### 8.2 Convert Sorted Linked List to BST (LC 109)

```cpp
TreeNode* sortedListToBST(ListNode* head) {
    if (!head) return nullptr;
    if (!head->next) return new TreeNode(head->val);
    
    // Find middle using slow/fast pointers
    ListNode *slow = head, *fast = head, *prev = nullptr;
    while (fast && fast->next) {
        prev = slow;
        slow = slow->next;
        fast = fast->next->next;
    }
    prev->next = nullptr;  // split list
    
    TreeNode* root = new TreeNode(slow->val);
    root->left = sortedListToBST(head);
    root->right = sortedListToBST(slow->next);
    return root;
}
```

### 8.3 Construct BST from Preorder Traversal (LC 1008)

**Intuition:** First element = root. Use bounds to decide when to stop.

```cpp
TreeNode* bstFromPreorder(vector<int>& preorder) {
    int idx = 0;
    return build(preorder, idx, INT_MIN, INT_MAX);
}

TreeNode* build(vector<int>& preorder, int& idx, int minVal, int maxVal) {
    if (idx >= preorder.size() || preorder[idx] < minVal || preorder[idx] > maxVal) {
        return nullptr;
    }
    
    TreeNode* root = new TreeNode(preorder[idx++]);
    root->left = build(preorder, idx, minVal, root->val);
    root->right = build(preorder, idx, root->val, maxVal);
    return root;
}
// Time: O(n) — each element visited once
// Key: Bound passing to determine subtree membership
```

### 8.4 Recover BST — Fix Two Swapped Nodes (LC 99) ⭐

**Intuition:** Inorder should be sorted. If two nodes are swapped, there will be violations in sorted order.

```cpp
void recoverTree(TreeNode* root) {
    TreeNode *first = nullptr, *second = nullptr, *prev = nullptr;
    
    function<void(TreeNode*)> inorder = [&](TreeNode* node) {
        if (!node) return;
        inorder(node->left);
        
        if (prev && node->val < prev->val) {
            if (!first) first = prev;  // first violation
            second = node;              // second violation (may update)
        }
        prev = node;
        
        inorder(node->right);
    };
    
    inorder(root);
    swap(first->val, second->val);
}
// O(n) time, O(h) space
// Adjacent swap: 1 violation. Non-adjacent: 2 violations.
```

## 9. Pattern 6: BST ↔ Sorted Data Structures

### 9.1 Convert BST to Sorted Doubly Linked List (LC 426) ⭐

**Intuition:** Inorder traversal. Rewire pointers: `left` → `prev`, `right` → `next`.

```cpp
TreeNode* treeToDoublyList(TreeNode* root) {
    if (!root) return nullptr;
    
    TreeNode *head = nullptr, *prev = nullptr;
    
    function<void(TreeNode*)> inorder = [&](TreeNode* node) {
        if (!node) return;
        inorder(node->left);
        
        if (prev) {
            prev->right = node;  // prev.next = node
            node->left = prev;   // node.prev = prev
        } else {
            head = node;  // first node = head
        }
        prev = node;
        
        inorder(node->right);
    };
    
    inorder(root);
    
    // Make circular
    head->left = prev;
    prev->right = head;
    
    return head;
}
```

### 9.2 Flatten BST to Sorted List / Increasing Order (LC 897)

```cpp
TreeNode* increasingBST(TreeNode* root) {
    TreeNode dummy(0);
    TreeNode* tail = &dummy;
    
    function<void(TreeNode*)> inorder = [&](TreeNode* node) {
        if (!node) return;
        inorder(node->left);
        
        node->left = nullptr;  // remove left pointer
        tail->right = node;    // attach to result list
        tail = node;
        
        inorder(node->right);
    };
    
    inorder(root);
    return dummy.right;
}
```

### 9.3 Balance a BST (LC 1382)

**Intuition:** Inorder → sorted array → build balanced BST from sorted array.

```cpp
TreeNode* balanceBST(TreeNode* root) {
    vector<int> sorted;
    
    function<void(TreeNode*)> inorder = [&](TreeNode* n) {
        if (!n) return;
        inorder(n->left);
        sorted.push_back(n->val);
        inorder(n->right);
    };
    inorder(root);
    
    return buildBalanced(sorted, 0, sorted.size() - 1);
}

TreeNode* buildBalanced(vector<int>& sorted, int l, int r) {
    if (l > r) return nullptr;
    int mid = l + (r - l) / 2;
    TreeNode* root = new TreeNode(sorted[mid]);
    root->left = buildBalanced(sorted, l, mid - 1);
    root->right = buildBalanced(sorted, mid + 1, r);
    return root;
}
// O(n) time + space. Simple but powerful.
```

### 9.4 Merge Two BSTs (LC 1305)

```cpp
vector<int> getAllElements(TreeNode* root1, TreeNode* root2) {
    vector<int> list1, list2;
    
    function<void(TreeNode*, vector<int>&)> inorder = [&](TreeNode* n, vector<int>& v) {
        if (!n) return;
        inorder(n->left, v);
        v.push_back(n->val);
        inorder(n->right, v);
    };
    
    inorder(root1, list1);
    inorder(root2, list2);
    
    // Merge two sorted arrays
    vector<int> result;
    int i = 0, j = 0;
    while (i < list1.size() && j < list2.size()) {
        if (list1[i] <= list2[j]) result.push_back(list1[i++]);
        else result.push_back(list2[j++]);
    }
    while (i < list1.size()) result.push_back(list1[i++]);
    while (j < list2.size()) result.push_back(list2[j++]);
    return result;
}
```

## 10. Balanced BSTs — AVL & Red-Black Trees

### 10.1 Why Balancing Matters

```
Unbalanced BST (insert 1,2,3,4,5):     Balanced BST:
1                                            3
 \                                          / \
  2                                        2   4
   \                                      /     \
    3                                    1       5
     \
      4         h = n → O(n) ops!       h = log n → O(log n) ops!
       \
        5
```

### 10.2 AVL Tree — Height-Balanced

**Rule:** For every node, `|height(left) - height(right)| ≤ 1`.

**Rotations:**

```
LEFT ROTATION (Right-heavy):        RIGHT ROTATION (Left-heavy):
    A                 B                 C                 B
     \               / \               /                 / \
      B      →      A   C            B          →      A   C
       \                             /
        C                           A

LEFT-RIGHT (LR):                    RIGHT-LEFT (RL):
    C          C          B            A          A          B
   /          /          / \            \          \        / \
  A    →     B    →     A   C            C    →    B  →   A   C
   \        /                           /           \
    B      A                           B              C
```

```cpp
// AVL Insert (conceptual)
struct AVLNode {
    int val, height;
    AVLNode *left, *right;
};

int getHeight(AVLNode* n) { return n ? n->height : 0; }
int getBalance(AVLNode* n) { return n ? getHeight(n->left) - getHeight(n->right) : 0; }

AVLNode* rightRotate(AVLNode* y) {
    AVLNode* x = y->left;
    y->left = x->right;
    x->right = y;
    y->height = max(getHeight(y->left), getHeight(y->right)) + 1;
    x->height = max(getHeight(x->left), getHeight(x->right)) + 1;
    return x;
}

AVLNode* leftRotate(AVLNode* x) {
    AVLNode* y = x->right;
    x->right = y->left;
    y->left = x;
    x->height = max(getHeight(x->left), getHeight(x->right)) + 1;
    y->height = max(getHeight(y->left), getHeight(y->right)) + 1;
    return y;
}

AVLNode* insert(AVLNode* node, int val) {
    if (!node) return new AVLNode{val, 1, nullptr, nullptr};
    
    if (val < node->val) node->left = insert(node->left, val);
    else if (val > node->val) node->right = insert(node->right, val);
    else return node;
    
    node->height = max(getHeight(node->left), getHeight(node->right)) + 1;
    int balance = getBalance(node);
    
    // Left Left → right rotate
    if (balance > 1 && val < node->left->val) return rightRotate(node);
    // Right Right → left rotate
    if (balance < -1 && val > node->right->val) return leftRotate(node);
    // Left Right → left rotate left, then right rotate node
    if (balance > 1 && val > node->left->val) {
        node->left = leftRotate(node->left);
        return rightRotate(node);
    }
    // Right Left → right rotate right, then left rotate node
    if (balance < -1 && val < node->right->val) {
        node->right = rightRotate(node->right);
        return leftRotate(node);
    }
    
    return node;
}
```

### 10.3 Red-Black Tree

| Property | Rule |
|---|---|
| 1 | Every node is Red or Black |
| 2 | Root is always Black |
| 3 | No two consecutive Red nodes (red parent can't have red child) |
| 4 | Every path from root to NULL has the same number of Black nodes |

> You almost NEVER implement RB trees manually. Use C++ `set`, `map`, `multiset` which are backed by Red-Black trees.

### 10.4 AVL vs Red-Black Comparison

| Feature | AVL | Red-Black |
|---|---|---|
| Balance strictness | Strict (height diff ≤ 1) | Relaxed (black-height balanced) |
| Search | Faster (shorter height) | Slightly slower |
| Insert/Delete | Slower (more rotations) | Faster (fewer rotations) |
| Used in | Databases, lookup-heavy apps | C++ STL, Java TreeMap, Linux kernel |
| Interview | Know concept + rotations | Know concept, use STL |

## 11. C++ STL: set, map, multiset, multimap

> These are all implemented as **Red-Black Trees** internally. They give you a balanced BST with O(log n) operations without manual balancing.

### 11.1 `set<T>` — Sorted Unique Elements

```cpp
set<int> s;
s.insert(5);  s.insert(3);  s.insert(7);  s.insert(3);  // {3, 5, 7}

s.count(5);          // 1 (exists)
s.count(4);          // 0
s.find(5);           // iterator to 5
s.erase(5);          // removes 5

*s.begin();          // 3 (min)
*s.rbegin();         // 7 (max)

s.lower_bound(4);    // iterator to 5 (first >= 4)
s.upper_bound(5);    // iterator to 7 (first > 5)
```

### 11.2 `map<K,V>` — Sorted Key-Value Pairs

```cpp
map<string, int> m;
m["apple"] = 3;
m["banana"] = 1;
m["cherry"] = 5;

for (auto& [key, val] : m) cout << key << ": " << val;
// apple: 3, banana: 1, cherry: 5  (sorted by key!)

m.lower_bound("b");   // iterator to "banana"
```

### 11.3 `multiset<T>` — Sorted with Duplicates

```cpp
multiset<int> ms;
ms.insert(5); ms.insert(3); ms.insert(5);  // {3, 5, 5}

ms.count(5);          // 2
ms.erase(ms.find(5)); // removes ONE 5 → {3, 5}
ms.erase(5);          // removes ALL 5s → {3}
```

### 11.4 When to Use STL vs Heap

| Need | Use |
|---|---|
| Only min/max, no searching | `priority_queue` (Heap) |
| Min/max + deletion of arbitrary elements | `set` / `multiset` |
| Sorted iteration | `set` / `map` |
| Key-value mapping, sorted keys | `map` |
| Duplicates allowed | `multiset` / `multimap` |
| Frequency counting, unsorted | `unordered_map` (Hash Map) |

## 13. Common Mistakes & Debugging

### 13.1 Top 10 BST Mistakes

| # | Mistake | Fix |
|---|---|---|
| 1 | **Checking only parent-child for valid BST** | Must check ALL descendants. Use bound passing (min, max). |
| 2 | **Forgetting inorder = sorted** | Always consider: "Would inorder traversal help here?" |
| 3 | **Not handling duplicates** | Decide: left ≤ root < right, or disallow duplicates? Be consistent. |
| 4 | **Deleting node with two children incorrectly** | Use inorder successor (leftmost in right subtree) or predecessor. |
| 5 | **Confusing BST with generic BT** | BST lets you go left OR right (not both). This is the speedup from O(n) to O(h). |
| 6 | **Assuming BST is always balanced** | Without balancing, insert 1,2,3,4,5 gives a linked list (h=n). |
| 7 | **Not using iterative approach** | For search/insert, iterative is often simpler and avoids stack overflow. |
| 8 | **Wrong bound types for validate** | Use `long` or `LONG_MIN/LONG_MAX` to handle INT_MIN/INT_MAX node values. |
| 9 | **Modifying tree while traversing** | Collect changes, apply after traversal. Or use careful pointer management. |
| 10 | **Using BT LCA approach for BST** | BST LCA is O(h) not O(n). Compare values, don't search both subtrees. |

### 13.2 Debugging Checklist

```
□ Is my BST actually using the BST property? (Go left OR right, not both)
□ Am I handling the base case: root == nullptr?
□ For validation: am I passing bounds (min, max) correctly?
□ For deletion: am I handling all 3 cases (leaf, one child, two children)?
□ For inorder: am I visiting left → root → right in the correct order?
□ Am I using long for bounds to avoid overflow with INT_MIN/INT_MAX?
□ Edge cases: empty tree, single node, all left (skewed), all right?
```

## 14. Interview FAQs

#### Q1: What makes BST different from a regular Binary Tree?

**A:** A BST has the **ordering invariant**: `left subtree < root < right subtree` for every node. This lets you search in O(h) instead of O(n). A generic binary tree has no ordering — you must search the entire tree.

#### Q2: What happens when a BST becomes unbalanced?

**A:** Operations degrade from O(log n) to O(n). Worst case: inserting sorted data (1,2,3,4,5) creates a linked list with h = n. Solution: use self-balancing BSTs (AVL, Red-Black) which guarantee h = O(log n) through rotations.

#### Q3: When should I use a BST vs a Hash Map?

**A:** Use a **Hash Map** (`unordered_map`) when you only need O(1) insert/find by key and don't need ordering. Use a **BST** (`map`/`set`) when you need:
- Sorted iteration
- Range queries (all elements in [lo, hi])
- `lower_bound` / `upper_bound` (nearest element)
- Kth smallest/largest

#### Q4: Why is inorder of BST sorted? Can you prove it?

**A:** By **induction**: 
1. Inorder visits left subtree first → all values < root (BST property).
2. Then root.
3. Then right subtree → all values > root.
Since left subtree and right subtree are also BSTs, recursively applying the same argument proves the entire inorder sequence is sorted.

#### Q5: How do I find the inorder successor efficiently?

**A:**
- **Case 1:** Node has a right child → successor is the leftmost node in right subtree.
- **Case 2:** Node has no right child → successor is the nearest ancestor where the node is in its LEFT subtree.
- **Using BST search:** Start from root. If `node.val < root.val`, root is a candidate, go left. Otherwise, go right. Last candidate = answer. O(h).

#### Q6: What are the 4 rotations in AVL trees?

**A:** Left-Left (right rotate), Right-Right (left rotate), Left-Right (left rotate child, then right rotate node), Right-Left (right rotate child, then left rotate node). You determine which rotation by checking the balance factor and which subtree caused the imbalance.

#### Q7: BST vs Heap — which to use when?

**A:** **Heap** = only need max/min repeatedly. O(1) peek, O(log n) insert/delete. **BST** = need search, ordered iteration, range queries, Kth element. O(log n) for all operations. If you need both kinds of operations, often `multiset` (BST-based) is more flexible than `priority_queue` (heap-based).

#### Q8: How does C++ `set` achieve O(log n)?

**A:** `set` is implemented as a **Red-Black Tree** (self-balancing BST). It guarantees O(log n) insert, delete, find, and ordered iteration. `unordered_set` uses a hash table for O(1) average but no ordering.

## 15. Real-World Scenarios

### 🔍 Scenario 1: Database Indexing (B-Trees)

**Problem:** _"How do databases execute `SELECT * FROM users WHERE age BETWEEN 20 AND 30` efficiently?"_

**Answer — DSA:** Databases use **B-Trees** (generalized BSTs) as indexes. The BST property enables range queries: find the first key ≥ 20, then scan right until > 30. Without an index, it's a full table scan O(n).

Used in: MySQL InnoDB, PostgreSQL, MongoDB indexes.

---

### 🔍 Scenario 2: File System Directory Structure

**Problem:** _"How does the OS find a file by name in a directory?"_

**Answer — DSA:** Many file systems use balanced BSTs (or B-Trees) to store directory entries sorted by name. Search is O(log n) per directory level instead of O(n) linear scan.

Used in: NTFS (Windows), ext4 (Linux), HFS+ (Mac).

---

### 🔍 Scenario 3: Autocomplete with Prefix Matching

**Problem:** _"Show all contacts starting with 'Jo' in sorted order."_

**Answer — DSA:** Store contacts in a BST (or Trie). Use `lower_bound("Jo")` to find the first match, then iterate forward until the prefix no longer matches. This is O(log n + k) where k = number of results.

Used in: Phone contacts, IDE autocomplete, search bars.

---

### 🔍 Scenario 4: Calendar / Event Scheduling

**Problem:** _"Check if a new meeting conflicts with existing ones."_

**Answer — DSA:** Store meeting intervals in a BST (`set<pair<int,int>>`). Use `lower_bound` to find the closest meeting. Check for overlap in O(log n). Much faster than scanning all meetings.

Used in: Google Calendar, Outlook, meeting room booking systems.

---

### 🔍 Scenario 5: Stock Price — Order Book

**Problem:** _"Match buy and sell orders at the best price."_

**Answer — DSA:** Buy orders in a max-BST (highest bid at top). Sell orders in a min-BST (lowest ask at top). When best bid ≥ best ask, a trade happens. BST allows O(log n) insert/delete/find.

Used in: Stock exchanges (NASDAQ, NYSE), crypto exchanges, forex platforms.

---

### 🔍 Scenario 6: Routing Tables (IP Lookup)

**Problem:** _"Find the most specific route matching an IP address."_

**Answer — DSA:** Longest prefix matching uses a BST-like structure (tries or binary trees). Routes are stored ordered by prefix. Binary search finds the matching route in O(log n).

Used in: Routers, network switches, BGP routing, DNS.

---

### 🔍 Scenario 7: Version Control — Git Commit History

**Problem:** _"Find the common ancestor of two branches."_

**Answer — DSA:** This is exactly the **Lowest Common Ancestor** problem! Git uses a DAG (directed acyclic graph) of commits. Finding the merge-base (LCA) of two branches determines where they diverged.

Used in: Git merge-base, GitHub pull request comparisons.

---

### 🔍 Scenario 8: Gaming — Leaderboard System

**Problem:** _"Show the current ranking. Insert new scores. Find a player's rank."_

**Answer — DSA:** Balanced BST (augmented with subtree sizes). Insert in O(log n). Rank = count of elements smaller + 1, computed in O(log n) using subtree sizes.

Used in: Online games, competitive programming platforms, sports rankings.

---

### 🔍 Scenario 9: Memory Allocator — Free Block Tracking

**Problem:** _"Find the smallest free memory block that fits a request."_

**Answer — DSA:** Store free blocks in a BST sorted by size. Use `lower_bound(requestSize)` to find the best-fit block in O(log n). Remove it, and if the block is larger than needed, split and re-insert the remainder.

Used in: OS memory allocators (malloc), garbage collectors, pool allocators.

---

### 🔍 Scenario 10: Ride-Sharing — Dynamic Pricing

**Problem:** _"Set price based on supply/demand in real-time."_

**Answer — DSA:** Track available drivers in a BST sorted by location/zone. `size()` gives supply count, request queue gives demand count. Ratio determines surge multiplier. BST enables fast add/remove as drivers go online/offline.

Used in: Uber surge pricing, Lyft, food delivery dynamic pricing.
