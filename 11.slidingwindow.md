## 1. What is Sliding Window?

### 1.1 The Definition

**Sliding Window** is an optimization technique primarily used on **arrays or strings**. It is a subset of the **Two Pointers** technique, where the two pointers act as the boundaries (`left` and `right`) of a "window". 

Instead of re-calculating information for every possible window from scratch (which takes $O(n^2)$ or $O(n^3)$), we slide the window by:
1. **Adding** the new element entering the window from the right.
2. **Removing** the old element leaving the window from the left.

### 1.2 The Core Tradeoff

| Approach | How it works | Time Complexity |
|---|---|---|
| **Brute Force** | Nested loops. Calculate sum/state for every subarray `i` to `j`. | $O(n^2)$ |
| **Sliding Window** | Maintain a window `[left, right]`. Update state incrementally. | $O(n)$ |

```
Array: [2, 1, 5, 1, 3, 2], k = 3

Brute Force:
[2, 1, 5] = 8
   [1, 5, 1] = 7  (re-added 1 and 5)
      [5, 1, 3] = 9
         [1, 3, 2] = 6

Sliding Window:
[2, 1, 5] = 8
    1. Subtract 2 (left element leaving)
    2. Add 1 (right element entering)
   [1, 5, 1] = 8 - 2 + 1 = 7  (constant time update!)
```

## 2. How to Identify Sliding Window Problems

Look for these **keywords and constraints** in the problem statement:

### 2.1 The 3 Conditions
1. **Linear Data Structure:** Arrays, strings, or linked lists.
2. **Contiguous Sequence:** Subarray, substring, or "consecutive elements". (If subsets or subsequences are asked, sliding window will NOT work. Use DP or Backtracking.)
3. **Target state to optimize or match:** 
   - Minimum/Maximum/Longest/Shortest length.
   - Exact sum, specific character count, or target condition.

### 2.2 Which Window Size?

| Problem Phrase | Window Type |
|---|---|
| "...subarray of size `k`..." | **Fixed-size Window** |
| "...longest subarray with sum `k`..." | **Variable-size Window** |
| "...shortest substring containing..." | **Variable-size Window** |
| "...number of subarrays with sum < `k`..." | **Variable-size Window** |

## 3. Pattern 1: Fixed-Size Window Template

In fixed-size window problems, the distance between `right` and `left` is always `k`. We first build a window of size `k`, and then slide it exactly one step at a time until the `right` pointer hits the end.

### 3.1 The Standard Template — MEMORIZE THIS

```cpp
int solveFixedWindow(vector<int>& arr, int k) {
    int left = 0;
    int right = 0;
    int current_state = 0; // Sum, count, map, etc.
    int ans = 0;           // Max, min, etc.
    
    while (right < arr.size()) {
        // 1. ADD: Include arr[right] into the window state
        current_state += arr[right];
        
        // 2. CHECK SIZE: If window is smaller than k, just grow it
        if (right - left + 1 < k) {
            right++;
        }
        // 3. WINDOW HIT: We have a window of size k
        else if (right - left + 1 == k) {
            // A. Update the answer based on current_state
            ans = max(ans, current_state);
            
            // B. REMOVE: Remove arr[left] from the window state
            current_state -= arr[left];
            
            // C. SLIDE: Move both pointers
            left++;
            right++;
        }
    }
    return ans;
}
```

## 4. Classic Fixed-Size Problems

### 4.1 Maximum Sum Subarray of Size K

**Problem:** Given an array of positive integers and a number `k`, find the maximum sum of a contiguous subarray of size `k`.

```cpp
int maxSumSubarrayOfSizeK(vector<int>& arr, int k) {
    int left = 0, right = 0;
    int sum = 0;
    int maxSum = INT_MIN;
    
    while (right < arr.size()) {
        sum += arr[right];
        
        if (right - left + 1 < k) {
            right++;
        } 
        else if (right - left + 1 == k) {
            maxSum = max(maxSum, sum);
            sum -= arr[left]; // remove element going out
            left++;           // slide window
            right++;
        }
    }
    return maxSum == INT_MIN ? 0 : maxSum;
}
// Time: O(n), Space: O(1)
```

### 4.2 First Negative Number in every Window of Size K

**Problem:** Find the first negative integer in every contiguous subarray of size `k`. If no negative integer exists, output `0`.

**Idea:** We need to keep track of the *order* of negative numbers. A `queue` is perfect. Store the negative numbers (or their indices) appearing in the current window.

```cpp
vector<int> firstNegativeInWindow(vector<int>& arr, int k) {
    int left = 0, right = 0;
    queue<int> q; // stores actual negative numbers
    vector<int> ans;
    
    while (right < arr.size()) {
        // Calculation
        if (arr[right] < 0) {
            q.push(arr[right]);
        }
        
        if (right - left + 1 < k) {
            right++;
        } 
        else if (right - left + 1 == k) {
            // Answer from current window
            if (q.empty()) ans.push_back(0);
            else ans.push_back(q.front());
            
            // Remove arr[left] from queue if it is the front element
            if (!q.empty() && arr[left] == q.front()) {
                q.pop();
            }
            
            // Slide
            left++;
            right++;
        }
    }
    return ans;
}
// Time: O(n), Space: O(k)
```

### 4.3 Count Occurrences of Anagrams (LC 438 - Find All Anagrams in a String)

**Problem:** Given string `txt` and string `pat`, return the starting indices of all anagrams of `pat` in `txt`.

**Idea:** An anagram means characters and their frequencies match. Use an array/map of size 26. `k = pat.length()`.

```cpp
vector<int> findAnagrams(string s, string p) {
    if (p.length() > s.length()) return {};
    
    vector<int> pCount(26, 0);
    vector<int> sCount(26, 0);
    int k = p.length();
    
    // Frequency map of pattern
    for (char c : p) pCount[c - 'a']++;
    
    vector<int> ans;
    int left = 0, right = 0;
    
    while (right < s.length()) {
        sCount[s[right] - 'a']++; // add
        
        if (right - left + 1 < k) {
            right++;
        } 
        else if (right - left + 1 == k) {
            if (sCount == pCount) ans.push_back(left); // compare frequencies (O(26) is O(1))
            
            sCount[s[left] - 'a']--; // remove
            left++;
            right++;
        }
    }
    return ans;
}
// Time: O(n), Space: O(1) (array size is fixed 26)
```

### 4.4 Maximum In Sliding Window (LC 239) — HARD

**Problem:** Return the max value from each sliding window of size `k`.
*Note: Using a max priority queue takes $O(n \log k)$. We can do $O(n)$ using a Deque.*

**Idea (Monotonic Deque):** Keep the deque strictly decreasing. If a new element comes in, it pops smaller elements from the back because they can never be the maximum anymore. The maximum is always at the `front`.

```cpp
vector<int> maxSlidingWindow(vector<int>& nums, int k) {
    deque<int> dq; // stores INIDICES
    vector<int> ans;
    int left = 0, right = 0;
    
    while (right < nums.size()) {
        // Calculation: maintain monotonic decreasing deque
        while (!dq.empty() && nums[dq.back()] < nums[right]) {
            dq.pop_back();
        }
        dq.push_back(right);
        
        if (right - left + 1 < k) {
            right++;
        } 
        else if (right - left + 1 == k) {
            // Answer is at front
            ans.push_back(nums[dq.front()]);
            
            // Remove arr[left] from deque if it's the front element
            if (dq.front() == left) {
                dq.pop_front();
            }
            
            left++;
            right++;
        }
    }
    return ans;
}
// Time: O(n) (each element pushed/popped at most once)
// Space: O(k)
```

## 5. Pattern 2: Variable-Size Window Template

In variable-size window problems, the window **grows** by moving `right` and **shrinks** by moving `left`. The window size is not fixed — it changes dynamically based on a **condition**.

### 5.1 Two Variants

| Variant | Goal | When to Shrink |
|---|---|---|
| **Longest / Maximum** | Find the longest subarray/substring satisfying a condition. | When the condition is **violated** → shrink from left. |
| **Shortest / Minimum** | Find the shortest subarray/substring satisfying a condition. | When the condition is **met** → shrink from left (try to do better). |

### 5.2 Template: Longest / Maximum — MEMORIZE THIS

```cpp
int longestWindow(vector<int>& arr, int target) {
    int left = 0, right = 0;
    int current_state = 0;
    int ans = 0;
    
    while (right < arr.size()) {
        // 1. ADD: Include arr[right] into window state
        current_state += arr[right];
        
        // 2. SHRINK: While condition is VIOLATED, remove from left
        while (current_state > target) {
            current_state -= arr[left];
            left++;
        }
        
        // 3. UPDATE: Current window is valid, update answer
        ans = max(ans, right - left + 1);
        
        right++;
    }
    return ans;
}
```

### 5.3 Template: Shortest / Minimum — MEMORIZE THIS

```cpp
int shortestWindow(vector<int>& arr, int target) {
    int left = 0, right = 0;
    int current_state = 0;
    int ans = INT_MAX;
    
    while (right < arr.size()) {
        // 1. ADD: Include arr[right] into window state
        current_state += arr[right];
        
        // 2. SHRINK: While condition is MET, try to minimize
        while (current_state >= target) {
            ans = min(ans, right - left + 1);  // update INSIDE the while
            current_state -= arr[left];
            left++;
        }
        
        right++;
    }
    return ans == INT_MAX ? 0 : ans;
}
```

> 🧠 **Key difference:** For "Longest", update `ans` AFTER the while loop (window is valid). For "Shortest", update `ans` INSIDE the while loop (every valid smaller window is a candidate).

### 6.1 Longest Subarray with Sum ≤ K

```cpp
int longestSubarrayWithSumAtMostK(vector<int>& arr, int k) {
```

### 6.3 Longest Substring Without Repeating Characters (LC 3) ⭐

```cpp
int lengthOfLongestSubstring(string s) {
    unordered_map<char, int> charIndex; // char → last seen index
    int left = 0, ans = 0;
    
    for (int right = 0; right < s.length(); right++) {
        // If char was seen and is inside current window, move left
        if (charIndex.count(s[right]) && charIndex[s[right]] >= left) {
            left = charIndex[s[right]] + 1;
        }
        charIndex[s[right]] = right;
        ans = max(ans, right - left + 1);
    }
    return ans;
}
// Time: O(n), Space: O(min(n, 26))
```

**Alternative using set:**
```cpp
int lengthOfLongestSubstring(string s) {
    unordered_set<char> window;
    int left = 0, ans = 0;
    
    for (int right = 0; right < s.length(); right++) {
        while (window.count(s[right])) {
            window.erase(s[left]);
            left++;
        }
        window.insert(s[right]);
        ans = max(ans, right - left + 1);
    }
    return ans;
}
```

### 6.6 Max Consecutive Ones III (LC 1004) ⭐

**Problem:** Given binary array and number `k`, return the maximum consecutive `1`s if you can flip at most `k` `0`s.

```cpp
int longestOnes(vector<int>& nums, int k) {
    int left = 0, ans = 0;
    int zeros = 0;
    
    for (int right = 0; right < nums.size(); right++) {
        if (nums[right] == 0) zeros++;
        
        while (zeros > k) {
            if (nums[left] == 0) zeros--;
            left++;
        }
        
        ans = max(ans, right - left + 1);
    }
    return ans;
}
```

## 7. Pattern 3: Sliding Window with Hash Map

### 7.1 Minimum Window Substring (LC 76) — HARD ⭐⭐

**Problem:** Given strings `s` and `t`, find the minimum window in `s` which contains ALL characters of `t` (including duplicates).

```cpp
string minWindow(string s, string t) {
    if (t.length() > s.length()) return "";
    
    unordered_map<char, int> need, have;
    for (char c : t) need[c]++;
    
    int required = need.size();  // number of unique chars needed
    int formed = 0;              // number of unique chars with desired frequency
    int left = 0;
    int ansLen = INT_MAX, ansLeft = 0;
    
    for (int right = 0; right < s.length(); right++) {
        char c = s[right];
        have[c]++;
        
        // Check if this char's frequency matches what's needed
        if (need.count(c) && have[c] == need[c]) {
            formed++;
        }
        
        // Shrink while we have a valid window
        while (formed == required) {
            // Update answer
            if (right - left + 1 < ansLen) {
                ansLen = right - left + 1;
                ansLeft = left;
            }
            
            // Remove from left
            char leftChar = s[left];
            have[leftChar]--;
            if (need.count(leftChar) && have[leftChar] < need[leftChar]) {
                formed--;
            }
            left++;
        }
    }
    
    return ansLen == INT_MAX ? "" : s.substr(ansLeft, ansLen);
}
// Time: O(|s| + |t|), Space: O(|s| + |t|)
```

### 7.2 Longest Repeating Character Replacement (LC 424) ⭐

**Problem:** Given string `s` and integer `k`, you can replace at most `k` characters. Return the length of the longest substring with all same characters.

```cpp
int characterReplacement(string s, int k) {
    int freq[26] = {};
    int left = 0, ans = 0;
    int maxFreq = 0;  // frequency of most common char in current window
    
    for (int right = 0; right < s.length(); right++) {
        freq[s[right] - 'A']++;
        maxFreq = max(maxFreq, freq[s[right] - 'A']);
        
        // Window size - most common char count = chars to replace
        // If chars to replace > k, shrink
        int windowSize = right - left + 1;
        if (windowSize - maxFreq > k) {
            freq[s[left] - 'A']--;
            left++;
        }
        
        ans = max(ans, right - left + 1);
    }
    return ans;
}
// Time: O(n), Space: O(1)
// Key insight: window is valid when (window_size - max_freq_in_window) <= k
```

### 7.3 Permutation in String (LC 567)

**Problem:** Return true if `s2` contains a permutation of `s1`.

```cpp
bool checkInclusion(string s1, string s2) {
    if (s1.length() > s2.length()) return false;
    
    vector<int> s1Count(26, 0), s2Count(26, 0);
    for (char c : s1) s1Count[c - 'a']++;
```

### 7.4 Subarrays with K Different Integers (LC 992) — HARD

**Problem:** Return the number of "good subarrays" of `nums` that have exactly `k` different integers.

> 🧠 **The "Exactly K" trick:**
> `exactly(K) = atMost(K) - atMost(K - 1)`

```cpp
int atMost(vector<int>& nums, int k) {
    unordered_map<int, int> freq;
    int left = 0, count = 0;
    
    for (int right = 0; right < nums.size(); right++) {
        freq[nums[right]]++;
        
        while (freq.size() > k) {
            freq[nums[left]]--;
            if (freq[nums[left]] == 0) freq.erase(nums[left]);
            left++;
        }
        
        count += right - left + 1;  // all subarrays ending at right
    }
    return count;
}

int subarraysWithKDistinct(vector<int>& nums, int k) {
    return atMost(nums, k) - atMost(nums, k - 1);
}
// Time: O(n), Space: O(k)
```

### 7.5 Count Number of Nice Subarrays (LC 1248)

**Problem:** Return the number of subarrays with exactly `k` odd numbers.

```cpp
// Same "exactly K = atMost(K) - atMost(K-1)" trick
int atMost(vector<int>& nums, int k) {
    int left = 0, count = 0, odds = 0;
    for (int right = 0; right < nums.size(); right++) {
        if (nums[right] % 2 == 1) odds++;
        while (odds > k) {
            if (nums[left] % 2 == 1) odds--;
            left++;
        }
        count += right - left + 1;
    }
    return count;
}

int numberOfSubarrays(vector<int>& nums, int k) {
    return atMost(nums, k) - atMost(nums, k - 1);
}
```

### 7.6 Binary Subarrays with Sum (LC 930)

```cpp
// Count subarrays with sum exactly = goal
// Same pattern: exactly(goal) = atMost(goal) - atMost(goal - 1)
int atMost(vector<int>& nums, int goal) {
    if (goal < 0) return 0;
    int left = 0, count = 0, sum = 0;
    for (int right = 0; right < nums.size(); right++) {
        sum += nums[right];
        while (sum > goal) {
            sum -= nums[left];
            left++;
        }
        count += right - left + 1;
    }
    return count;
}

int numSubarraysWithSum(vector<int>& nums, int goal) {
    return atMost(nums, goal) - atMost(nums, goal - 1);
}
```

## 8. Pattern 4: Two Pointers (Directional)

Two Pointers is the parent technique. Sliding Window is a specific case where both pointers move in the **same direction**. Two Pointers also covers the opposite-direction case.

### 8.1 Two Types of Two Pointers

```
Same Direction (Sliding Window):
left →                          right →
[═══════════ window ═══════════]

Opposite Direction:
left →                          ← right
[━━━━━━━━━━━━━━━━━━━━━━━━━━━━]
```

### 8.2 Two Sum II — Sorted Array (LC 167)

```cpp
vector<int> twoSum(vector<int>& numbers, int target) {
    int left = 0, right = numbers.size() - 1;
    while (left < right) {
        int sum = numbers[left] + numbers[right];
        if (sum == target) return {left + 1, right + 1}; // 1-indexed
        else if (sum < target) left++;
        else right--;
    }
    return {};
}
// Time: O(n), Space: O(1)
```

### 8.3 3Sum (LC 15) ⭐

```cpp
vector<vector<int>> threeSum(vector<int>& nums) {
    sort(nums.begin(), nums.end());
    vector<vector<int>> result;
    
    for (int i = 0; i < (int)nums.size() - 2; i++) {
        if (i > 0 && nums[i] == nums[i-1]) continue; // skip dups
        
        int left = i + 1, right = nums.size() - 1;
        while (left < right) {
            int sum = nums[i] + nums[left] + nums[right];
            if (sum == 0) {
                result.push_back({nums[i], nums[left], nums[right]});
                while (left < right && nums[left] == nums[left+1]) left++;
                while (left < right && nums[right] == nums[right-1]) right--;
                left++; right--;
            }
            else if (sum < 0) left++;
            else right--;
        }
    }
    return result;
}
// Time: O(n²), Space: O(1) excluding output
```

### 8.4 Container With Most Water (LC 11) ⭐

```cpp
int maxArea(vector<int>& height) {
    int left = 0, right = height.size() - 1;
    int ans = 0;
    
    while (left < right) {
        int area = min(height[left], height[right]) * (right - left);
        ans = max(ans, area);
        
        if (height[left] < height[right]) left++;
        else right--;
    }
    return ans;
}
// Time: O(n), Space: O(1)
// Why: Moving the shorter side inward can potentially find a taller wall.
```

### 8.5 Trapping Rain Water (LC 42) ⭐⭐

```cpp
int trap(vector<int>& height) {
    int left = 0, right = height.size() - 1;
    int leftMax = 0, rightMax = 0;
    int water = 0;
    
    while (left < right) {
        if (height[left] < height[right]) {
            if (height[left] >= leftMax) leftMax = height[left];
            else water += leftMax - height[left];
            left++;
        } else {
            if (height[right] >= rightMax) rightMax = height[right];
            else water += rightMax - height[right];
            right--;
        }
    }
    return water;
}
// Time: O(n), Space: O(1)
```

### 8.6 Remove Duplicates from Sorted Array (LC 26)

```cpp
int removeDuplicates(vector<int>& nums) {
    if (nums.empty()) return 0;
    int slow = 0;
    for (int fast = 1; fast < nums.size(); fast++) {
        if (nums[fast] != nums[slow]) {
            slow++;
            nums[slow] = nums[fast];
        }
    }
    return slow + 1;
}
// Time: O(n), Space: O(1)
// "slow" and "fast" pointers — same direction
```

### 8.7 Sort Colors / Dutch National Flag (LC 75)

```cpp
void sortColors(vector<int>& nums) {
    int low = 0, mid = 0, high = nums.size() - 1;
    while (mid <= high) {
        if (nums[mid] == 0) swap(nums[low++], nums[mid++]);
        else if (nums[mid] == 1) mid++;
        else swap(nums[mid], nums[high--]);
    }
}
// Time: O(n), Space: O(1), Single pass!
```

### 8.8 Two Pointers Pattern Summary

| Pattern | Pointer Movement | Use When |
|---|---|---|
| **Fixed Sliding Window** | Both → (same dir, fixed gap) | "Subarray of size K" |
| **Variable Sliding Window** | Both → (same dir, variable gap) | "Longest/shortest subarray with condition" |
| **Opposite Direction** | ← left, right → | Sorted arrays, pair finding, area problems |
| **Slow & Fast** | Both → (different speeds) | Remove dups, cycle detection, middle |

## 10. Common Mistakes & Debugging

### 10.1 Top 10 Sliding Window Mistakes

| # | Mistake | Fix |
|---|---|---|
| 1 | **Using nested loops (brute force)** | Recognize the sliding window pattern; add right, remove left |
| 2 | **Forgetting to shrink from left** | Always have a `while(condition violated) { left++ }` |
| 3 | **Updating answer at the wrong time** | Longest → update AFTER shrink loop. Shortest → update INSIDE shrink loop. |
| 4 | **Not removing left element from state** | When `left++`, also `freq[s[left]]--` or `sum -= arr[left]` |
| 5 | **Erasing from map when freq hits 0** | Always `if (freq[x] == 0) freq.erase(x);` so `freq.size()` is accurate. |
| 6 | **Off-by-one: window size** | Window size = `right - left + 1`, NOT `right - left`. |
| 7 | **Confusing fixed vs variable** | Fixed: window exactly K. Variable: window grows/shrinks by condition. |
| 8 | **Not handling empty string/array** | Check `if (arr.empty()) return 0;` upfront. |
| 9 | **Using wrong data structure** | Need order of negatives → Queue. Need max → Deque. Need freq → Map. |
| 10 | **Forgetting the "atMost" trick** | "Exactly K" ≠ direct sliding window. Use `atMost(K) - atMost(K-1)`. |

### 10.2 The Shrink Direction Rule

```
┌─────────────────────────────────────────────────────────┐
│        LONGEST / MAXIMUM Problems                       │
│  → Shrink when condition is INVALID                     │
│  → Update answer AFTER the while loop                   │
│  → ans = max(ans, right - left + 1)                     │
├─────────────────────────────────────────────────────────┤
│        SHORTEST / MINIMUM Problems                      │
│  → Shrink when condition is VALID (try to do better)    │
│  → Update answer INSIDE the while loop                  │
│  → ans = min(ans, right - left + 1)                     │
└─────────────────────────────────────────────────────────┘
```

### 10.3 Debugging Checklist

```
□ Is my left pointer ever moving backward? (It should only move forward.)
□ Am I correctly adding arr[right] to the state before checking?
□ Am I correctly removing arr[left] from the state when shrinking?
□ Is my window size calculation right? (right - left + 1)
□ For "exactly K": Am I using the atMost(K) - atMost(K-1) trick?
□ Am I updating the answer at the correct location?
□ Edge cases: empty input, k = 0, all same elements?
```

## 11. Interview FAQs

#### Q1: What's the difference between Sliding Window and Two Pointers?

**A:** Sliding Window IS a type of Two Pointers where both pointers move in the **same direction** (left to right). Two Pointers is the broader technique that also includes **opposite-direction** pointers (e.g., Two Sum in sorted array, Container With Most Water). All sliding window problems are two-pointer problems, but not vice versa.

#### Q2: How do I know if a problem is Fixed-Size or Variable-Size?

**A:** If the problem says "window/subarray of size K" → Fixed. If it says "longest/shortest subarray such that..." → Variable. If it says "count subarrays with exactly K..." → Variable + atMost trick.

#### Q3: When does Sliding Window NOT work?

**A:** Sliding window requires a **monotonic property** — as the window grows, the condition should only get "worse" (or stay the same), and as it shrinks, it should only get "better." It does NOT work when:
- Array has **negative numbers** and you're looking for sum conditions (use prefix sum + hash map instead).
- You need **non-contiguous** elements (use DP or backtracking).
- The condition is not monotonic (adding elements can both help and hurt).

#### Q4: What's the "atMost(K) - atMost(K-1)" trick?

**A:** Sliding window naturally finds "at most K" conditions. To get "exactly K", compute: `exactly(K) = atMost(K) - atMost(K-1)`. This works because `atMost(K)` counts windows with 0,1,...,K distinct elements, and `atMost(K-1)` counts 0,1,...,K-1. Subtracting gives exactly K.

#### Q5: How do I count the number of valid subarrays?

**A:** For each position of `right`, the number of valid subarrays ending at `right` is `right - left + 1`. Sum this up across all positions. This counts all subarrays `[left..right], [left+1..right], ..., [right..right]`.

#### Q6: Why use a Deque for Sliding Window Maximum?

**A:** A priority queue (heap) can't efficiently remove the element leaving the window (it takes O(log n) and requires lazy deletion). A **monotonic deque** keeps elements in decreasing order and supports O(1) push/pop from both ends. The max is always at the front. Elements are popped from the back if smaller than the incoming element (they can never be the max again).

#### Q7: Can Sliding Window work on linked lists?

**A:** Technically yes, but it's uncommon. Sliding window requires O(1) access to the elements at both boundaries, which linked lists don't provide efficiently. It's almost always used on arrays or strings.

#### Q8: What's the time complexity of Sliding Window?

**A:** O(n) for most problems. Even though there's a nested `while` loop, each element is added at most once and removed at most once. Total operations = 2n = O(n). The inner while loop doesn't make it O(n²) because `left` only moves forward across the entire execution.

## 12. Real-World Scenarios

### 🔍 Scenario 1: Network Packet Rate Limiting

**Problem:** _"Only allow 100 API requests per second per user."_

**Answer — DSA:** Fixed sliding window of 1 second. Count requests in the current second. If count exceeds 100, reject. Slide the window as time passes.

Used in: API gateways (Nginx, AWS API Gateway), DDoS protection, rate limiters.

---

### 🔍 Scenario 2: Video Streaming — Buffering

**Problem:** _"Maintain a 10-second buffer ahead of the current playback position."_

**Answer — DSA:** Sliding window on video frames. The left boundary is the current playback position. The right boundary is 10 seconds ahead. As playback advances, old frames are discarded (left moves) and new frames are loaded (right moves).

Used in: YouTube, Netflix, Spotify, all streaming services.

---

### 🔍 Scenario 3: Stock Price — Moving Average

**Problem:** _"Calculate the 50-day moving average of a stock price."_

**Answer — DSA:** Fixed-size sliding window of 50 elements. Add new day's price, subtract the price from 50 days ago. Each update is O(1).

```
Day 51 average = (Day 50 average * 50 - Day 1 price + Day 51 price) / 50
```

Used in: Trading platforms, financial analytics, time series analysis.

---

### 🔍 Scenario 4: Text Editor — Find and Replace

**Problem:** _"Find all occurrences of a pattern in a document."_

**Answer — DSA:** Sliding window of `pattern.length()`. Slide across the text, comparing character by character. (KMP and Rabin-Karp are optimized versions of this.)

Used in: Every text editor (Ctrl+F), `grep`, search engines.

---

### 🔍 Scenario 5: CDN Cache — Sliding Window Log

**Problem:** _"Track the most popular 10 resources in the last hour."_

**Answer — DSA:** Variable-size sliding window on access times. Keep a log of (timestamp, resource) pairs. As time passes, remove entries older than 1 hour from the left. Maintain a frequency map to find top-K.

Used in: CDN analytics, trending topics, real-time dashboards.

---

### 🔍 Scenario 6: DNA Sequence Analysis

**Problem:** _"Find all 10-letter-long sequences (substrings) that occur more than once in a DNA string."_ (LC 187)

**Answer — DSA:** Fixed-size sliding window of 10 characters. Store each window in a hash set. If already seen, it's a repeated sequence.

Used in: Bioinformatics, genome analysis, BLAST algorithm.

---

### 🔍 Scenario 7: Network Congestion — TCP Window

**Problem:** _"How many bytes can a sender transmit before waiting for an acknowledgment?"_

**Answer — DSA:** TCP uses a **sliding window protocol**. The sender maintains a window of bytes it can send. As acknowledgments come back (left advances), new bytes can be sent (right advances). Window size grows/shrinks based on congestion.

```
Sent & Acked | Sent & Unacked | Can Send | Cannot Send
[───────────] [════════════════] [--------] [............]
              left              right
              ← sliding window →
```

Used in: TCP/IP networking, flow control, congestion control.

---

### 🔍 Scenario 8: Log Analysis — Error Spike Detection

**Problem:** _"Alert if more than 50 errors occur in any 5-minute window."_

**Answer — DSA:** Variable-size sliding window on error timestamps. Keep errors in a deque. Remove errors older than 5 minutes from the front. If deque size > 50, trigger alert.

Used in: Monitoring systems (Datadog, Grafana), alerting, SRE tools.

---

### 🔍 Scenario 9: Gaming — Combo System

**Problem:** _"Player must press 5 specific keys in sequence within 2 seconds for a combo."_

**Answer — DSA:** Sliding window on key presses with a time constraint. Window contains recent key presses. If window matches the combo pattern and time elapsed ≤ 2s, trigger combo.

---

### 🔍 Scenario 10: Recommendation Engine — Session Analysis

**Problem:** _"Find the longest user browsing session where they stayed within the same product category."_

**Answer — DSA:** Variable-size sliding window. `right` extends the session. When the user visits a different category, shrink from `left`. Track the longest such window.

Used in: Amazon, Netflix, Spotify recommendation engines.
